# 1) 目標・資源（Why / With what）

* 目的・KPIの一点集中

  * 決める：最上位KPIを1つ（例：30日後の保存数・登録者増・資料請求など）＋補助指標2つ（視聴維持率/ER）。
  * 既定値：短期は「保存率」「視聴維持率」を最重視。
* 週あたり工数バジェット

  * 決める：合計○時間/週、プラットフォーム優先度。
  * 既定値：合計6–8h/週。優先：YouTube（Shorts＋長尺）＞Instagram（Reels/ストーリーズ）＞X（拡散/対話）。
* スキル・機材・環境

  * 決める：撮影環境（固定照明/マイク）、編集ワークフロー、テンプレの有無。
  * 既定値：撮影は固定カメラ＋定常照明。編集はテンプレ化（イントロ/アウトロ/テロップ）。

# 2) コンテンツ設計（What / Format / Tone）

* コンテンツ柱（3〜5本）

  * 決める：連載できる柱のみ採用。各柱で10本ネタが即出るかで判定。
  * 既定値：①超短ハウツー②検証/比較③Vlog（長尺）④失敗談・学び⑤ライブQ\&A。
* 尺・フォーマット・工数上限

  * 決める：1本あたり上限時間、縦横、字幕方針。
  * 既定値：ショート≤60分/本、長尺Vlog≤10時間/本（Premiere可）。縦動画基軸、字幕は自動生成＋誤字だけ手直し。
* フックと構成テンプレ

  * 決める：冒頭3秒の約束、型（Hook→価値→証拠→CTA）。
  * 既定値：Hookは「驚きデータ」「ビフォー→アフター」「やりがち失敗」の3型を使い回し。

# 3) 運用プロセス（How to make）

* SOP（1人用）

  1. ネタ出し（30分）→ 2. 台本ドット（要点箇条書き15分）→ 3. まとめ撮り（60–90分）→
  2. バッチ編集（120–180分）→ 5. サムネ/タイトル（60分）→ 6. 投稿＆拡散（各10分）→ 7. 返信（投稿後15分×2回）。
* バッチ運用

  * 既定値：週1日で翌週分ショートを撮り溜め。長尺は隔週で撮影→翌週編集。
* 再利用設計（一次制作→二次活用）

  * 既定値：長尺→クリップ3本→ショート化→IG Reels→Xに要点テキスト。B-roll素材はループ再利用。

# 4) 配信・拡散（Where / Distribution）

* チャネル役割分担

  * YouTube：検索/深掘り、IG：発見/保存、X：拡散/対話。
* 投稿カレンダー（MVC：最小努力線）

  * 既定値：ショート4本/週（火・木・土・日 19時）、長尺1本/2週（日 18時）、IGストーリーズ3回/週、Xポスト3回/週。
* メタデータ運用

  * 既定値：タイトルは“成果/失敗×具体数値×期限”を含む。タグは固定＋企画固有で上限内に。

# 5) 計測・改善（Measure / Learn）

* 指標と閾値

  * 既定値：ショート視聴維持率35%未満が3本連続→フック差し替え。保存率2%超→同テーマに倍配分。長尺は平均視聴時間8分超を合格ライン。
* 週次レビュー（30分）

  * 勝ちパターン抽出→来週の配分変更→捨てる企画を決定。A/Bはサムネ→タイトル→Hookの順で。

# 6) 収益・リスク（Monetize / Risk）

* 収益化の里程標

  * 既定値：まず“指名検索/メール獲得”を中間KPIに。無料リード磁石（チェックリスト/テンプレ）→LP導線。
* 権利・炎上対策

  * 既定値：BGM/画像は権利クリアのみ、PRは明記、個人特定/医療・法律表現はガイドライン準拠。簡易レビュー表で事前チェック。
