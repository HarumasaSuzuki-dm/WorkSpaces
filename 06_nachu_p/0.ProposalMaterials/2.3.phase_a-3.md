### **提案資料作成のための情報収集・整理タスクリスト（時系列版）**

#### **大タスク1：提案の基盤となる戦略・ストーリーの確定**
（目的：資料作成に着手する前に、伝えるべきメッセージの核を固め、論理的な一貫性を確保する）

*   **小タスク1.1：インプット情報の再整理と最終確認**
    *   [ ] `0.Prerequisites.md`と`2.1.phase_a.md`を読み込み、クライアントの課題、ニーズ、そして自身の提供価値に関する仮説を再確認する。
    *   [ ] `1.Issues.md`を基に、「課題→原因→解決策」の論理構造が、提案全体を貫く背骨として機能することを再認識する。
    *   [ ] **競合パーソナルジムの市場調査（簡易版）**
        *   [ ] 商圏（〇〇駅周辺）の競合ジム3〜5社をリストアップする。
        *   [ ] **ポジショニング分析：** 各ジムの「ターゲット顧客」「提供価値」「価格帯」を調査し、整理する。
        *   [ ] **Web/SNS発信状況の分析：** 特にYouTubeでの「発信の有無」「コンテンツ内容」「集客への動線」を調査し、整理する。

*   **小タスク1.2：コアメッセージとキービジュアルの構想**
    *   [ ] **【攻めのメッセージ】** 斉藤氏の専門性をどう言語化し、YouTubeで「誰に・何を・どのように」届けるかの核心部分を一行で表現できるように準備する。（例：「コンテストを目指す層に、日本トップレベルの指導を届ける専門チャンネル」）
    *   [ ] **【守りのメッセージ】** 業務効率化がもたらす価値を一行で表現できるように準備する。（例：「事務作業をゼロにし、トレーナー業務に100%集中できる環境を構築」）
    *   [ ] **【キービジュアルの構想】** 提案の核となる「攻守両輪」のコンセプトと、「理想の事業サイクル」をどのように図解するか、手書きのラフスケッチなどでイメージを固める。

#### **大タスク2：各パートのコンテンツ作成と具体化**
（目的：確定した戦略に基づき、骨子案の各要素を具体的なスライドコンテンツとして作成・言語化する）

*   **小タスク2.1：【現状分析】パートの言語化と図解**
    *   [ ] **課題の言語化：** 「新規顧客獲得の壁」「事業運営の壁」について、クライアントが「まさにその通りだ」と共感するような具体的な言葉で説明文を作成する。
    *   [ ] **課題の定量化：** クライアントの現状業務にかかる時間の推定を行い、「痛み」を可視化する。
    *   [ ] **原因の構造化：** 「価値の未伝達」と「スケールできない業務プロセス」の因果関係を、ロジカルで分かりやすい図（相関図など）に落とし込み、その解説文を作成する。
    *   [ ] **市場環境の視覚化：** 調査結果を基に、「ポジショニングマップ」または「競合比較表」を作成し、クライアントの独自の価値と勝機を明確に伝えるための解説文を作成する。

*   **小タスク2.2：【解決策】パートの具体化と視覚化**
    *   [ ] **解決策の全体像の視覚化：** 小タスク1.2で構想した「攻守両輪」のコンセプト図を、プレゼンテーションツール上で実際に作成する。
    *   [ ] **成功後の未来像の描写：** 小タスク1.2で構想した「理想の事業サイクル」の図を作成し、プロジェクト成功後のポジティブな変化が具体的にイメージできるような説明文を添える。
    *   [ ] **投資対効果（ROI）の概算：** プロジェクト成功後の事業インパクトを概算し、投資価値を提示する。

*   **小タスク2.3：【実行計画】パートの詳細化**
    *   [ ] **ロードマップの作成：** 3つのフェーズ（戦略設計、実行体制の構築、事業拡大）それぞれの期間の目安と、主要なアウトプットを時系列で示したロードマップ図を作成する。
    *   **各フェーズのタスクの具体化：** 各フェーズで実施するタスク内容について、クライアントが「これをやれば成功しそうだ」と納得できるレベルまで、具体的なアクションを記述する。

*   **小タスク2.4：【クロージング】パートのオファー作成**
    *   [ ] **パートナーシップの定義：** 「戦略的パートナー」として、具体的にどのような役割を担うのか（例：月1回の戦略会議、週次の進捗確認など）を明文化する。
    *   **ヒアリング項目の選定：** 提案の場でクライアントとの対話を生むための「簡易ヒアリング」の質問を、最終的な3項目に絞り込む。
    *   **初回アクションの具体化：** 「最初のステップ」として提案するキックオフミーティングで「何を」「どこまで」決めるのか、そのゴールを明確に言語化する。特に、「初期費用不要」という強力なオファーの伝え方を洗練させる。
    *   [ ] **協力依頼事項の明文化：** プロジェクト成功のためにクライアントに協力をお願いしたいこと（ミーティング時間の確保など）を具体的に記述する。
    *   [ ] **リスクの事前提示：** 想定されるリスク（例：成果が出るまでの期間）と、その対策を誠実に伝えるための表現を準備する。

#### **大タスク3：資料の統合と最終レビュー**
（目的：作成した各パートを統合し、一つのプレゼンテーションとして完成度を高める）

*   **小タスク3.1：全体の統合とデザイン調整**
    *   [ ] 作成したすべてのスライドを一つのファイルに統合し、ページ番号を振る。
    *   [ ] 全体を通してデザイン（フォント、色、レイアウト）に一貫性を持たせる。

*   **小タスク3.2：論理・表現の最終チェック**
    *   [ ] 全体を通して読み返し、ストーリーの流れがスムーズか、論理に飛躍がないかを確認する。
    *   [ ] 誤字脱字や不適切な表現がないかを徹底的に校正する。
    *   [ ] 信頼できる第三者（いれば）にレビューを依頼し、客観的なフィードバックを得る。
