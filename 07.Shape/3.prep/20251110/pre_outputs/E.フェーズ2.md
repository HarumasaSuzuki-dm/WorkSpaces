## 1. 研修コンセプト（案）

  * **研修名称（案）:** SHAPE ハイパフォーマンス・リーダー研修
  * **対象:** 管理職（※優先設計）
  * **コンセプト:**
    パーソナルトレーニングで培った\*\*「SHAPEメソッド（やり切らせる伴走力）」\*\*を、管理職のリーダーシップ開発に応用します。
  * **提供価値:**
    1.  **マインドセット変容:** 「自己理解→挑戦」を起点に、ヒアリング課題であった「数字」を「統制可能な“挑戦の結果”」として再定義します。
    2.  **行動変容:** 「理念」を「日次の行動（先行KPI）」にまで分解し、現場での実践（4週間スプリント）を徹底的に伴走支援します。
    3.  **成果（売上接続）:** 行動変容（先行KPI）が、いかにして業務KPI、ひいては売上向上に接続するかを「評価設計」として明示し、オーナーの投資対効果（ROI）に応えます。

## 2. モジュール設計表（Draft）

「理念→KPI→売上」のロジックチェーン（自己理解→挑戦→数字意識→売上向上）と、ヒアリング課題（給与固定、ビジョン未接続）への対応を、具体的なモジュールと先行KPIとして設計しました。

| モジュール | 目的・ゴール | 主要アクティビティ（例） | 成果物（例） | 先行KPI（計測例） |
| :--- | :--- | :--- | :--- | :--- |
| **M0: 導入設計** | 期待値の合意と測定設計の確定 | 研修ゴール説明、測定KPIの確定、データ取得合意 | 研修合意メモ | 測定同意率 |
| **M1: 自己理解** | 自己の価値観・強みの言語化 | バリューカード、強み・弱みの棚卸し | 価値観キャンバス | 価値観言語化スコア |
| **M2: ビジョン接続** | 個人ビジョンと会社ビジョンの接続点を発見 | 両ビジョンのマッピングWS | 接続ステートメント | ビジョン接続度サーベイ |
| **M3: 数字の再定義** | 「数字」＝「挑戦の結果」としてマインドセット変容 | ケース討議（数字と行動の接続）、自部門KPIの分解 | 自部門KPI分解表 | **前日売上把握率** |
| **M4: 行動設計** | 「挑戦→数字」へ落とし込む4週間の行動計画策定 | IF-THENプランニング、障害予防策の検討 | 4週間スプリント計画 | 週次行動実行率 |
| **M5: 現場実践** | 計画の実行と内省サイクル（PDCA）の定着 | 週次15分の1on1伴走、実行ログの記録 | 実行ログ | **施策実行サイクル** |
| **M6: 振り返り** | 事前事後の変化を可視化し、行動を「型」として内製化 | KPIレビュー、Good/Bad事例共有 | 事後レポート | **改善提案回数** |

## 3. 提供フロー図（Draft）

モジュールを時系列に配置し、「導入→研修（WS）→実践（OJT）→振り返り」のフローを明確化しました。

```
[ W0: 導入/設計 ] ──> [ W1: 本編WS (M1-4) ] ──> [ W2-4: 現場実践 (M5) ] ──> [ W5: 振り返り (M6) ]
  評価KPIの合意        価値観→接続→数字→行動設計        週次1on1伴走 / 実行ログ       事前事後の比較・効果測定
```

## 4. 評価設計（Draft）

「オーナー視点での効果測定」「売上への接続ロジック」を最重要視し、数値で続行可否を判断できる設計案とします。

### 4-1. 評価の構成

| タイミング | 評価項目 | 測定指標（例） |
| :--- | :--- | :--- |
| **事前 (Baseline)** | 業務KPI（結果） | 部門売上、粗利、LTV、稼働率など |
| | **先行KPI（行動）** | **前日売上把握率**、**改善提案回数**、**施策実行サイクル** |
| | 定性サーベイ | 自己効力感、数値回避傾向、ビジョン接続度 |
| **運用中 (During)** | プロセス指標 | 週次行動実行率、実行ログ完遂率、1on1実施率 |
| **事後 (Post)** | **変化量（最重要）** | **先行KPIの事前事後差分**（例：売上把握率 +30pt） |
| | 業務KPIへの影響 | 業務KPIのトレンド変化、意思決定リードタイム短縮 |
| | ROI（仮説） | 研修費用（例：50万/月）に対する回収期間の試算 |

### 4-2. "理念→KPI→売上" 接続ロジック（案）

研修の核となる「自己理解→挑戦→数字意識→売上向上」のロジックを、具体的な指標で接続します。

| 理念要素 | 行動（先行KPI） | 業務KPI（結果） | 売上接続ロジック（仮説） |
| :--- | :--- | :--- | :--- |
| **自己理解 → 挑戦** | **改善提案回数**（挑戦の量）<br>**施策実行サイクル**（挑戦の速さ） | 施策成功率、<br>CVR、生産性 | 挑戦の量と質が向上し、改善速度が上がることで、業務KPIが改善し、売上向上に寄与する。 |
| **数字意識の向上** | **前日売上把握率**<br>（数字への意識） | チーム売上、粗利 | 数字を「自分事」として把握することで、次の行動（改善提案）への感度が高まり、結果として売上改善につながる。 |

## 5. 体制・役割一覧（Draft）

SHAPE側とクライアント（先方）側の役割（R\&R）を明確化します。

| ロール | 主担当 | 主要責務 |
| :--- | :--- | :--- |
| **講師 (リード)** | SHAPE | M1-4の研修設計・登壇、M6の総括、研修品質の管理 |
| **ファシリテーター** | SHAPE | 研修当日の演習進行、場の設計、議論の促進 |
| **1on1コーチ** | SHAPE | M5（現場実践）における週次15分の伴走、障害除去、ログ管理 |
| **データ/評価担当** | SHAPE | 評価設計、データ（KPI）のダッシュボード整備、事前事後の測定・分析 |
| **事務局** | SHAPE / 先方 | 日程調整、参加者管理、資料準備、ログ回収の周知 |
| **先方決裁者 (オーナー)** | クライアント | 目的・ゴールの合意、測定KPIの承認、研修成果のジャッジ |
| **先方現場リード** | クライアント | 参加者アサイン、現場での実行サポート、SHAPEへの障害共有 |
