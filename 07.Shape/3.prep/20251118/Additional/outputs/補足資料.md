## 補足3：費用発生タイミングと「合トレ」活用シナリオ

本戦略の重要な顧客獲得チャネルとして、なーすけ様が参加する「リアルバリュー（合トレ）」を最大限に活用します。ここでの接点は、一般的な営業活動とは異なる「高単価・高信頼」な契約（=パイロット導入）に直結する重要な機会です。

**基本思想:**

  * **Phase A/B（信頼構築）:** 0円。経営者仲間として「個人」の課題解決（SHAPEメソッドの体験）に貢献し、信頼を獲得します。
  * **Phase C/D（法人契約）:** 有料。その「個の変容」のロジックを「組織（管理職）」へ展開する提案を行い、法人契約（パイロット）に切り替えます。

-----

### シナリオA：「組織課題」起点（経営者の悩みに直結させる）

経営者が「管理職の数字意識」や「理念浸透」に明確な課題認識を持っている場合の、最も直接的なシナリオ。

| フェーズ | フロー | 活用する研修モジュール（思考） | 費用 |
| :--- | :--- | :--- | :--- |
| **Phase A: 関係構築** | 合トレ後の雑談（サウナなど）で、経営者が「うちの管理職は数字（売上）意識が低くて…」という悩みを漏らす。 | （傾聴）<br>※VEATM社の**課題1「数字意識の希薄さ」** と同じニーズ。 | **0円** |
| **Phase B: 変化実感（デモ）** | なーすけ様がSHAPEの「型」を提供。<br>「その悩みは、数字を『ノルマ』と捉えているからです。SHAPEでは**『M3: 数字の再定義』** という手法で、数字を『挑戦の結果（フィードバック）』として捉え直させます」とメソッドの一部を無償でコンサル（デモ）する。 | **M3: 数字の再定義**<br>**型①: 商品開発の型** | **0円**<br>（法人契約へのデモ） |
| **Phase C: パイロット導入** | 経営者が「そのメソッドを、うちの管理職5名に試したい」と反応。<br>「では、御社向けにROIロジック を設計し、1ヶ月のパイロット（M1〜M5） を実行しましょう」と提案。 | **M1〜M5のフルパッケージ**<br>（VEATM社モデル） | **ここから費用発生**<br>（例：有償 50万円）<br>※法人契約 |
| **Phase D: 本格導入** | パイロット成功（Gate基準クリア） 後、6ヶ月の標準パッケージを契約。 | - | **標準価格で費用発生** |

-----

### シナリオB：「経営者個人の変容」起点（パーソナルメソッドの応用）

経営者自身が「個人の変容（CJM）」 に興味を持っている場合の、SHAPEの強みを活かすシナリオ。

| フェーズ | フロー | 活用する研修モジュール（思考） | 費用 |
| :--- | :--- | :--- | :--- |
| **Phase A: 関係構築** | 合トレで、経営者がなーすけ様の「自己の軸」 に基づく在り方に共感する。 | （なーすけ様自身の体現） | **0円** |
| **Phase B: 変化実感（デモ）** | 経営者個人に対し、「パーソナルカウンセリング（SHAPEメソッドの原型）」 を無償で提供。<br>経営者自身が**「M1: 自己理解」** と**「M2: ビジョン接続」** のプロセスを体験し、「自分軸」 が明確になる感覚を得る。 | **M1: 自己理解**<br>**M2: ビジョン接続** | **0円**<br>（法人契約へのデモ） |
| **Phase C: パイロット導入** | 経営者が「この『自己理解→ビジョン接続』の体験を、自社の幹部にも受けさせたい」と反応。<br>「ありがとうございます。この『個の変容』を『企業の成果（ROI）』に接続するのが法人研修です」と、M3〜M5を付加したパイロットを提案。 | **M1〜M5のフルパッケージ** | **ここから費用発生**<br>（例：有償 50万円）<br>※法人契約 |
| **Phase D: 本格導入** | パイロット成功後、標準パッケージを契約。 | - | **標準価格で費用発生** |

-----

### シナリオC：「ROIロジック」起点（成果コミットへの懐疑）

経営者が「研修は効果がない（ROIが不明瞭）」と懐疑的な場合の、逆張りシナリオ。

| フェーズ | フロー | 活用する研修モジュール（思考） | 費用 |
| :--- | :--- | :--- | :--- |
| **Phase A: 関係構築** | 合トレ後の雑談で、経営者が「最近流行りの1on1とか研修とか、結局『やりっぱなし』で売上に繋がらない」と発言。 | （傾聴） | **0円** |
| **Phase B: 変化実感（デモ）** | なーすけ様が「全く同感です。だからSHAPEは『やりっぱなし』を防ぐ**『型③: 成果測定の型』** を設計しました」と返答。<br>VEATM社の事例 を見せながら、**「先行KPIの設計」と「Gate基準（判定基準）」** のロジックを説明。「成果が出なければ続行不要」と断言する。 | **型③: 成果測定の型**<br>（VEATM社のROIロジック） | **0円**<br>（法人契約へのデモ） |
| **Phase C: パイロット導入** | 経営者が「そこまで言うなら（成果を数値で証明できるなら）、一度試してみる価値はある」と反応。<br>「では、御社専用のROIロジックを設計するM0（導入設計） から始めましょう」と、パイロット導入を提案。 | **M0: 導入設計（ROI設計）** がフックとなる。 | **ここから費用発生**<br>（例：有償 50万円）<br>※法人契約 |
| **Phase D: 本格導入** | パイロット成功後、標準パッケージを契約。 | - | **標準価格で費用発生** |
