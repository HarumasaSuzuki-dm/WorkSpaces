# 【v0.1 ドラフト】P2：課題可視化組織図（たたき台）

**目的:** 本資料は、「組織課題の構造マップ(ID 2.5)」で特定した根本原因（役割重複、業務の隙間、指揮命令系統の曖昧さなど）を、現状の組織構造（As-Is）上にマッピングし、課題の発生箇所を視覚的に特定することを目的とします。

**作成者:** 鈴木、中井
**作成期間:** W10-W13

---

## 2\. 表記ルール（凡例）

| 記号 / 線 / 色          | 課題カテゴリ（憲章より）                  | 意味・使用方法                                             |
| :---------------------- | :---------------------------------------- | :--------------------------------------------------------- |
| **赤色の破線**          | ② R\&R：役割の重複                        | 重複している役割（ボックス）間を接続する                   |
| **`[VACANT]` ボックス** | ② R\&R：業務の隙間                        | 担当者不在の業務・機能をボックスとして明記する             |
| **`?` マーク**          | ① ガバナンス：指揮命令系統の曖昧さ        | 曖昧なレポートライン（実線）上、またはボックス間に配置する |
| **赤色ハイライト**      | ① ガバナンス：リソース集中 / 意思決定遅延 | 承認や業務が過度に集中しているボックスを強調表示する       |
| **青色の波線**          | ③ エコシステム：内部と外部の連携不備      | 連携ルールが未整備な内部と外部のボックス間を接続する       |
