### 2. プロジェクトの目的とゴール

#### 2-1. 背景と課題

現在、会計上のPL（見かけ）と実質PL（実態）に大きな乖離（約24,000千円）が発生しています。また、外注費や原価の中身がブラックボックス化しており、どこに最適化の余地があるか判断できない状態です。この「感覚経営」の状態が、正しい経営判断を妨げる最大のリスクとなっています。

#### 2-2. プログラム全体における役割

本プロジェクトは、プログラム全体の「守り」を担当します。「正しい数値に基づく経営判断の土台」を築き、コスト構造を最適化することで、利益率を改善する役割を担います。

#### 2-3. プロジェクトの目的

本プロジェクトの目的は、以下のように定義します。

> **“正しい数字で経営を判断できる”状態をつくる。**
> （＝感覚ではなく構造で意思決定するための土台を整備する）

#### 2-4. 成功の定義（3ヶ月後）

マイルストーン①（3ヶ月完了時点）で、以下の状態を実現します。

  * 経営陣が「今、何が利益を押し下げているのか」を、実質PLとコスト構造分析レポートに基づき、**構造で説明できる**。
  * 次の「戦略実行フェーズ」において、“どのレバーを引くべきか”（どのコストを削減し、どこに投資すべきか）を、**KPIツリーを用いて議論できる状態**になっている。

### 3. プロジェクト・スコープ（〜3ヶ月）

#### 3-1. やること（In Scope）

マイルストーン①（診断・設計フェーズ）の完了に必要な、以下3つのフェーズを実行します。

1.  **Phase 0: データ整備・実態把握**
      * 会計事務所との調整（業務委託費の除外処理など）
      * PL再分類と「実質PL（v1）」の試算
2.  **Phase 1: コスト構造の可視化**
      * 外注費・原価の明細化と分類
      * 費用対効果分析と「コスト構造分析レポート」の作成
3.  **Phase 2: 経営判断基盤（KPI構造）設計**
      * 実質PLを起点とした「KPIツリー（v1）」の設計
      * 主要レバーを定義した「経営判断マップ」の作成
      * 3ヶ月後以降を見据えたデータ取得体制のロードマップ策定

#### 3-2. やらないこと（Out of Scope）

本プロジェクト（〜3ヶ月）では、「診断」と「設計」に集中し、以下の活動は**行いません**。

  * 「戦略レバーマップ」に基づく、コスト削減施策や投資施策の**本格的な実行**
  * 経営ダッシュボードの**システム導入**（※構造設計までとする）
  * 収益ポートフォリオの再配分（※Phase 3以降のタスク）

### 4. 主要成果物と受入基準（Definition of Done）

| 主要成果物 | 受入基準（＝これが出来たら完了） |
| :--- | :--- |
| **1. 実質PL (v1)** | ・会計事務所と調整合意が取れ、経営実態と無関係な費用（業務委託費等）が除外・再分類されている |
| **2. コスト構造分析レポート** | ・外注費・原価が明細化され、「外注構造マップ」として可視化されている<br>・「削減仮説」が明記され、経営陣が議論できる状態になっている |
| **3. KPIツリー (v1)＋経営判断マップ** | ・実質PL（売上・原価・固定費）と連動したKPI構造が設計されている<br>・経営判断の主要レバー（例：CVR、LTV、外注比率等）が定義されている |
| **4. データ取得体制ロードマップ** | ・3ヶ月後以降のデータ取得・分析体制に関する課題が特定され、対応方針のロードマップが合意されている |

### 7. 主要なリスク（新規追加）

本プロジェクト（マイルストーン①）の成功を妨げる可能性がある主要なリスクは以下の通りです。

  * **R1: 正しい数値の確定遅延リスク**
      * **内容:** Phase 0における会計事務所との調整 が難航し、「実質PL（v1）」の確定が遅れる。
      * **影響:** この成果物は後続タスク（コスト分析、KPI設計）の絶対的な前提であるため、プロジェクト全体のスケジュールが遅延する。
  * **R2: リソースのボトルネック化リスク**
      * **内容:** 本プロジェクトの実行責任者（しおん様） は、プログラム全体のキーパーソンでもある。他プロジェクトの業務が集中した場合、本プロジェクトのデータ整備や分析作業が停滞する。
      * **影響:** 経営判断の土台となる数値の確定が遅れ、意思決定の遅延に直結する。
