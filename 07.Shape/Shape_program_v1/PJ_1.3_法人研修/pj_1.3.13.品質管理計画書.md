## 🛡️ プロジェクト「法人研修事業」 個別 品質管理計画（案）

| ドキュメント名 | 個別 品質管理計画（案） |
| :--- | :--- |
| プロジェクト名 | 法人研修事業立ち上げ（攻め）プロジェクト |
| 実行責任者 (PM) | なーすけ様 |
| 作成日 | （案）2025年11月12日 |
| バージョン | 1.0 (素案) |

## 1\. 本計画の目的

  * 本書は、プロジェクト「法人研修事業」で作成される主要成果物が、スポンサー（なーすけ様）の期待、およびクライアント（VEATM社）の要求を満たし、「完了（受入）」と判断されるための品質基準と検証プロセスを定義します。
  * 本書の基準は、プロジェクトスコープ・ベースライン（`pj_3.0.3.スコープ・ベースライン.md`） および関連資料 で定義された「あるべき姿」に基づいています。

## 2\. 品質管理の体制

| 役割 | 氏名 | 主な責務 |
| :--- | :--- | :--- |
| **受入責任者** | **なーすけ様 (Sponsor)** | 内部成果物（マテリアル、レポート）の\*\*最終承認（受入）**を行う。 |
| 品質保証 | なーすけ様 (PM) | 成果物が受入基準を満たす品質であるかを**日常的に担保（保証・レビュー）**する。 |
| 品質管理 | 鈴木（支援） | 成果物（ROIロジック、分析レポート）作成プロセスにおいて、定義された品質基準を**管理（コントロール）\*\*する。 |

## 3\. 主要成果物と品質検証プロセス

本プロジェクトでは、以下の3つの主要成果物 の品質を検証します。

### 成果物 1：VEATM社向け提案資料

  * **品質基準（受入基準）**
      * 1.  SHAPEの「3つの価値」 が反映されていること。
      * 2.  VEATM社のヒアリング課題（例：数字意識の希薄さ） に対応したモジュール（M1-M5） が設計されていること。
      * 3.  WBS 2.2 で確定した「ROIロジック」と「判定基準（Gate）」 が明記されていること。
  * **品質検証プロセス**
      * **検証活動:** WBS タスク 2.5「提案資料作成とリハーサル」
      * **検証タイミング:** （案）W6（M1承認時）
      * **検証方法:** 鈴木（支援）がROIロジック 部分を、なーすけ様（PM）が全体をレビュー。最終的にVEATM社に提案し、**パイロット導入の承認（Goサイン）** をもって「受入」とする。

### 成果物 2：研修マテリアル (v1)

  * **品質基準（受入基準）**
      * 1.  提案資料 で提示した5つのモジュール（M1-M5） の内容を網羅していること。
      * 2.  SHAPEの「3つの型」（商品開発、実行、成果測定） が反映されたマテリアルであること。
  * **品質検証プロセス**
      * **検証活動:** WBS タスク 2.3「研修マテリアルの具体化」
      * **検証タイミング:** （案）W6（WBS 3.2「本編WS」 の実施前まで）
      * **検証方法:** なーすけ様（PM）が講師目線で、鈴木（支援）が設計思想 との一貫性目線でレビューし、なーすけ様が承認する。

### 成果物 3：効果測定レポート（パイロット結果）

  * **品質基準（受入基準）**
      * 1.  WBS 2.2 で定義した「先行KPI」のPre-Post（事前事後）比較 が含まれていること。
      * 2.  WBS 2.2 で定義した「判定基準（Gate）」 に基づき、パイロットの「続行／拡張／ピボット」 の評価が明確に記載されていること。
      * 3.  「理念」が「行動」に変わった「再現性ある成功事例」 が言語化されていること。
  * **品質検証プロセス**
      * **検証活動:** WBS タスク 3.5「レポート作成とパイロット続行判定」
      * **検証タイミング:** （案）W13（M2完了時）
      * **検証方法:** 鈴木（支援）と しおん様（データ） が分析・作成したレポートを、なーすけ様（PM/Sponsor）がレビューし、Gate判定結果を含めて最終承認する。
