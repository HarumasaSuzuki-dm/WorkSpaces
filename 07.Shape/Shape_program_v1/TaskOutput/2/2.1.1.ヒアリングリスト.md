## 【資料1：ヒアリングリスト】

**文書名:** PJ2.0 組織診断 (Phase 0) ヒアリングリスト
**目的:** 根本原因の特定 に向け、インタビュー実施者（鈴木）が仮説の検証とファクト収集を行うためのガイド。
**対象者:** インタビュー実施者（鈴木）
**管理:** 本資料は、具体的な質問仮説を含むため、原則として**内部資料**とし、インタビュー対象者には共有しない。

-----

### 1. インタビュー実施の基本方針

1.  **「症状」と「原因」の分離:**
      * 対象者が語る事象（例：「Aさんの承認が遅い」）は「**症状**」として捉える。
      * 実施者は、その背景にある「**構造的な原因**」（例：Aさんに承認が集中するR\&Rになっている、ガバナンスが不明確）を深掘りする。
2.  **仮説の検証:**
      * 憲章 やタスク定義 に基づく3つの論点（ガバナンス、R\&R、エコシステム） を「根本原因の仮説」とし、それらを検証するための質問を行う。
3.  **網羅性 (MECE):**
      * タスク定義（`2.1.1.md`）の品質期待値 にある通り、3つの論点を網羅的にカバーする。

### 2. 全体共通質問（アイスブレイク）

| No. | 質問項目 | 質問の意図（確認したいこと） |
| :--- | :--- | :--- |
| 0-1 | （アイスブレイク）まず、現在の組織や体制について、「これは強みだ」「非常にうまくいっている」と感じる点があれば教えてください。 | ポジティブな側面から入ることで、現状認識のバランス（良い点・悪い点）を確認する。 |
| 0-2 | 逆に、ご自身の業務を進める上で「最も非効率だ」あるいは「無駄が多い」と感じるプロセスや業務はありますか？ | 対象者が最も課題認識を持っている「症状」のトップ3を把握する。 |

### 3. 論点別：深掘り質問リスト（仮説ベース）

**【論点1：ガバナンス】**

  * *仮説：意思決定プロセスや指揮命令系統の曖昧さが、「意思決定の遅延」や「リソースの無駄」を生んでいる。*

| No. | 質問項目 | 質問の意図（確認したいこと） |
| :--- | :--- | :--- |
| G-1 | 最近、ご自身が関与した業務で「意思決定に時間がかかった（遅延した）」と感じる具体例はありますか？ | 「遅延」の症状が発生している具体的な業務プロセスを特定する。 |
| G-2 | (G-1の深掘り) その際、承認プロセス（誰の承認が必要か）は明確でしたか？どのステップで時間がかかりましたか？ | 承認プロセスの「明確性」 と「ボトルネック」の箇所を特定する。 |
| G-3 | （経営層・管理者向け）現在の組織図や体制において、指揮命令系統（誰が誰に報告・指示するか）が曖昧になっていると感じる部分はありますか？ | 組織図（たたき台）作成 に向けた、現状の課題認識を収集する。 |
| G-4 | （現場向け）業務で困った際、「これは誰に相談・報告すべきか」迷うことはありますか？ | G-3を現場視点で確認する。 |

**【論点2：R\&R (役割と責任)】**

  * *仮説：役割の「重複」や「隙間」が、非効率や業務の取りこぼしを生んでいる。*

| No. | 質問項目 | 質問の意図（確認したいこと） |
| :--- | :--- | :--- |
| R-1 | （経営層向け）憲章にもある「CMOとSFF」の役割について、それぞれに期待されている役割（責任範囲）はどのように定義されていますか？ | 経営陣の「戦略的意図（あるべき姿）」を確認する。 |
| R-2 | （現場・横断向け）「この業務は、AさんとBさんのどちらが担当なのか？」と役割が重複していて迷う、あるいは二重対応になっている事例はありますか？ | 「役割の重複」 が発生している具体的な業務を特定する。 |
| R-3 | （現場・横断向け）逆に「この業務は、本来誰がやるべきなのか？」と担当者が不明確で、ボールが落ちてしまう（業務の隙間）事例はありますか？ | 「業務の隙間」 が発生している具体的な業務を特定する。 |
| R-4 | （現場向け）「リソースの無駄」 について、0-2で伺った以外に具体例があれば教えてください。（例：同じ資料を別々の人が作っている、など） | RACI設計 のインプットとなる「無駄」の具体例を収集する。 |

**【論点3：エコシステム】**

  * *仮説：内部組織と外部パートナーが協働する際の「連携ルール」が未整備なため、無駄や非効率が発生している。*

| No. | 質問項目 | 質問の意図（確認したいこと） |
| :--- | :--- | :--- |
| E-1 | 外部パートナー（業務委託、コンサルなど）と連携する際、どのようなプロセス（依頼、進捗確認、検収など）で業務を進めていますか？ | 「As-Is（現状）」の連携プロセスを把握する。 |
| E-2 | その連携プロセスにおいて、非効率や手戻り（例：期待値と違うものが納品される、など）が発生している部分はありますか？ | ルール未整備による「症状」を特定する。 |
| E-3 | 「内部でやるべきこと」と「外部に任せるべきこと」の切り分けが曖昧なことで、問題（コスト増や品質低下など）が起きた事例はありますか？ | 「内部と外部の管理の混在」 に起因する課題を特定する。 |
