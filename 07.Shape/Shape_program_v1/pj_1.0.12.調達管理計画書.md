## プログラム 調達管理計画書 (v1.0)

**作成日:** 2025年11月12日
**作成者:** 鈴木（パートナーコンサルタント）
**承認者:** なーすけ様（スポンサー）、中井様（プログラムPM）

### 1\. 本計画書の目的

本計画書は、3つのプロジェクト（守り・組織・攻め）の実行において発生するすべての外部調達（外部コンサルティング、業務委託、パートナー連携、ツール購入など）を、**プログラム全体で一貫したプロセスで管理**するために作成するものです。

プロジェクトごとに個別の判断で調達が行われることを防ぎ、コスト管理の統一化、コンプライアンスの担保、外部パートナーとの関係性の一元管理を行うための「**統一ルールブック**」として機能します。

### 2\. 調達管理の役割と責任

本プログラムにおける調達関連の役割と責任を、既存の体制に基づき以下の通り定義します。

| 役割 | 担当者 | 主な責務 |
| :--- | :--- | :--- |
| **最終承認者**<br>（スポンサー） | **なーすけ様** | ・プログラム総予算の承認<br>・新規の大型契約や、予算枠を超える調達に関する最終的な意思決定 |
| **予算執行管理者** | **しおん様** | ・スポンサー承認済みの経営予算の**執行管理**を担当する<br>・各プロジェクトからの調達要求が予算内であるかを確認・管理する |
| **調達起案・検収責任者** | **各プロジェクトの実行責任者**<br>（守り：しおん様、組織：中井様、攻め：なーすけ様） | ・プロジェクトに必要な調達（外注）のニーズを特定し、起案する<br>・契約したベンダーのパフォーマンスを管理し、成果物を「検収」（受入）する |
| **調達プロセス連携** | **内部：「調達・購買・経理」** | ・（クライアントの）社内規定に基づく契約実務（見積取得、発注、支払い）の実行<br>・本プログラムの調達活動との実務的な連携 |
| **PMO支援** | **鈴木（コンサル）** | ・調達プロセスのファシリテーション支援<br>・例：会計事務所との調整支援、ベンダー選定の論点整理 |

### 3\. 既知の調達・パートナー連携ニーズ

本プログラム開始時点（M1）で特定されている主要な調達・連携対象は以下の通りです。

| 対象プロジェクト | 調達・連携対象 | 分類 | 概要 |
| :--- | :--- | :--- | :--- |
| 全体 | 外部コンサルティング費用 | 業務委託 | 本プログラムのPMO支援およびコンサルティング業務（鈴木） |
| 守り | 会計事務所 / R社(CAO) | 業務委託 | Phase 0の「実質PL」確定に向けたデータ整備・調整作業 |
| 攻め | VEATM社 | パートナー連携 | Phase 2（パイロット実行）の連携先 |
| 全体 | フレアネスト社 (CTO) | 外部パートナー | プログラム全体に関わるシステム・技術基盤の連携 |
| 組織 | 香 (CHRO) | 外部パートナー | 組織基盤強化プロジェクトに関する外部知見の連携 |

### 4\. 調達管理の統一プロセスとルール

本プログラムにおけるすべての調達は、以下の統一プロセスに従って実行・管理されます。

1.  **【起案】（各PM）**

      * 各プロジェクトの実行責任者（しおん様、中井様、なーすけ様）は、必要な調達のスコープ、予算、納期を定義し、調達起案書を作成します。

2.  **【連携・確認】（予算執行管理者＋内部部署）**

      * 起案書は、まず予算執行管理者（しおん様） に提出され、プログラム全体の予算枠との整合性が確認されます。
      * 同時に、社内の「調達・購買・経理」部門 と連携し、社内規定のプロセス（相見積もり、契約書レビュー等）を開始します。

3.  **【審議・決定】（プログラム会議体）**

      * 新規のベンダー契約や、一定金額（※別途定義）を超える調達、予算超過の可能性がある案件は、**「週次プログラム定例」の正式なアジェンダ**として上程されます。
      * プログラムPM（中井様） が論点を整理し、PMO（鈴木） が支援します。

4.  **【最終承認】（スポンサー）**

      * 「週次プログラム定例」での審議に基づき、スポンサー（なーすけ様）が調達の実行（または却下）を\*\*最終承認（裁定）\*\*します。

5.  **【契約・発注】（内部部署）**

      * 承認された調達は、社内の「調達・購買・経理」部門 を通じて正式に契約・発注されます。

6.  **【ベンダー管理・検収】（各PM）**

      * 発注後、起案者であるプロジェクト実行責任者（例：「守り」における会計事務所の管理は、しおん様）が、ベンダーの進捗と成果物を管理します。
      * 成果物が憲章や契約の「受入基準」 を満たしていることを確認し、「検収」を行います。検収後、経理部門に支払いプロセスを依頼します。
