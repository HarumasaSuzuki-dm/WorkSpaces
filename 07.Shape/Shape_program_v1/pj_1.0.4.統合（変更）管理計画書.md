## 🛡️ プログラム統合（変更）管理計画書

| ドキュメント名 | プログラム統合（変更）管理計画書 |
| :--- | :--- |
| プログラム名 | Shapeプログラム |
| バージョン | 1.0 |
| 作成日 | 2025年11月12日 |

## 1. 本計画書の目的
本計画書は、Shapeプログラム全体（経営体質最適化、組織基盤強化、法人研修）の整合性を維持するため、プロジェクトのベースライン（スコープ、スケジュール、コスト等）に対する変更を管理し、特にプロジェクト横断で発生するリソース競合の解決プロセスを定義するものです。

## 2. 変更管理プロセス
すべての変更要求は、以下のプロセスを経て管理されます。
1.  **変更要求の起票:** 変更要求者は、プログラムPM（中井様）に変更内容と理由を文書で提出します。
2.  **影響分析:** プログラムPMは、各プロジェクトPMと連携し、変更がプログラム全体に与える影響（スコープ、スケジュール、コスト、リソース）を分析します。
3.  **レビューと承認:** プログラムPMは、分析結果をスポンサー（なーすけ様）に報告し、変更の承認を仰ぎます。
4.  **結果の伝atal:** 承認または却下の決定は、すべての関係者に速やかに伝達され、関連ドキュメントが更新されます。

## 3. リソース競合時の解決ルール
プログラム横断でキーパーソン（なーすけ様, 鈴木, しおん様, 中井様など）のリソース競合が発生した場合、以下の優先順位付けルールに基づき、プログラムPMが調整案を策定し、スポンサーが最終決定を行います。

| ルールID | 解決ルール | 理由と適用方針 |
| :--- | :--- | :--- |
| **Rule-1** | **最優先は「守り」のQuick Win** | 経営基盤の安定なくして「攻め」は成り立ちません。「経営体質最適化」プロジェクトの**Quick Win達成**に関連するタスクを最優先で実行します。 |
| **Rule-2** | **ボトルネックリソースの負荷平準化** | 特定の専門家（例：鈴木）への過度な負荷集中は、プログラム全体のリスクとなります。タスクの**時期分散**や、他メンバー（各PJのPM等）への**タスク移管**を検討し、負荷を平準化します。 |
| **Rule-3** | **スポンサーの「意思決定時間」を死守** | スポンサー（なーすけ様）の承認遅延はプログラムの致命的な停滞に繋がります。レビュー会議は**論点を事前整理し、準備を徹底**することで、スポンサーの**判断時間を最小化**するプロセスを最優先で構築・維持します。 |
