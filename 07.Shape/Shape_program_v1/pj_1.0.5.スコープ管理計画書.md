## 1. プログラム・スコープの目的

本プログラムは、SHAPE社の持続的成長を実現するため、「守りの経営基盤強化」と「攻めの新規事業創出」を両輪で推進することを目的とします。以下の3つの構成プロジェクトを通じて、短期的な経営課題の解決と、中長期的な事業の柱の構築を同時に実現します。

1.  **経営体質最適化（守り）:** 「正しい経営判断」を可能にするための会計基盤とKPI構造を設計します。
2.  **組織基盤強化（守り）:** 事業拡大に対応できる、筋肉質な組織OS（役割・責任）を設計します。
3.  **法人研修事業（攻め）:** SHAPE社の独自性を活かした高付加価値な新規事業を立ち上げ、成功モデルを確立します。

## 3. プログラム WBS (Work Breakdown Structure)

本プログラムのスコープは、以下のWBSで定義されます。

*   **1.0 プログラムマネジメント**
    *   1.1 プログラムキックオフ
    *   1.2 プログラム週次定例会
    *   1.3 統合変更管理
    *   1.4 プログラム完了報告
*   **2.0 プロジェクト: 経営体質最適化**
    *   2.1 Phase 1: 明確化（構造理解）
    *   2.2 Phase 2: 最適化（設計）
*   **3.0 プロジェクト: 組織基盤強化**
    *   3.1 Phase 0: 組織診断 (As-Is)
    *   3.2 Phase 1: 課題の構造化・分析
    *   3.3 Phase 2: 組織・R&R設計 (To-Be)
*   **4.0 プロジェクト: 法人研修事業**
    *   4.1 Phase 0: サービス設計・検証
    *   4.2 Phase 1: パイロット準備
    *   4.3 Phase 2: パイロット実行・検証

## 4. プログラム・スコープ外の項目

以下の項目は、本プログラムのスコープ外とします。

*   各プロジェクトで定義された「設計」「計画」「パイロット検証」以降の**全社的な実行・導入・定着化フェーズ**。
    *   （例: 経営体質最適化における会計システムへの実装、組織基盤強化における新体制の全社展開、法人研修事業におけるサービスの本格展開など）
*   各プロジェクトのスコープ・ベースラインに記載されている個別のスコープ外項目。

## 5. スコープ管理プロセス

1.  **スコープ・ベースラインの維持:**
    *   本計画書および各プロジェクトのスコープ・ベースライン（`pj_1.x.3`）が、プログラム全体の正式なスコープ定義となります。
2.  **変更管理プロセス:**
    *   スコープ・ベースラインに対する全ての変更要求は、`pj_0.0.6.プログラム統合（変更）管理計画書.md` に定義されたプロセスに従って評価・承認・否認されます。
    *   プロジェクト間のスコープに影響を与える変更は、必ずプログラム週次定例会で審議されます。
3.  **成果物の受入プロセス:**
    *   各プロジェクトの成果物は、それぞれのスコープ・ベースラインで定義された受入基準に基づき、担当の承認者によって検証・承認されます。
    - プログラム全体の完了は、全プロジェクトの主要成果物が承認されたことをもって判断されます。
