# コスト管理計画書

## 1. 目的
本プロジェクトのコストを適切に管理し、承認された予算内でプロジェクトを完了させるためのプロセスを定義する。

## 2. 役割と責任
- **プロジェクトマネージャー（PM）:** コストの実績監視、予測、報告、および変更管理プロセスの起案を行う。
- **スポンサー（なーすけ様）:** 予算変更に関する最終的な承認を行う。

## 3. コスト管理プロセス
### 3.1. 予算超過の管理閾値
以下のいずれかの条件を満たす、あるいは満たすと予測される場合、PMは速やかに「4. 予算変更管理プロセス」を開始する。
- **管理閾値:**
    - 予備費（コンティンジェンシー）の使用が必要となった場合
    - コスト・ベースラインの総額を **5%** 以上超過する見込みとなった場合

### 4. 予算変更管理プロセス
1.  **変更要求の起案:** PMは、コスト超過の理由、影響、および対策案を明記した「変更要求書」を作成する。
2.  **レビュー:** 作成された「変更要求書」は、まずプログラム全体を管理する鈴木がレビューし、他のプロジェクトへの影響を評価する。
3.  **承認:** レビュー後、スポンサー（なーすけ様）に変更要求書を提出し、最終的な承認を得る。
4.  **ベースラインの更新:** 承認後、PMは関連するコスト・ベースライン等のドキュメントを更新し、関係者に周知する。
