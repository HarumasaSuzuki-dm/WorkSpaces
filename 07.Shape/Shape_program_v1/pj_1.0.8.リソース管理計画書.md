## 1. 本計画書の目的

本計画書は、プログラムを構成する3つのプロジェクト（①経営体質最適化、②組織基盤強化、③法人研修）のリソース要求を統合し、プログラム全体のリソース配分と潜在的なボトルネックを可視化することを目的とする。

これは、プログラム計画「フェーズ3（統合・調整）」のアウトプットであり、特定されたリソース競合を解決し、実行可能な全体計画を策定するための基礎情報となる。

## 3. プログラム全体の潜在的ボトルネック（リソースリスク）

リソースの統合分析の結果、プログラム全体の成功に影響を及ぼす可能性のある、以下の潜在的ボトルネックが特定された。

### 【最重要リスク】鈴木 (全プロジェクト支援) : 58.5 人日 (全体の52%)

*   **リスク内容:**
    *   全工数の半分以上が鈴木に集中している。特に、各プロジェクトの成果物の品質を左右する分析・設計・可視化といった専門的なタスクを担っており、代替が困難。
    *   特に「組織基盤強化（25.75人日）」と「経営体質最適化（20.25人日）」の負荷が高い。
*   **影響:**
    *   鈴木の稼働がプログラム全体の進捗の律速段階となる。タスクが特定期間に集中した場合、一部のプロジェクトで遅延が発生する可能性が極めて高い。

### 【重要リスク】なーすけ様 (Sponsor / PM) : 24.0 人日 (全体の21%)

*   **リスク内容:**
    *   プログラム全体の最終承認者（スポンサー）でありながら、「法人研修」プロジェクトではPMおよび実行責任者（講師、コーチ）を兼務しており、役割が集中している。
    *   工数自体も2番目に大きく、特に「法人研修」の実作業（18.25人日）の負荷が高い。
*   **影響:**
    *   重要な意思決定の遅延リスクと、実作業の遅延リスクの両方を抱えている。特に、全プロジェクトの節目での承認が滞ると、プログラム全体が停止する可能性がある。

### 【調整要リスク】しおん様 (PM / データ支援) : 19.75 人日 (全体の18%)

*   **リスク内容:**
    *   「経営体質最適化」のPMとして17.75人日の責任を負いつつ、「法人研修」のデータ支援（2.0人日）も担当する。
    *   特に「経営体質最適化」のQuick Win達成に必要なデータ取得・分析タスク（タスク1.1〜1.4, 2.3）は、他の業務との兼ね合いで遅延するリスクがある。
*   **影響:**
    *   担当プロジェクトの進捗遅延。特に、他プロジェクトの支援業務と自身のPM業務が輻輳した場合、ボトルネックとなる可能性がある。

## 4. リソース競合の調整方針（案）

特定されたリソースリスクを軽減し、プログラムを円滑に推進するため、次フェーズ「フェーズ4（実行計画策定）」において、以下の調整を行う。

1.  **鈴木のタスク優先順位付けと平準化:**
    *   プログラム全体のWBSを基に、全プロジェクトにおける鈴木のタスクを洗い出し、依存関係を明確にする。
    *   クリティカルパス上のタスクを最優先とし、実行時期を調整することで、特定期間への負荷集中を避ける（平準化）。
    *   一部の定型的な作業については、他メンバーへの移譲が可能か検討する。

2.  **なーすけ様の役割分担の再定義:**
    *   「法人研修」プロジェクトにおける実作業（マテリアル作成、1on1伴走など）の一部を、他のメンバー（例: 中井様、外部委託）に委任できないか検討する。
    *   スポンサーとしての意思決定（承認）が最優先タスクであることを全関係者で合意し、レビューに必要な時間を事前に確保する（スケジュールロック）。

3.  **PM間の連携強化と進捗の可視化:**
    *   週次のプログラム定例MTGを設置し、各PM（なーすけ様、しおん様、中井様）とプログラムマネージャー（鈴木）が、進捗、課題、リソース状況を密に連携する場を設ける。
    *   これにより、特定のリソースに負荷が集中する兆候を早期に検知し、迅速な再調整を行う。
