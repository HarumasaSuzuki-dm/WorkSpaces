# SFF 30日チャレンジ
## オンライン参加型スポンサー企画

**「挑戦する人を増やす。大会の外でも、みんなで続ける。」**

Shape Fit Festival 2026

---

# Executive Summary

## 提案の核心

現在のスポンサー商品は**会場の物理的制約**（座席・ブース数）に縛られ、スポンサー数の拡大が困難。

<div class="highlight-box">
<p><strong>解決策：</strong>大会前後に「オンライン参加型チャレンジ」を開催し、<strong>参加人数の上限がないスポンサー商品</strong>を創出する</p>
</div>

### 期待効果

| 指標 | 現状（会場型） | 提案後（オンライン型） |
|------|---------------|----------------------|
| リーチ可能人数 | 1,200人（観客上限） | **2,000〜5,000人以上** |
| スポンサー枠数 | 物理制約あり | **実質無制限** |
| 成果報告 | 定性的 | **数値化・可視化** |

---

# 01
背景と課題

---

# 現状の課題：物理的制約の壁

## SFFスポンサー商品の構造的問題

```
選手数：120人（固定）
観客数：最大1,200人（会場キャパシティ）
ブース数：有限（スペース制約）
```

### 何が起きているか

1. **スポンサー数を増やすと枠が足りなくなる**
   - 席・ブース・露出機会は物理的に有限
   - 新規スポンサー獲得に「断る」ケースが発生

2. **効果測定が曖昧になりやすい**
   - 「何人がブースに来たか」の把握が困難
   - スポンサーの社内稟議が通りにくい

3. **リーチの天井が見えている**
   - どれだけ努力しても1,200人が上限

---

# 市場機会：会場外のポテンシャル

## SFFコミュニティの実態

会場に来る人は、SFFに関心を持つ人の**一部**に過ぎない

```
SFF関心層（潜在）     ████████████████████ 10,000人+
↓
オンライン接点あり    ████████████ 5,000人
↓
会場来場            ████ 1,200人 ← 現在のスポンサー商品はここだけ
```

### 見逃している機会

- SNSフォロワー・視聴者層へのリーチ
- 「行きたいけど行けない」層の取り込み
- 大会前後の**継続的な接点**の構築

> **結論：会場外のタッチポイントをスポンサー商品化する**

# 02
提案内容

---

# 企画概要：SFF 30日チャレンジ

## コンセプト

大会の**4〜6週間前**から開始するオンライン運動習慣化プログラム

<div class="highlight-box">
<p>週3回の運動を30日間継続 → 達成者にスポンサー提供の特典を付与</p>
</div>

### 基本設計

| 項目 | 内容 |
|------|------|
| **期間** | 30日間（大会4〜6週間前開始） |
| **参加方法** | スマホ完結（LINE / Googleフォーム） |
| **ルール** | 週3回運動したらチェック（自己申告） |
| **ゴール** | 達成者に特典（割引コード / 景品） |
| **専用アプリ** | 不要（参入障壁を最小化） |

---

# ユーザー導線

## 参加者体験フロー

```
Step 1: 認知・申込
────────────────────────────────────────────────────────→
SNS / HP から申込    LINEで参加確定    週1回リマインド配信

Step 2: チャレンジ期間
────────────────────────────────────────────────────────→
週3回運動           フォームで申告      進捗確認・励まし

Step 3: 達成・特典
────────────────────────────────────────────────────────→
30日完了            達成認定           スポンサー特典獲得

Step 4: 大会接続（オプション）
────────────────────────────────────────────────────────→
達成者表彰          SNS/配信で紹介     コミュニティ継続
```

> **ポイント：専用アプリ不要で参加ハードルを徹底的に下げる**

---

# スポンサーへの提供価値

## 3つの核心的メリット

### A. 参加者に直接届く（配れる）
- クーポン配布（参加者全員 / 達成者限定）
- サンプル配布（希望者のみ・同意取得）
- **会場の物理制約なし**

### B. 数字で報告できる（成果が見える）
- 参加者数 / 達成者数 / 達成率
- クーポン配布数 / **利用数**
- アンケート結果（購買意向・認知度）
- **→ スポンサーの社内説明が通りやすい**

### C. 応援の文脈で露出（ブランド好感度）
- 「挑戦を応援する企業」としてのポジショニング
- 広告っぽくならない好意形成

スポンサーメニュー設計

# メニュー構造：独占枠 + 協賛枠の2層

## スケーラビリティを担保する設計思想

```
                    ┌─────────────────────────────┐
   高単価・独占性    │  タイトルスポンサー（1社）   │
                    └─────────────────────────────┘
                              ▼
                    ┌─────────────────────────────┐
   中単価・複数     │    サポーター（複数社）      │
                    └─────────────────────────────┘
                              ▼
                    ┌─────────────────────────────┐
   低単価・無制限    │   景品提供のみ（無制限）    │
                    └─────────────────────────────┘
```

> **独占枠で収益性を確保 × 協賛枠で数を稼ぐ**

---

# メニュー詳細①：タイトルスポンサー

## 1社限定・最上位パッケージ

### 表記・露出
- **「SFF 30日チャレンジ Presented by ○○」**
- 募集ページ / 参加者向け案内 / 結果レポート表紙にロゴ掲載

### 独自特典
- 達成者への特典提供権（クーポン or 景品）
- 大会当日の達成者表彰ネーミング権（例：○○アワード）
- 週次配信へのメッセージ挿入権

### 想定価格

| 提供内容 | 価格帯 |
|----------|--------|
| ベーシック | 50万円 |
| スタンダード（詳細レポート付） | 100万円 |
| プレミアム（カスタム設計） | 150万円 |

---

# メニュー詳細②：サポーター / 景品提供

## スケーラブルな協賛枠

### サポーター（複数社）

| 項目 | 内容 |
|------|------|
| **露出** | 募集ページにロゴ / 参加者案内に社名掲載 |
| **特典権** | 達成者向けクーポン提供 / 抽選景品（10名程度） |
| **価格帯** | 10〜30万円 |

### 景品提供のみ（無制限）

| 項目 | 内容 |
|------|------|
| **露出** | 社名掲載 + 景品紹介 |
| **特典権** | 景品提供のみ |
| **価格帯** | 3〜10万円（または現物提供） |

> **少額でも参加しやすく、スポンサー数を最大化**

---

# ブース vs オンライン：使い分けガイド

## スポンサーのニーズに応じた最適提案

<div class="columns">
<div class="column">

### ブースが向いている場合

- 試飲・試着・体験が重要
- 濃い層に直接当てたい
- その場で商談したい
- B2B / 採用目的

**→ 会場の「体験」で勝負**

</div>
<div class="column">

### オンラインが向いている場合

- 人数上限を外したい
- 数字で成果を出したい
- クーポンで行動を促したい
- 運営負担を減らしたい

**→ スケールと計測で勝負**

</div>
</div>

> **大手スポンサー：オンライン + ブースのセット提案が最強**

# 04
業種別提案パターン

---

# サプリメント業界への提案

## 「30日で実感」との高い親和性

### ブース型（体験で勝つ）

**狙い**：その場で「試す → 好きになる → 買う」まで作る

- 試飲・試食（プロテイン / BCAA / ドリンク）
- おすすめの飲み方POP設置
- QRで後日購入クーポン配布

**KPI**：試飲数 / QR読込数 / クーポン利用数

### オンライン型（人数と数字で勝つ）

**狙い**：会場上限を外し、「利用数」で効果証明

- 達成者に割引コード or サンプル提供
- 参加時アンケートで悩みセグメント化

**KPI**：参加人数 / クーポン利用数 / 購買意向

---

# アパレル業界への提案

## ビジュアル × コミュニティ訴求

### ブース型（試着・SNSで勝つ）

**狙い**：「試す → 似合う → 撮る → 拡散」の導線

- 試着スペース設置（人気サイズ中心）
- 撮影スポット設置（ロゴ入り背景）
- 会場限定ECクーポン

**KPI**：試着数 / 投稿数（#SFF） / EC流入

### オンライン型（全員露出で勝つ）

**狙い**：参加者全員の画面にブランド露出

- 「#SFFチャレンジコーデ」投稿企画
- 週1でコーデ例配信
- 投稿者に抽選で景品 + クーポン

**KPI**：参加人数 / 投稿数 / クーポン利用数

---

# 美容業界への提案

## 「継続」という美容の本質と合致

### ブース型（診断・相談で勝つ）

**狙い**：「自分ごと化」で購買意向を一気に高める

- 肌 / 髪の簡易診断（1〜3分）
- 「大会前に整える」ミニ提案
- 予約 / 購入導線（QR）

**KPI**：診断実施数 / QR予約数 / 来店率

### オンライン型（継続接点で勝つ）

**狙い**：週1配信で継続的なブランド接点

- 「30日 整えるチャレンジ」：運動 + ケア
- 週1で簡単ケアTips配信
- 達成者に施術優待 or サンプル

**KPI**：参加人数 / 達成率 / クーポン利用数

運用計画

---

# スケジュール

## タイムライン（大会T週基準）

```
T-8週  ┃ スポンサー営業開始
       ┃ タイトル枠を優先確定
       ▼
T-6週  ┃ スポンサー確定・素材回収
       ┃ LP / フォーム準備
       ▼
T-5週  ┃ 募集開始・SNS告知
       ┃ 選手 / 関係者にシェア依頼
       ▼
T-5〜1週 ┃ チャレンジ運用期間
         ┃ 週1配信・申告管理
         ▼
T週     ┃ 締め・達成判定
        ┃ 特典配布開始
        ▼
T+1週   ┃ 結果レポート提出
        ┃ スポンサーへ報告完了
```

---

# 運用体制

## 最小構成での実行プラン

| 役割 | 担当人数 | 主な業務 |
|------|----------|----------|
| **企画責任者** | 1名 | 全体設計 / スポンサー調整 / 最終判断 |
| **運用担当** | 1名 | 参加者管理 / 週次配信 / データ集計 |
| **デザイン/制作** | 随時 | バナー / LP / 結果レポート整形 |
| **スポンサー窓口** | 1名 | 素材回収 / 特典内容確認 / 納品対応 |

### スモールスタート時

> 最小構成：**責任者 + 運用**の2名体制でも実行可能
> 
> デザインは既存テンプレート活用、スポンサー窓口は責任者が兼務

# 07
リスク管理

---

# 想定リスクと対策

## 事前に潰しておくべき論点

### リスク1：参加者が集まらない

| 対策 | 詳細 |
|------|------|
| 特典強化 | 「達成で○○」「参加だけで○○」のインセンティブ設計 |
| 拡散仕組み | 選手/関係者のSNS告知テンプレを配布 |
| 低ハードル | 専用アプリ不要・LINE完結の設計維持 |

### リスク2：不正申告（やっていないのに達成）

| 対策 | 詳細 |
|------|------|
| 性善説運用 | 最初は信頼ベース + 少額特典で開始 |
| 抽選活用 | 高額景品は抽選にして不正の旨みを減らす |
| 段階的厳格化 | 実績を見て次回以降で調整 |

---

# 想定リスクと対策（続き）

### リスク3：個人情報管理の不安

| 対策 | 詳細 |
|------|------|
| 最小限収集 | 基本は「ニックネーム + メール」のみ |
| 明示的同意 | 住所回収は景品発送時のみ、同意必須 |
| セキュリティ | Googleフォーム等の信頼性あるツール使用 |

### リスク4：スポンサー期待値とのギャップ

| 対策 | 詳細 |
|------|------|
| 事前合意 | KPI目標を契約時に明文化 |
| 進捗共有 | 週次で途中経過を報告（サプライズを防ぐ） |
| 保守的設定 | 初年度は控えめな目標で信頼構築 |

# 08
営業トーク

---

# 30秒エレベーターピッチ

## スポンサー候補への第一声

> 「SFFの会場は選手120人・観客1,200人が上限なので、席やブースには物理的な限界があります。
> 
> そこで今回、大会の前に**『オンライン30日チャレンジ』**を開催します。参加者にクーポンや景品を提供する形でご協賛いただける仕組みです。
> 
> オンラインなので**参加人数の上限がなく**、**参加数・達成率・クーポン利用数**といった**数字で効果を確認**いただけます。」

### 続けるべきキーメッセージ

# Appendix：会場型との効果比較試算

## 想定シナリオでの比較

### ブース出展の場合（会場1,200人）

```
来場者数          1,200人
  ↓
ブース前通過       600〜840人（50-70%）
  ↓
立ち止まり         120〜300人（10-25%）
  ↓
試す/QR読込        36〜120人（3-10%）
  ↓
購入等の行動       〜100人未満
```

### オンラインチャレンジの場合

```
参加者数          2,000人
  ↓
達成者            800人（達成率40%）
  ↓
クーポン利用      120人（利用率15%）
```

> **設計次第で行動人数を会場型の1.2〜3倍に拡大可能**
