# 4月以降について：音声メモを受けての整理

## なーすけさんが話していたこと
- 2月は毎週オーナーと30分の1on1を組めた
- 1on1の中身：Mustを守れているか確認 → 守りづらそうなところを早めに見つける → ネクストアクションを決める
- 2月末に全体MTGを組んで、3月にどう収束させるか決める予定
- 4月までこれを続けつつ、ななこちゃんにミーティングの権限を徐々に渡していく
- ただし、ここで詰まっている
	- 4〜5月はオーナーもMustを守るので精一杯。1on1で新しいことを求めるのは無理
	- じゃあ4月過ぎたら1on1で何を話すのか？が描けない
	- Make Your Shapeのオフライン構想が進む中、ジムの未来も決めないといけない
	- さらに、Mustを守れないオーナーが出たらどうするかも決まっていない
- 一言でいうと「2月の動きは見えている。でもその先が白紙」

## 論点は3つ

### 論点1：1on1で話すテーマを、時期ごとに変えていく
- 今の1on1は「Must守れてる？」がテーマ
- これがずっと続くと、オーナーも飽きるし、Mustが当たり前になった後に話すことがなくなる
- だから「いつ・何のテーマに切り替えるか」を先に決めておく

#### 時期ごとのテーマ案
- 2〜3月：「Mustを守る」に集中
	- 1on1で聞くこと：4つのMust、ちゃんとやれてる？どこが大変？
	- 例：「セッションログ、毎回出せてますか？」「24時間レスポンス、きつい場面ありますか？」
	- なーすけさんが全店舗を担当
	- 3月末の全体MTGで「2月、回ったか？」をログで振り返る
- 4〜5月：「Mustをやってみて出てきた困りごと」を一緒に解く
	- 1on1で聞くこと：Mustは回っているけど、現場でこういう時に困る、という具体の話
	- 例：「ログは出してるけど、何を書けばいいかまだ迷う」「NPSの回収率が低い、どうすればいい？」
	- Mustの「なぜこれをやるのか」をオーナー自身が体感で分かってくる時期
	- この時期からななこちゃんが1on1に一緒に入り始める。なーすけさんは横で聞く側に
- 6〜7月：「あなたの店舗、これからどうしたい？」を話す
	- 1on1で聞くこと：Mustが当たり前になった上で、オーナー自身がやりたいこと・目指す姿
	- 例：「柏の店舗、来年どうなっていたい？」「顧客との関わり方で変えたいことある？」
	- ここで初めて「ジムの将来像」（教える場→再起動の場）を会話の素材として使える
	- Mustが安定している店舗から順番にこのテーマに入る
- 8月〜：オーナーが自分で立てた目標を追う
	- 1on1で聞くこと：店舗ごとに決めた取り組みの進捗
	- ななこちゃんが1on1の主担当。なーすけさんは月1で全体を見るだけ

#### この設計で先に決めておくこと
- 「Mustが定着した」をどう判断するか
	- 数字で見る例：4つのMust全部で遵守率90%以上が2週続いた
	- 感覚で見る例：オーナーが自分でズレに気づいて、言われなくても直せている
	- 両方使うのが現実的。数字でまずOKライン → 感覚で「もう大丈夫」を判断
- 店舗ごとに進むスピードが違うことを許すか
	- 全店舗一緒に進める方が管理はラク
	- でも、できている店舗を待たせるとやる気が下がる
	- おすすめ：店舗ごとに今どのフェーズにいるか管理する（ななこちゃんの負荷と相談）
- 「いつまでもMustが定着しない店舗」はどうするか
	- 次の論点2で扱う

### 論点2：Mustを守れないオーナーが出たら、どうするか
- 今の設計：「声かけして、支援して、仕組みを直す」
- でもこれは「やろうとしているけど、うまくいかない」人向けの対応
- 「やろうとしていない」場合のルールがない

#### まず「なぜ守れないか」を3つに分ける
- 仕組みが悪い
	- 例：フォームが使いにくい、提出の締め切りが現実的じゃない
	- 対応：本部が仕組みを直す（オーナーの問題ではない）
- 手が足りない
	- 例：トレーナーが少なくて回らない、オーナー自身が忙しすぎる
	- 対応：一緒に負荷を減らす方法を考える（人を増やす、やり方を変えるなど）
- やる気・優先度の問題
	- 例：仕組みも人も揃っているのに、後回しにしている
	- 対応：ここだけがエスカレーション（段階を上げていく）の対象
- 2月の1on1で未達が出たら「この3つのどれ？」を毎回メモしておく。これが3月以降の判断材料になる

#### 段階を上げていく流れ
- ステップ1：事実を見せる
	- 「先週のログ見ると、ここが未提出ですね」と確認するだけ。責めない
- ステップ2：原因を一緒に探して、手を打つ
	- 仕組みが悪い → 直す。手が足りない → 支援する
	- ここまでは「一緒にやる」スタンス
- ステップ3：期限を決める
	- 仕組みも支援も整えたのに変わらない → 「いつまでに改善するか」を一緒に決める
	- ここから「姿勢の問題」として扱い始める
- ステップ4：条件を見直す
	- 期限を過ぎても変わらない → FC契約の条件を話し合う
- 大事なこと：ステップ1→2→3→4、それぞれ「何回」「何ヶ月」で次に進むかを先に決めておく
	- 決めておかないと、結局なーすけさんの「もうちょっと待とう」で曖昧になる

#### このルールをオーナーに事前に見せるか
- 全部見せるメリット：公平。「知らなかった」が起きない
- 全部見せるリスク：テスト運用なのに「罰則」の話をすると萎縮する
- おすすめ：段階的に見せる
	- ステップ1〜2（事実の確認と支援）は2月の最初から「こうやっていきますね」と伝える
	- ステップ3〜4（期限設定・条件見直し）は3月末の全体MTGで「テスト結果を踏まえて、こうします」と伝える
	- 2月のテスト期間は「学びのための記録」スタンスなので、ステップ3以降はまだ出さなくていい

#### 進め方
- 2月：1on1で未達のたびに「仕組み？手が足りない？やる気？」をメモする（データをためる）
- 3月：実際のデータを見て、ステップの移行条件を具体的に決める
- 3月末の全体MTG：「こういうルールでいきます」とオーナーに共有する

### 論点3：ジムの将来像を、いつ・どうやってオーナーに共有するか
- Make Your Shapeの方で「オフラインは教える場じゃなくて、再起動の場」という方向が見えてきている
- これは今のShapeFitジムの営業のやり方とかなり違う
- だから「いつ・どうやって共有するか」を間違えると面倒なことになる

#### タイミングの問題
- 早く言いすぎると
	- 「Mustもまだなのに、また新しい話？」とオーナーが混乱する
	- Must定着に使うべきエネルギーが分散する
- 遅く言いすぎると
	- Mustを頑張って定着させた後に「実はジムの方向性変わります」→ 裏切られた感覚
	- 「先に言ってよ」となる
- ちょうどいい順番
	- 4〜5月の1on1で「あなたの店舗、これからどうしたい？」とオーナーに先に聞く
	- オーナー自身の言葉で「こうしたい」「ここに悩んでいる」が出てくる
	- その上で6〜7月の全体MTGで「本部としてはこういう方向を考えている」と重ねる
	- 本部のビジョンを先にドンと出すより、オーナーの声と接続させてから出す方が納得感が全然違う

#### 伝える量の問題
- 「オフラインの理想」には3つの案（承認店舗/イベント分散/コーチ拠点）まである
- これを全部見せるとオーナーは「で、うちはどうなるの？」と混乱する
- 最初に伝えるのは一つだけ
	- 「ShapeFitのジムは"やり方を教える場所"から"お客様が自分の軸に戻れる伴走の場所"に変わっていく」
	- この一文だけ。具体のモデルはオーナーと一緒に考える
	- ここまで「共創」でやってきたのに、ビジョンだけ本部が決めて降ろしたら矛盾する

#### 全店舗同時か、やりたい店舗からか
- ビジョン（方向性）の共有 → 全店舗同時にやる
- 具体のアクション → 手を挙げた店舗から始める
- 「全員に同じ景色を見せるけど、動き出すタイミングは自分で決めていい」

#### 進め方
- 2月：ビジョンの話はしない。Must定着だけに集中
- 3月：「オフラインの理想」を、オーナーに見せられる1枚の資料にまとめる（今の対話ログのままだと使えない）
- 4月〜：1on1でオーナーの声を先に聞き、6〜7月で本部ビジョンと接続

## 3つの論点をまとめると
- 全部「1on1のフェーズ管理」で束ねられる
	- 1on1のテーマ切り替え → フェーズ設計（論点1）
	- 守れない店舗の対処 → フェーズの分岐（論点2）
	- ビジョンの共有 → フェーズCのタイミング（論点3）
- 2月はやることが明確。追加の判断はいらない
- 3月が一番大事な月
	- Must定着の判定基準を決める
	- 守れない場合のステップを固める
	- ビジョン文書を1枚にまとめる
- この3つが3月中に揃えば、4月以降は1on1フェーズに沿って自然に回る
