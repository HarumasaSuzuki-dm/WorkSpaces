# ShapeFit オンライン伴走サービス｜プロジェクト概要

> **目的：** 本資料はプロジェクト全体の設計・体制・スケジュールを1枚に集約し、意思決定と全体把握を可能にするものである。詳細は各個別資料（テスト設計／運用規定／カリキュラム／体制案／スケジュール）を参照。

---

## 1. サービス定義

**一言定義：**「自分らしく続けられる仕組み」を手に入れ、"ステージに立つ自分"を自走で作れるようになる伴走サービス。

**提供価値の核：** 情報提供ではなく「行動が続く仕組みの実装」。見るだけ・聞くだけでは価値が成立しない設計。

| 項目 | 内容 |
|------|------|
| 対象 | SFF出場者／出場候補者。自信が持てない・続かない・一人で迷う人。最低限の提出コミットができることが前提 |
| 非対象 | 提出拒否／極端な制限志望／医療領域／パーソナルトレーニングのみ希望 |
| 価格（テスト） | 22万円／半年 |
| 価格（本ローンチ） | 44万円／半年（コミュニティ・卒業生制度等を追加） |

**受講後に起きてほしい変化：** 方向性を迷わない → 継続できる → 崩れても戻せる → 情報に振り回されない → 自信がつく → 健康的に絞れる

## 2. 提供メニュー（3レイヤー構造）

| レイヤー | 内容 | 頻度・形式 |
|----------|------|-----------|
| **理念・行動原理（OS）** | なーすけさん登壇の全体セッション。チェックイン→講義→Q&A→チェックアウト→宿題 | 月1〜2回（オンライン） |
| **実行支援（伴走）** | マネージャーが日常の迷いを整理し、次の一手に落とす。チャット＋定期面談＋提出フォロー | チャット常時／面談隔週 |
| **知識（辞書）** | カリキュラム動画（5〜10分×短尺）。**1動画＝1ワーク提出**が絶対ルール | 全100本（24週間ロードマップ） |

## 3. テスト設計

### 3.1 テストの目的

**「売上」ではなく「検証」。** 本ローンチで失敗しないための再現性を確認する。

**検証したい3つの問い：**
1. マネージャー伴走があると"続く"が起きるか
2. 動画＋ワーク提出で"自走"が育つか（見るだけになっていないか）
3. 月1〜2回のなーすけ講義＋特別講師が、熱量と文化を維持できるか

### 3.2 成功条件（Go / No-Go）

| Go（本ローンチへ） | Pivot（再設計） | Stop（この形は危険） |
|---|---|---|
| 提出KPIが目標超過 | 提出率が低い（ワークが重い／導線不備） | "情報商材"扱いで信用毀損 |
| 自走化指標が伸びる | マネージャー品質が統一できない | クレーム増でブランド悪影響 |
| 運用が壊れない | 講義が一方通行で文化にならない | — |

### 3.3 テスト規模・期間

| 項目 | 設定値 |
|------|--------|
| 参加者 | 30名（最低）〜50名（理想） |
| マネージャー | 2〜3名（1名は属人化リスク大） |
| 1人あたり担当 | 10〜25名 |
| 期間 | 8〜12週間（推奨8週）→ 約半年の受講期間内に実施 |
| カリキュラムMVP | 27本／8週間（100本のうち先行制作） |

### 3.4 KPI設計

**最重要（提出KPI）：** 提出が回れば改善できる。回らなければ設計が間違っている。

| 区分 | KPI | 目標 |
|------|-----|------|
| 提出 | 週次チェックイン提出率 | 70%以上 |
| 提出 | ワーク提出完了率 | 60%以上 |
| 自走化 | 相談の質の変化（「答えください」型 → 「整理→選択→意思」型） | 後者比率の上昇 |
| 自走化 | "崩れた時の戻し方"自己説明率 | 定性で確認 |
| 体験 | "情報商材っぽい"違和感 | 0〜低 |
| 体験 | 運用事故（返信遅延・面談すっぽかし等） | 0件 |

## 4. マネージャー運用規定（要点）

### 4.1 役割定義

> 受講者が"自走"できるように、日常の迷いを整理し、行動が続く仕組みへ落とす伴走者。

### 4.2 SLA

| 項目 | 基準 |
|------|------|
| チャット一次返信 | 24時間以内（対応日換算） |
| チャット実質回答 | 48時間以内 |
| 面談リクエスト確定 | 48時間以内 |
| チェックインFB | 提出後48時間以内 |
| ワーク提出FB | 提出後72時間以内 |
| 対応時間 | 10:00〜20:00／週5日（水・日休み） |

### 4.3 対応範囲

| OK | NG |
|----|-----|
| 目標・行動設計、優先順位づけ | 医療・診断・治療 |
| 進捗レビュー・立て直し | 危険な減量指導（脱水・極端制限等） |
| メンタル支援（非医療）、不安の言語化 | 結果保証（「必ず痩せる」等の断定） |
| 一般的な栄養・トレーニングの考え方提示 | 24時間対応の約束 |

### 4.4 禁止事項

人格否定・比較煽り・専門家ぶった断定・体調不良時の無理強い・極端な制限推奨・受講者情報の無断共有・SNS投稿強制

### 4.5 エスカレーション

| 区分 | 条件 | 対応 |
|------|------|------|
| 即時 | 体調不良・摂食異常・自傷他害示唆・ケガ悪化・強クレーム | 一次返信→当日中にQAへ共有 |
| 24h以内 | 悩みループ・判断迷い・信頼関係問題・提出2週連続停止 | QA/PMへ報告 |

### 4.6 離脱予兆対応

- **1週停止：** 責めない確認 → 最小提出を提案
- **2週連続：** エスカレ → 15分面談提案
- **3週連続：** 一時停止扱い → 再開条件を明文化

## 5. カリキュラム骨子

### 5.1 設計原則

- **1動画（5〜10分）＝ 1ワーク提出（5分で終わる最小単位）**
- ワークは「書く／選ぶ／撮る／数える」で提出物が残る形式
- 学びの順番は固定：迷いを消す → 続く仕組み → 実行 → 大会期 → 卒業（自走）

### 5.2 カリキュラム構造（5つの束）

| 束 | テーマ | 主な内容 |
|----|--------|---------|
| A. スタート | 迷いを消す | 目標の立て方、自分ルール、判断基準、優先順位 |
| B. 継続 | 習慣化・環境設計 | 環境／時間設計、崩れた時の戻し方、メンタル、記録 |
| C. 実行 | 食事・運動の実装 | 食事設計、PFC、外食対応、筋トレ、停滞期処理 |
| D. 大会期 | SFF/撮影のピーク | 逆算設計、直前リスク管理、本番メンタル、リバウンド防止 |
| E. 卒業 | 自走 | 自走ルール1枚化、次の目標設計、ギブ側への移行 |

### 5.3 24週間ロードマップ（全100本）

| 週 | フェーズ | 本数 |
|----|---------|------|
| Week 0〜3 | オンボ・迷いゼロ化・習慣の土台 | 17本 |
| Week 4〜7 | 食事設計・イベント対応・停滞処理・回復 | 17本 |
| Week 8〜11 | 筋トレ導入・フォーム基礎・メニュー設計 | 16本 |
| Week 12〜15 | ポージング導入・魅せ方・ルーティン・仕上げ | 17本 |
| Week 16〜20 | 大会期メンタル・直前調整・当日設計・統合 | 21本 |
| Week 21〜23 | 卒業準備・自走・物語化 | 12本 |

**テストMVP：** Week 0〜7の27本を先行制作し、8週間で提出KPIを検証。

## 6. 体制

### 6.1 Phase1（〜SFF：テスト運用＋SFF完走）

**基本方針：** 動画出演はなーすけさん固定。Phase1→2で同じ役割は同じ人。テスト中は「作る」より「回して学ぶ」を優先。

| 機能 | 役割 | 担当 | 週次工数 |
|------|------|------|---------|
| 経営判断 | 事業責任者/PO | なーすけさん | 1〜2h |
| | PM/PMO | 鈴木 | 6〜10h |
| | オペ統括 | かおんさん | 4〜8h |
| コア提供 | マネージャー（1〜3名） | 未定 | 各10〜20h |
| | QA（セカンドオピニオン） | 鈴木（暫定） | 2〜4h |
| | 運用台帳/漏れゼロ管理 | かおんさん（暫定） | 3〜5h |
| コンテンツ | 動画出演・講義登壇 | なーすけさん（固定） | 収録週20〜40h／通常週2〜4h |
| | カリキュラム設計 | 鈴木 | 6〜10h |
| | 講義運営 | かおんさん（暫定） | 1〜2h |
| | 台本・撮影・編集 | 未定 | 制作月30〜70h |
| 基盤 | プラットフォーム・フォーム管理 | 未定 | 初月10〜20h→週1〜2h |
| 評価 | テスト評価・レビュー回収 | 鈴木＋かおんさん | 初月8〜12h→週1〜2h |

**未充足ポジション（要アサイン）：** マネージャー、台本/教材制作、撮影オペ、編集、プラットフォーム管理、CS窓口、規約/法務

### 6.2 Phase2（SFF以降：本サービス本格化）

Phase1体制を維持しつつ以下を追加：
- **講師制度**（外部/内部）：テーマ別講義で集客効果も狙う
- **コミュニティ運営**：日次/週次の場の設計・運用
- **マーケ/セールス/CS**：LP・募集導線・広告・解約理由回収
- **マネージャー採用・育成**：スケールに向けた組織化

| 月 | マイルストーン |

- **週次：** 月＝未提出者復帰導線、火〜金＝マネージャー伴走、週末＝チェックイン集計＋FB
- **月次：** なーすけ講義（月1〜2回）、KPIレビュー、運用規定・テンプレ更新

## 8. リスクと対策

| リスク | 影響 | 対策 |
|--------|------|------|
| 情報商材に見える | ブランド毀損・信用喪失 | テスト→改善→ローンチの段階設計。UGC中心の見せ方。「提出必須」で実行サービスであることを証明 |
| マネージャー品質のバラつき | 満足度の谷・クレーム | 運用規定＋SLA＋QA枠＋週次品質共有＋テンプレ標準化 |
| カリキュラム工数爆発 | スケジュール遅延 | 束単位で設計し最小セット（27本/8週）で検証。週2本ペースで積み上げ |
| マネージャー燃え尽き | サービス崩壊 | 1人あたり担当上限設定、対応時間/休日の厳守、エスカレーション文化の定着 |
| 提出が回らない | テスト失敗・改善不能 | ワーク最小化（5分）、未提出検知→復帰導線の自動化、責めないテンプレ |
| 医療領域への越境 | 法的リスク・事故 | 禁止事項明文化、即時エスカレーション、「医療優先」テンプレ運用 |
