# SFF組織体制改善プロジェクト 基本設計書
**～「チームSFF・防波堤プロジェクト」実行計画～**

## 1. 背景と経緯（Background）

クライアント（しおん様）より、SFF（Shape Fit Festival）および関連事業の運営における「タスク分担（RACI）」と「組織マネジメント」に関する悩み相談を受けたことが発端。

提供された音声データ（計5本）より、以下の状況が明らかになった。

* **現状の体制:**
    * [cite_start]**しおん様（代表）:** 最終決定、クオリティ管理、詳細の詰め、進捗確認、イレギュラー対応など、全方位的に関与。「相談（C）」も「説明責任（A）」もすべて自身に集中しており、心理的・実務的負担が限界に近い [cite: 4]。
    * [cite_start]**神保様（中心メンバー）:** 実務全般を担当するが、期限遵守やタスク漏れ、論理的な詰めにおいて課題があり、しおん様の信頼（ハイジーン・ファクター）を損なっている [cite: 4, 10]。
    * [cite_start]**かおん様・おしりちゃん:** それぞれの強み（論理・詳細管理など）を持つが、神保様のマネジメントや防波堤としては十分に機能定義されていない [cite: 4]。
    * [cite_start]**町田様:** 事務タスクの移譲先候補だが、引き継ぎがうまくいかず、戦力化が遅れている [cite: 10]。

* **しおん様の心情（ぼやき）:**
    * 「任せたいが、クオリティが心配で手放せない」
    * 「神保さんのペースより自分の焦りが勝ってしまい、結局自分がやってしまう」
    * 「期日を守る等の『当たり前』の基準が合わず、信頼関係の構築に疲弊している」
    * [cite_start]「自分が止めなければお客様に迷惑がかかるというプレッシャーがある」 [cite: 9, 4]

## 2. 本質的な課題（Core Issues）

表面的な「忙しさ」の奥にある、構造的な3つの課題を特定。

1.  **「詳細×論理（しおん）」vs「全体×感覚（神保）」の断絶**
    * [cite_start]リーダーと現場責任者の思考特性（OS）が真逆であり、共通言語がない。「神保様にはしおん様の求める詳細が見えていない」ため、直接のやり取りが互いのストレス源となっている [cite: 4]。
2.  **「ハイジーン・ファクター（基礎的信頼）」の欠落**
    * [cite_start]高度な業務委譲の以前に、「期日を守る」「漏れがない」という社会人としての基礎部分で信頼が置けず、しおん様が「マイナスをゼロにする監視」から抜け出せない [cite: 4]。
3.  **責任（Accountability）定義の心理的ブロック**
    * [cite_start]しおん様の中で「責任＝自分の理想通りに100%仕上げること」と定義されているため、他者が入り込む余地がなく、結果として「全責任の抱え込み」が発生している [cite: 9]。

## 3. プロジェクトの目的（Objectives）

**「人を変えるのではなく、配置と情報の流れを変える」**

神保様の性格矯正を目指すのではなく、チームメンバー（かおん様・おしりちゃん）の才能を「防波堤」として活用し、以下の状態を実現する。

* **定量目標:** しおん様の「進捗確認」「未完成品のチェック」にかける工数を極小化する。
* **定性目標:** しおん様が神保さんの「マイナス面（管理不足）」を見なくて済む環境を作り、本来の「プラス面（熱量・巻き込み力）」を評価できる健全な関係性を取り戻す。

## 4. 解決策：チームSFF・防波堤構想（Solution）

しおん様と神保様の「直通ライン」を遮断し、メンバーの特性を活かしたフィルターを通すことで、しおん様を守る。

1.  **進捗管理の防波堤（おしりポリス）**
    * 「感覚×詳細」タイプのおしりちゃんに、進捗管理とリマインド権限を完全移譲。しおん様は神保様を追わず、おしりちゃんからの報告のみを受ける。
2.  **論理品質の防波堤（かおん検問）**
    * 「全体×論理」タイプのかおん様を、しおん様への相談前の「15分検問所」とする。論理的な破綻がないかフィルターを通し、しおん様の脳内ノイズを減らす。
3.  **実務の切り出し（町田さんへの強制移管）**
    * 神保様が抱えている定型業務（まずは2つ）を、強制的に町田様へ移管し、リソースの空きを作る。

## 5. 実行プロセス上の工夫（Implementation Strategy）

しおん様の負担を最小限にするため、以下のルールで進行する。

* **音声メモ活用:** コンサルタント（鈴木）からの質問に対し、しおん様・神保様は「音声メモ」で回答するだけでよい。
* **コンサルタントによる代行:** マニュアル作成、チェックリスト作成などのドキュメントワークはすべて鈴木が行う。
* **スモールスタート:** 移管タスクは最初から全部やろうとせず、まずは「2つ」に絞り、成功体験を作る。
