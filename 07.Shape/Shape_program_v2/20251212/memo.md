# 動画編集者募集・選考フロー設計（提案）

## 1. 目的

* 「なーすけFitness」編集パートナーを **1名採用**し、安定して動画制作を回せる体制をつくる。
* ペルソナ（通勤中の音なし“ながら見”前提）に刺さる編集品質を、再現性高く担保する。

## 2. 前提（募集要項の重要条件）

* 業務：40〜50分素材 → 20〜30分Vlog、フルテロップ、整音・BGM、自然なつなぎが最重要。
* MUST：Final Cut Pro / Vrew / 大容量データ運用 / セキュリティ。
* 納期目安：素材共有から中2〜3日（要相談）。納品：完パケ＋プロジェクト。
* 契約：3ヶ月トライアル、報酬減額なし。評価観点は「反映力/安定性/報連相/理解力」。

## 3. 結論：最適な選考フロー（仕組み化込み）

> “応募を集める”よりも “相性と再現性で落とす” が重要。
> DM対応を減らし、比較可能なテストで判断する。

### 全体フロー（推奨）

**Step0：募集導線（Instagram → 応募フォームに集約）**

* IG投稿/ストーリーズで募集 → 反応者には **DMでフォームURLを送るだけ**（個別対応の削減）
* フォーム送信後は **自動返信メール**（次ステップ/所要日数/注意事項）

**Step1：一次選考（フォーム審査：足切り＋相性確認）**

* 足切り（機械的にOK/NG）

  * MUST環境（Final Cut / Vrew / データ運用/セキュリティ）
  * 実績URL、PC/OS、自己PR、筋トレ経験（募集要項の応募項目を踏襲）
* 相性確認（ミニ設問：3問程度）

  * 音なし視聴前提で何を優先するか（フルテロップ必須）
  * 「間（ま）」を残す/削る判断軸（人柄の“間”を残す）
  * 指示が曖昧な時の確認方法（報連相）
* 推奨通過人数：**応募多数なら 10〜15人 → 3〜5人へ**

**Step2：二次選考（有償テスト編集：短く・同条件で比較）**

* 目的：本番に直結する「つながりの自然さ」「テロップ精度」「整音/BGM」「コミュニケーション」を同条件で比較する。 
* テスト設計（推奨）

  * 素材：10〜15分（同一素材を配布）
  * 成果物：3〜5分（カット＋フルテロップ＋軽い整音＋BGM＋自然なフェード）
  * 報酬：**固定のテスト報酬**（例：5,000〜10,000円目安 ※最終は運用都合で決定）
  * 期限：依頼から2〜3日を目安（本番納期の感覚に寄せる）
* 推奨残し人数：**最終候補2名（第一候補＋次点）**

  * 1名採用予定でも「次点」を残すと、相性事故/辞退/納期崩れに強い。

**Step3：最終（ショート面談 → オファー → 3ヶ月トライアル開始）**

* 面談：15〜30分（稼働/納期感/修正対応姿勢/報連相の癖を確認）
* 契約：募集要項どおり **3ヶ月トライアル**で開始（報酬減額なし、フィードバック&修正対応あり）

## 4. 評価シート（採点軸：募集要項と統一）

* つなぎの滑らかさ・違和感のなさ（最重要）
* テロップ（正確さ、表記、ルール順守）
* 整音/BGM（声より弱く、感情補強、自然フェード）
* 修正指示への反映力 / 速度と安定性 / 報連相 / 理解と改善姿勢（トライアル評価基準）

## 5. 仕組み化（最小運用）

* 応募管理：スプレッドシートでステータス管理
  例）応募 → 一次OK → テスト依頼 → 提出 → 面談 → オファー → トライアル
* テンプレ文面：DM/メールを各ステータスで1本ずつ用意（運用が一気に軽くなる）

## 8. 選考フロー（予定）

本募集は、以下の流れで選考します。

1. **一次選考（応募内容の確認）**
   ご応募時にご提出いただく内容（実績URL、使用環境、自己PR、筋トレ経験など）をもとに確認いたします。
   ※本プロジェクトは **Final Cut Pro / Vrew / 大容量データ運用** が必須です。

2. **二次選考（テスト編集：有償）**
   一次選考を通過された方に、同一素材をお渡しし、短尺のテスト編集をご依頼します。

* テスト報酬：**[固定◯◯円]（有償）**
* 形式：カット編集＋テロップ＋簡易整音＋BGM（本番の要点を確認します）
* 目的：Vlogとしての「つながりの滑らかさ・違和感のなさ」を特に重視します。

3. **最終確認（オンライン面談） → ご契約**
   稼働イメージ、納期感、修正対応、コミュニケーション面を確認の上、契約に進みます。

## 9. 契約開始後について（トライアル）

契約開始後は **3ヶ月間のトライアル期間**を設けます。
トライアル期間中も **報酬条件は本契約と同一**で、品質向上を目的としたフィードバックおよび修正対応が発生することを前提とします。
評価は「修正指示への反映力」「業務スピード/安定性」「報連相」「編集意図の理解と改善姿勢」などを総合的に判断します。
