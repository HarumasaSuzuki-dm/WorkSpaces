# 📂 05_採用トライアル (For Applicants)

このページは、書類選考を通過された方への「実技テスト（トライアル）」の案内です。
マニュアルをどこまで読み込み、**「なーすけFitness」の意図を再現できるか**を見ています。

---

## 📝 トライアルの概要と目的

単に動画を編集する能力だけでなく、以下の3点を重点的に評価します。

1.  **マニュアルの遵守力:** フォント、音量、リスク管理（モザイク）のルールを守れているか。
2.  **ペルソナへの理解:** 「30代疲れた会社員」に寄り添う演出ができているか。
3.  **コミュニケーションコスト:** 納期を守れるか、報告・連絡がスムーズか。

---

## 🎬 課題内容（Instruction）

以下の素材を使用し、指定の要件で動画を作成してください。

### 1. 素材データ
* **元動画:** [ここにGoogleドライブ等のURLを貼る]
    * ※なーすけ様の日常会話が含まれる未編集データです。
* **使用BGM・SE:** フリー素材、または自身でお持ちのものを使用してください（マニュアルのトーン＆マナーに合わせること）。

### 2. 編集範囲と尺
* **作成範囲:** 動画の冒頭から「メインの食事が終わるまで」のパート。
* **仕上がり尺目安:** 5分〜8分程度
    * ※40分の素材を全て編集する必要はありません。要点を掴んで構成してください。

### 3. 必須要件（Must）
* **フルテロップ:** 誤字脱字チェック必須。
* **モザイク処理:** 住所特定につながる情報の完全排除。
* **VREW + FCPX連携:** マニュアルの手順通りにデータを整理しているか。

### 4. 納期
* **依頼日から3日以内**（例：月曜依頼→木曜23:59まで）
    * ※早さよりも「丁寧さ」と「ルールの遵守」を優先しますが、遅れる場合は必ず相談してください。

## 📦 提出・納品ルール

以下の形式で提出してください。この形式が守られていない場合、ファイルを開封せずに不採用となる場合があります。

1.  **提出ツール:** GigaFile便（保存期間：**100日**に設定）
2.  **ファイル名:** `【トライアル】氏名_なーすけFitness.zip`
3.  **格納データ:**
    * 書き出した動画ファイル（.mp4 または .mov）
    * プロジェクトファイル（FCPXライブラリ、またはXML）
    * サムネイル画像（1枚作成してください）
4.  **提出先:** [採用担当者のLINE または メールアドレス]

## ✅ 評価チェックリスト（採点基準）

私たちはあなたの動画を以下のポイントでチェックします。
提出前にご自身でも確認してください。

### 🛡 レベル1：最低限のライン（足切り基準）
* [ ] **リスク管理:** 電柱、建物、車のナンバー、個人の特定情報にモザイク漏れがないか。
* [ ] **マニュアル遵守:** フォントの種類・サイズは規定通りか。
* [ ] **誤字脱字:** 明らかな変換ミスがないか。

### 🖌 レベル2：編集センス（採用ライン）
* [ ] **カットの違和感:** 会話の「間」を詰めすぎていないか（人柄を殺していないか）。
* [ ] **BGM選曲:** 「マッチョ＝怖い」という印象を与えない、日常に馴染む選曲ができているか。
* [ ] **視認性:** スマホの小画面で見てもストレスのない文字配置か。

### 💡 レベル3：加点ポイント（即戦力候補）
* [ ] **演出の意図:** テロップやBGMの演出に「なぜそうしたか」の意図が感じられるか。
* [ ] **インサイト:** ペルソナ（疲れた会社員）が「ふっ」と笑えるポイントを作れているか。

## ⚠️ クライアント様へ（このページの運用について）

このページを応募者に送る前に、以下の項目を決定・記入してください。
これらが曖昧だと、応募者からの質問が増え、選考コストが上がります。

1.  **テスト素材の選定:**
    * 過去の動画で「編集が難しかったもの」や「要素（食事・会話・移動）が詰まっているもの」を選ぶのがおすすめです。
2.  **報酬の有無:**
    * トライアルは無償で行うか、一律◯◯円（Amazonギフト券など）を支払うか。
    * *（推奨：トラブル防止のため、少額でも「テスト編集費」として設定するとプロ意識の高い層が集まりやすいです）*
3.  **合否連絡:**
    * 「合格者のみに連絡」か「全員に連絡」か。

> **採用のヒント:**
> 応募の際に、**「マニュアルの中に隠されたキーワード（例：『イキヤショウ！』）をメッセージに添えてください」**といったルールを追加すると、マニュアルをちゃんと読んでいる人を自動的にフィルタリングできます。
