## 2. 何が根っこの問題なのか

### 2-1. 表に見えている3つの問題

- Shapeの考え方がトレーナーまで届いていない
  - トレーナーが「なんでこれをやるのか」を実感できていない
    - 本店：フーテロちゃんが「語る・言葉にする」ことの大事さをまだつかめていない
    - 本店：MVS発表が12月から終わっていない → そもそもバリューの理解が浅いかも → 完了済み
    - 柏店：オーナー自身が「柏店としてこれでいいのかな」と感じている
  - オーナーとトレーナーが見ている方向がずれている
    - 本店：ななこちゃん＝Shapeの未来 ↔ フーテロちゃん＝今日の目の前のお客様
    - 柏店：本部が言っていること ↔ 自分の店の実感にギャップ
    - Mustが「上から言われた作業」に見えてしまうリスク
  - 考え方を伝えるやり方が決まっておらず、オーナー個人の力に頼りきり
- オーナーが「人を育てる人」として動けていない
  - Mustの意味を自分の言葉で説明できていない
    - 「なんでこれをやるとセッション数が増えるのか」が自分の中でつながっていない
  - 育て方のスキルに差がある
    - ななこちゃん（本店）：どう対話するかを自分で考えられている
    - ぶーちゃん（上板橋）：仲良くなるのは得意だけど、記録や報告が苦手
    - 都築さん（大阪天六）：トレーナーの声は丁寧に拾えるが、人によって差がある時の対応が課題
    - 玲奈さん（柏店）：そもそもMustに納得するところから
  - オーナーへのフォローがなーすけさん1人に集中している
    - なーすけさんが4店舗すべて1人で週次MTG
    - 本店以外にはかおりさんのようなサポート役がいない
- トレーナーが育つ環境が整っていない
  - 1人で抱え込みやすい状態
    - 本店：フーテロちゃんが1人の時間が長く、迷いが溜まる
    - 上板橋：シフトがバラバラで3人が揃う時間が少ない
    - 大阪天六：プリプリとなおやさんの動き方に差がある
  - 「良いセッションって何？」の基準がない
    - トレーナーの感覚頼みになっている
  - 「自分は成長している」と思える仕組みがない
    - 「一番悪かったもの」だけ記録 → 改善には役立つけどモチベーションが下がるリスク

### 2-2. 一番深いところにある原因

- 「Shapeの考え方 → 日々の行動 → お客様への成果」のつながりがピンときていない
  - 考え方はある（Shapeの資料に書いてある）
  - やることも決まっている（Mustで決めてある）
  - でも「このMustをやるとなぜShapeらしくなれるのか」「なぜセッション数が増えるのか」がつながっていない
- 「挑戦と物語」という考え方が、オーナー・トレーナーに届くまでに薄まっている
  - 本部 → オーナー → トレーナーと伝わる中で「なぜ」が抜け落ちる
- そもそもの負荷が大きい
  - オーナーが自分でセッションもしながら、トレーナーも育てないといけない
  - 「人を育てる人をどう育てるか」の仕組みがない

## 3. どうやって解決していくか

### 論点A：オーナーに「なるほど、だからやるのか」と思ってもらうには

- A-1. Mustの「なぜ」を店舗ごとの言葉に置き換える
  - 「全店共通の理由」ではなく「うちの店にとっての理由」を見つける
  - 例：柏店は半年以内にやめるお客様が多い → ログで「やめそうなサイン」を早く見つけられる
- A-2. オーナー同士が横でつながる場を作る
  - 本店ななこちゃんの対話のやり方が他のオーナーの参考になる
  - 大阪天六のトレーナーの声の拾い方が柏店のヒントになる
  - 今はなーすけさん経由だけ → オーナー同士が直接話せる場の検討
- A-3. 小さな「やってよかった」を早く作る
  - ログから1件でも「この対応で改善できた」という事例を作る
  - NPSで1件でも「紹介をお願いしたら本当に紹介が来た」という事例を作る

### 論点B：Shapeの考え方を伝える「型」をどう作るか

- B-1. トレーナーがまず自分の言葉で語る経験を積む
  - MVS発表を全店舗の育成ステップにする
  - 「お客様に語ってもらう前に、まず自分が語る」という順番を明確に
- B-2. セッションの良し悪しの基準を揃える
  - 「お客様が本音で話せている」の判断ポイントを具体的に決める
  - 良いセッション・もう少しのセッションの実例をみんなで共有する
- B-3. オーナーがトレーナーに説明するときのガイドを作る
  - 各Mustの「なんのため → 何をする → どうなるか」の3点セット
  - 難しい言葉を使わない、現場で使える説明の型

### 論点C：トレーナーが1人で抱え込まない仕組みをどう作るか

- C-1. オーナーとトレーナーが話す機会を仕組みにする
  - 1on1の回数と話す内容のガイドライン（今は店舗ごとにバラバラ）
  - 本店のように週2回の短い対話を基本にするか検討
- C-2. 成長を「見える化」するフィードバックの流れを作る
  - 「改善できたセッション」も記録する
  - 月に1回「先月と比べてここが良くなったね」とオーナーから伝える
- C-3. トレーナー同士で学び合える場を作る
  - まずは同じ店舗の中でセッションの振り返りを一緒にやる
  - うまくいったら店舗をまたいだ事例共有（月1回オンラインなど）

## 4. フェーズBに進む基準

### 4-1. 移行の条件

- **4つのMust全てが「できている」状態が3週間続いたらOK**
- 判定の方法：今使っているMUST運用シートをそのまま見るだけ（新しい集計は不要）

### 4-2. 「できている」とは何か

- セッションログ：その週に1件以上出している
- 24時間レスポンス：大事な連絡に返事が漏れていない（スタンプやリアクションでOK）
- NPS：その月の分が終わっている
- オーナー対話：対話の記録が1on1の前日までに書かれている

### 4-3. なぜ「3週間」なのか

- 2週間だと短い：やり方を直したばかりの店舗は、1回できただけでは習慣とは言えない
- 4週間だと長い：本店のようにもう全部できている店舗を待たせるとやる気が下がる
- 3週間なら：本店は2月末に届く。他の店舗も3月中に狙える

### 4-4. いつ頃フェーズBに進めそうか

- 本店：2月末〜3月初め（あと2週くらい）
  - すでに1on1の中身がトレーナー育成やShapeの考え方の話になっている
- 大阪天六：3月半ば
  - レスポンスは安定。ログも「1日1件」に調整済みで改善の見込みあり。なおやさんの提出がカギ
- 上板橋：3月半ば〜後半
  - ゼロからのスタートだけど、オーナーの姿勢は前向き。やり方は調整済み
- 柏：3月後半〜4月
  - やり方の誤解が解消されたばかり。まず「やってみて意味を感じる」期間が必要

### 4-5. フェーズBに進んだら話せるテーマ

- 本店：フーテロちゃんの指導の考え方をどう腹落ちさせるか、交流会の企画

## 5. Mustを守れなかった場合の対応フロー

### 5-1. 4つのステップ

- ステップ1：事実の確認
  - いつ動くか：今週できていないMustが1つでもある時
  - 何をするか：1on1でなぜできなかったか確認して、メモに記録
  - 誰がやるか：1on1の担当者
- ステップ2：サポート
  - いつ動くか：原因が「やり方が悪い」か「手が足りない」とわかった時
  - 何をするか：やり方の調整や負担を減らす工夫をする
  - 誰がやるか：1on1の担当者
- ステップ3：期限を決める
  - いつ動くか：同じMustが3週続けてできていない時
  - 何をするか：「いつまでにやるか」を具体的に決めて、やってもらう
  - 誰がやるか：なーすけさん
- ステップ4：条件の見直し
  - いつ動くか：ステップ3を2回やっても改善しない時
  - 何をするか：FC契約の条件について話し合う
  - 誰がやるか：なーすけさん

### 5-2. ステップ3のくわしいルール

- 期限：次の火曜日まで（1on1が月曜の想定なので）
- 伝え方：1on1で直接伝える ＋ チャットワークでも文面を送る（記録を残すため）
- 伝える内容：「○○（Must名）を水曜日までにやってください。やったらチャットで教えてください」
- やった連絡がなかった場合
  - 水曜にチャットワークでリマインド（1文でOK）
  - 木曜までに連絡がなければ、次の1on1の最初に確認
- 期限内にやれた場合：通常に戻す（連続カウントもリセット）
- 期限内にやれなかった場合：翌週もう一度期限を設定（2回目）→ 2回ともダメならステップ4へ

### 5-3. 「なぜできなかったか」で対応を分ける

- 「やり方が悪い」「手が足りない」が原因の場合
  - ステップ3には進まない
  - まずやり方を直すことを優先して、連続カウントにも入れない
  - ただし、やり方を直してから2週間たっても変わらなければ → 「やる気・優先度」の問題として判断し直す → ステップ3へ
- 「やる気・優先度」が原因の場合
  - 連続カウント開始
  - 3週続いたらステップ3へ
- 誰が判断するか
  - 1on1をやっている人が、1on1の中で判断する
  - 迷ったら事務局と相談して決める
  - 判断した結果はMUST運用シートの「守れなかった理由」の欄に記録

### 5-4. ステップ4で何を話すか（3段階で考える）

- まず：Mustの中身を変える
  - 「このMustは今の状況では難しいので、別のやり方に変えられないか」
- 次に：サポートの体制を変える
  - 「本部がもっと手を入れる（例：事務局がログの入力を一時的に代わる）」
- 最後に：FC契約の条件を見直す
  - 上の2つで解決しない場合にだけ、ロイヤリティの条件や契約の継続について話す
- 最終的に決めるのはなーすけさん
  - 事務局（ななこちゃん・鈴木）は事実の整理と選択肢の準備をする

### 5-5. 誰がやるかの移行プラン

- 今（2〜3月）
  - ステップ1〜4：全部なーすけさん
- 4月以降
  - ステップ1〜2：できそうなら、ななこちゃんに引き継ぎ（通常のフェーズから外れるので、なーすけさんのままでもOK）
  - ステップ3〜4：なーすけさんのまま（プレッシャーがかかる判断なので）
  - ななこちゃんが1on1に同席してステップ3の場面を見ておくことで、将来の引き継ぎに備える

### 5-6. オーナーにどこまで伝えるか

- 2月（今）：「1on1で状況を一緒に確認して、困っていることがあればサポートします」（ステップ1〜2だけ）
- 3月末の全体MTG：フロー全体を見せる
  - 「テスト期間でわかったのは、できなかった原因のほとんどはやり方の問題でした。改善しました」と事実を先に伝える
  - 「4月からは、やり方を整えた上でちゃんとやれるかを見ていきます」と方針を伝える
  - ステップ4の存在は隠さないけど強調もしない
- 4月以降：個別に「今ステップ○です」とは言わない。ステップ3に入る時は「期限を決めましょう」と自然に伝わる形

## 6. ロードマップ

### Phase 1（2月半ば〜末）：まず動かす

  - 「なぜできなかったか」を聞いて、やり方を柔軟に変える
  - 相談フォームを使ってもらって、小さな困りごとを拾う

    - 週2回の対話を続けて、「言葉にすること」の大事さを体験で伝える

  - 大阪天六（都築さん）
    - なおやさんのセッションログの記入を始めてもらう（プリプリとの差を埋める）
    - セッションログを1日1件に減らしてトレーナーの負担を下げる
    - ボイスチェックの頻度の問題は3月に向けてなーすけさんが持ち帰り
  - 上板橋（ブーちゃん）
    - 1on1の記録をミーティング前に打つルールを定着させる（合意済み）

  - 週次MTGの中で、オーナーの育てる力を伸ばす問いかけを意識的に入れる
    - 例：「このデータを見て、トレーナーにどう伝えますか？」
  - ある店舗でうまくいったことを他の店舗にも広げる役割

### Phase 3（4月〜5月）：仕組みとして定着させる
