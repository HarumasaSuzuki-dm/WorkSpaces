# FC体制の方向転換：課題整理・方向性・スケジュール（v2）

> **作成日**：2025年2月14日｜**更新日**：2025年2月15日（v2：PMレビュー反映）
> **作成者**：鈴木（事務局B）
> **目的**：FC体制を「独立・卒業」へ移行するにあたり、課題・方向性・スケジュールを一覧化し、なーすけさんとの方針確定に使う

## 1. なぜ方向転換するのか（課題の構造）

### 1.1 本部側の課題

| 領域 | 事実 |
|------|------|
| **経済合理性** | ロイヤリティ10%→5%に下げても、支援コスト（システム月20万＋人的工数）に見合わない |
| **リソース配分** | 本部メンバーのリソースがFC支援に食われ、本店経営・SFF・SFC等のコア事業に集中できていない |
| **温度差** | 1年間、気持ちを動かそうとしてきたが、Must運用でも腹落ちしていない構造は変わらなかった |
| **システム依存** | 宮崎さんへの依存（月20万）。Stripe手数料も未回収。初期費用も本部負担（柏・上板橋） |

### 1.2 FC側の課題

| 領域 | 事実 |
|------|------|
| **自由への欲求** | 「自由にやりたい」「ロイヤリティを下げたい」という声が複数店舗から出ている |
| **Must運用の形骸化** | できない原因の大半は「やり方」だが、そもそも腹落ちしていないため改善が進まない |
| **価値認識のズレ** | 「ロイヤリティ払ってる意味あるんですか？」が繰り返される構造 |

### 1.3 核心の気づき

> **双方の望みは実は一致している**
> - 本部：FC支援から解放されたい → コア事業に集中したい
> - FC：自由にやりたい → ロイヤリティを下げたい・なくしたい

## 2. 方向性の決定

### 2.1 基本方針

「**解約**」ではなく「**独立・卒業**」として設計する

- 本部が切るのではなく、オーナーが自分の店として自由にやれるようになる話
- 全店舗一律ではなく、店舗ごとに最適な形を選んでもらう

### 2.2 選択肢の検討結果（社内MTG結論）

| 選択肢 | 内容 | 判定 | 理由 |
|--------|------|------|------|
| **A. 完全独立** | 自分の屋号で運営／ロイヤリティなし／移行期間あり | **採用** | 唯一、構造的な問題を解消できる |
| B. ブランドライセンス | ShapeFit名継続／ロイヤリティ縮小／支援最低限 | 却下 | コスト残存＋不満再燃リスク＋メンタル工数 |
| C. 卒業ネットワーク | 契約終了／勉強会等には参加可能 | 却下 | 「甘いところだけ吸われる」リスク（SFF等） |

### 2.3 伝え方の設計方針

- 方針は「完全独立」一択。オーナーが「自分の力で次に進む」とポジティブに受け取れるよう設計する
- 伝える順序：「本部の反省 → あなたたちの力の承認 → 独立の提案」
- 対話のゴール：独立の「可否」ではなく、独立の「時期・条件・移行方法」を一緒に決める場にする

### 2.4 二択提示の設計（追加）

オーナーに「自分で決めた」と感じてもらうため、以下の二択として提示する。

| 選択肢 | 条件 | 実質的な意味 |
|--------|------|-------------|
| **一緒にやる** | Must4項目への完全合意が前提。今後の説明の繰り返しはしない。未遵守が続く場合は強制解約 | 条件を厳しく設定することで、実質的に独立を選びやすくする |
| **独立する** | 自分の屋号で自由に運営。移行期間あり。ロイヤリティなし | 本部が推奨する選択肢 |

> **注意**：「一緒にやる」の条件定義はステップ2で具体化する（タスク2-7）。曖昧なまま対話に入ると、オーナーが「一緒にやる」を選んだ場合に収拾がつかなくなる。

## 3. ケアすべきリスク

| リスク | 対策 | 担当 |
|--------|------|------|
| **コミュニティへの影響** | SFFコミュニティにジム利用者が多い。変な噂・誤解が広まらないよう公式発表の内容を丁寧に設計する | なーすけ＋鈴木 |
| **契約の不備** | 正式な契約書が巻けていない可能性あり。2/19までに有無を確認し、不備があれば弁護士レビューを3月第1週までに完了させる | 鈴木 |
| **オーナーの感情** | 「切られた」と感じさせない。卒業・独立というポジティブなフレーミングを徹底 | なーすけ |
| **移行期間中の混乱** | システム移行・顧客対応の引き継ぎ期間を十分に確保する | 鈴木 |
| **店舗間の情報伝播（追加）** | FCオーナー同士が横でつながっているため、最初の1on1内容が非公式に他店舗へ伝わるリスク。柏店との対話後1週間以内に他店舗へ「順番にお話しする」旨の一報を入れる | なーすけ |
| **移行期間中のMust運用（追加）** | 独立を伝えた後にMust遵守を求め続けるのは矛盾する。一方で完全に放棄するとサービス品質が崩壊するリスク。ステップ2でMust運用の段階的縮小方針を決定する | 鈴木＋なーすけ |
| **システム移行のリードタイム（追加）** | 各店舗が独自で予約・決済システムを立ち上げる必要がある。選定・構築に3ヶ月以上かかる可能性があり、移行全体のクリティカルパスになり得る | 鈴木 |

## 7. PMとしてのアドバイス

### やるべきこと

1. **契約確認を最優先に**：正式な契約書が巻けていない可能性がある以上、法的リスクの把握が最初。ここが曖昧なまま対話に入ると、後から揉める原因になる
2. **数字で語れる状態を作る**：「見合わない」を感覚ではなく、具体的なP/L（ロイヤリティ収入 vs システム費＋人件費＋機会損失）で示す。なーすけさんの意思決定にも、オーナーへの説明にも使える
3. **柏店を「モデルケース」にする**：自らロイヤリティ相談をしている柏店は、最も前向きな対話が期待できる。ここで成功パターンを作り、他店舗に展開する
4. **システム移行のリードタイムを早期に把握する**：各店舗が予約・決済・顧客DBを自力で立ち上げるには、選定・構築で3ヶ月以上かかる可能性がある。ここが移行全体のクリティカルパスになるため、ステップ1で見積もりを完了させる

### 気をつけるべきこと

1. **「切られた」と感じさせない伝え方の徹底**：言葉選び・順序・場の設定すべてが重要。トークスクリプトは必ず事前にリハーサルする
2. **SFFコミュニティへの波及を甘く見ない**：ジム利用者が多い以上、オーナーの感情がそのままコミュニティに伝播する。公式発表のタイミングと内容は慎重に
3. **移行期間を十分に取る**：急ぎすぎるとオーナーの準備が追いつかず、「追い出された」という印象になる。対話後の検討期間＋移行実行期間で計5ヶ月（3月〜7月）を確保している
4. **Must運用プロジェクトとの整合**：これまでのMust運用の取り組みが「無駄だった」と感じさせないフレーミングが必要。「Mustを通じて各店舗の力が見えた→だからこそ独立できる」という文脈をつくる
5. **店舗間の情報伝播を管理する**：柏店との対話後、他店舗に「順番にお話しします」と一報を入れること。オーナー同士は横でつながっているため、非公式に情報が伝わると不信感の原因になる
6. **移行期間中のMust運用を放置しない**：独立を伝えた後にMustを求め続けるのは矛盾だが、完全放棄はサービス品質の崩壊を招く。ステップ2で段階的縮小の方針を決めておく
7. **初期費用の扱いは対話前に方針を決めておく**：柏・上板橋の初期費用を回収するか放棄するかで、対話のトーンがまったく変わる。数字の整理と方針案はセットで準備する
