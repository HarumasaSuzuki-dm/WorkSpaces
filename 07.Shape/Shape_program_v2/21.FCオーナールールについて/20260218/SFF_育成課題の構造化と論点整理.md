# SFF FC店舗 育成課題の構造化と論点整理

---

## 1. 課題の構造化：何が起きているのか

### 1-1. 理念の浸透が止まっている

- **トレーナーが「なぜやるのか」を体感していない**
  - 本店：フーテルちゃんが「語る・言語化」の価値を腑に落とせていない
  - 本店：MVS発表が12月から未完了 → バリュー自体の理解が浅い可能性
  - 柏店：オーナー自身が「柏店としてどうなんだろう」と感じている
- **オーナーとトレーナーの見ている景色がずれている**
  - 本店：ななこちゃん＝未来のShapeの広がり ↔ フーテルちゃん＝今の目の前のお客様
  - 柏店：本部の方針 ↔ 自店舗の現実のギャップを感じている
  - Mustが「上から降ってきた作業」に見えるリスク
- **理念浸透の「型」がない**
  - 本店：ビジョンブリッジでMVS発表を試みている
  - 大阪天六：トレーナーに直接伝えて前向きに動いている
  - 柏店：オーナー自身の納得が先にある
  - → 各店舗のオーナーの力量に完全に依存している状態

### 1-2. オーナーが「育成者」として動けていない

- **Mustの意味を自分の言葉で翻訳できていない**
  - 柏店：「腑に落ちない」まま1週間がほぼ活動なし
  - 上板橋：自然なコミュニケーションはできるが、構造的な育成に変換できない
  - 「なぜこれをやるとセッション数が伸びるのか」の因果関係が自分の中でつながっていない
- **育成スキルにバラつきがある**
  - ななこちゃん（本店）：対話設計を自分で考えられている／かおりさんと分担もしている
  - ブーちゃん（上板橋）：フランクな関係構築は得意だが、形式的な記録・報告に抵抗感
  - 都築さん（大阪天六）：トレーナーの声を丁寧に拾えるが、トレーナー間の差への対応が課題
  - 玲奈さん（柏店）：Must自体の納得感が先にある段階
- **オーナーへのフォローが属人的**
  - なーすけさんが4店舗すべて1人で週次MTG
  - 本店はかおりさんとの連携があるが、他店舗にはそうした分担がない

### 1-3. トレーナーが育つ環境が構造的に整っていない

- **孤立の問題**
  - 本店：フーテルちゃんが1人の時間が長く、迷いが溜まる
  - 上板橋：シフトがバラバラで3人が揃う時間が少ない
  - 大阪天六：プリプリと中屋さんの動き方に差がある
  - → フィードバックが入らないまま自己流が続く
- **「良いセッション」の基準が共有されていない**
  - 大阪天六・プリプリ：「全員開示していた」と記載
  - 本店・フーテルちゃん：「話してくれていない人がいた」と記載
  - 柏店：「うまくいきすぎる例ばかりになる」
  - → 判断基準がトレーナーの主観に完全依存
- **成長実感を得られる仕組みがない**
  - データが溜まり始めたばかりで、振り返りサイクルが未確立
  - 「一番悪かったもの」だけ記録するルール → 改善には有効だがモチベーション面のリスク
  - 「記録を取らされている」感覚になる可能性

## 2. 本質的な課題：何が根っこにあるのか

### 2-1. 根本原因

- **「挑戦と物語」の理念が、仕組みの運用指示に変換される過程で意味が薄まっている**
  - 本部 → オーナー → トレーナーの伝言ゲームで「なぜ」が抜け落ちる
  - Mustの各施策（セッションログ、NPS、1on1）が「作業」に見える
  - やることは伝わっているが、やる理由が伝わっていない

### 2-2. 構造的な問題

- **オーナーが「プレイヤー」と「育成者」を兼務する負荷**
  - 自分もセッションしながら、トレーナーを育てなければならない
  - 育成のための時間確保が構造的に難しい
  - 特に上板橋・柏はオーナー自身もセッションを持っている
- **「育成者を育成する」仕組みがない**
  - オーナーに対して「Mustをやって」と渡しているが、「Mustの意味をどう伝えるか」の支援がない
  - オーナーの育成スキル向上は個別のMTGでの暗黙的なOJTに依存
  - オーナー同士が学び合う場もない

### 2-3. 一言でいうと

- **「理念 → 行動 → 成果」のつながりが、オーナー・トレーナーの中で腹落ちしていない**
  - 理念は語れる（Shapeの資料にある）
  - 行動は決まっている（Mustで定義されている）
  - しかし、「この行動がなぜ理念の実現につながるのか」「この行動がなぜセッション数に効くのか」がつながっていない

## 3. 解くべき論点の構造化

### 論点A：オーナーの「腹落ち」をどう作るか

- **A-1. Mustの「なぜ」を店舗ごとに翻訳する**
  - 各店舗の現状課題とMustの施策がどう結びつくかを個別に紐づける
  - 例：柏店は半年未満の離脱が多い → セッションログで離脱予兆を早期発見
  - 「全店共通の理由」ではなく「うちの店にとっての理由」を見つける
- **A-2. オーナー同士の横のつながりを作る**
  - 本店ななこちゃんの対話設計が他のオーナーの参考になる
  - 大阪天六のトレーナーからの声の拾い方が柏店のヒントになる
  - 現状はなーすけさん経由の情報伝達のみ → オーナー同士が直接学び合う場の検討
- **A-3. 小さな成功体験を早く作る**
  - セッションログから1件でも「この対応で改善できた」事例を作る
  - NPSで1件でも「この人に紹介プッシュしたら紹介が生まれた」事例を作る
  - 「やる意味あるんだ」を体感させることが最大の腹落ち

### 論点B：理念浸透の「型」をどう作るか

- **B-1. トレーナーが「語る」体験を先に積む設計**
  - MVS発表を全店舗共通の育成ステップにする
  - バリューについての自分の言葉での発表機会を定期的に設ける
  - 「お客様に語らせる前に、まず自分が語る」の順序を明確にする
- **B-2. セッションの質の「基準」を揃える**
  - 「本音で話せている」の判断基準を具体的に定義する
  - 良いセッション・課題のあるセッションの実例を共有する
  - トレーナーの主観に頼らず、チェックポイントで判断できるようにする
- **B-3. オーナーからトレーナーへの伝え方のガイドを作る**
  - 「Mustをこう説明してね」のトークスクリプト的なもの
  - 難しい言葉を使わない、現場向けの説明フレーム
  - 各施策の「目的 → やること → 期待される効果」の3点セット

### 論点C：トレーナーが孤立しない仕組みをどう作るか

- **C-1. オーナーとトレーナーの接点を構造化する**
  - 1on1の頻度と内容のガイドライン（現状は店舗ごとにバラバラ）
  - 本店のように週2回の短い対話を標準にするか検討
  - 「話した内容の記録」を簡単にする（ミーティング前に1分で打てるレベル）
- **C-2. 成長を可視化するフィードバックループ**
  - 悪かったセッションだけでなく「改善できたセッション」も記録する
  - 月次で「先月と比べてここが良くなった」をオーナーからフィードバック
  - NPS改善やお客様の声の変化をトレーナーに共有する
- **C-3. トレーナー同士の学び合いの場**
  - 同一店舗内：セッションの振り返りを2人で行う時間
  - 店舗横断：他店舗の好事例を共有する仕組み（月1回のオンラインなど）
  - まずは同一店舗内から始めて、軌道に乗ったら横展開

## 4. ロードマップ：いつ何をやるか

### Phase 1（2月中旬〜2月末）：土台固め ── まず動かす

- **目標：全店舗でMustが「実施済み」になる状態を作る**
- やること
  - セッションログを「一日1件・一番悪かったもの」に統一（全店舗で合意済み）
  - NPSアンケートの送付を全店舗で実施開始
  - 1on1の記録をミーティング前に簡易入力するルールを定着させる
  - チャットリレーのリアクションを習慣化する
- なーすけさんの動き
  - 週次MTGで「事実確認 → ネクストアクション」のサイクルを回す
  - 「なぜできなかったか」の原因を特定して、やり方を柔軟に変える
  - 相談フォームの活用を促して、小さな困りごとを拾う

### Phase 2（3月）：意味づけ ── なぜやるかを腹落ちさせる

- **目標：オーナーが「自分の店舗にとってMustをやる意味」を語れる状態にする**
- やること
  - 2月のデータを集約して「こんなことがわかった」を各店舗にフィードバック
  - 店舗ごとに「うちの店にとってのMustの意味」を言語化するセッション実施
  - オーナー同士の横の情報共有の場を1回試験的に実施
  - MVS発表を全店舗のトレーナーに実施させる（本店の事例をモデルに）
- なーすけさんの動き
  - 週次MTGの中にオーナーの育成力を高める問いかけを意識的に組み込む
  - 例：「このデータ見て、トレーナーにどう伝えますか？」
  - 各店舗の成功事例を他店舗に横展開する役割

### Phase 3（4月〜5月）：型化 ── 仕組みとして定着させる

- **目標：オーナーが自走して育成サイクルを回せる状態にする**
- やること
  - セッションの質の「基準」を定義してガイド化
  - 理念浸透の育成ステップを文書化（MVS発表 → バリュー実践 → セッションへの反映）
  - トレーナー向けの成長可視化の仕組みを導入（月次フィードバックシート等）
  - オーナー間の定期的な学び合いの場を月1回で固定化
- なーすけさんの動き
  - 週次MTGの頻度を徐々に隔週に移行できるか検討
  - ななこちゃんに一部店舗のサポート役を委任する可能性を検討
  - フォロー体制の属人性を下げる

### Phase 4（6月〜7月）：深化 ── 大会に向けた実践

- **目標：SFF2026大会に向けてトレーナー全員がShapeの理念を体現できる状態**
- やること
  - トレーナーが「お客様の挑戦の物語」を語れるレベルになっているか確認
  - 大会出場を目指すお客様への特別なセッション設計をトレーナー主導で行う
  - 協賛企業との接点でもトレーナーがShapeの価値を伝えられるようにする
  - 各店舗の育成成果を振り返り、次年度に活かす

## 補足：各店舗の優先アクション（Phase 1）

### 本店（ななこちゃん）

- フーテルちゃんのMVS発表を確実に実施する
- 週2回の対話を継続し、「語ること」の価値を体験的に伝える
- セッションログの振り返りを1on1の材料として活用し始める

### 柏店（玲奈さん・山本さん）

- まず「セッションログ一日1件」と「NPSをLINEで送る」の2つだけに絞る
- なーすけさんとの週次MTGで「柏店にとってなぜこれが必要か」を一緒に言語化する
- 半年未満の離脱客のデータを使って、ログの意味を実感させる

### 上板橋（ブーちゃん）

- 1on1の記録をミーティング前に打つルールを定着させる（すでに合意済み）
- セッションログ一日1件をトレーナーに依頼する
- NPSはLINE一斉送信で2月中に全員分を取る

### 大阪天六（都築さん）

- 中屋さんのセッションログ記載を開始させる（プリプリとの差を埋める）
- セッションログを一日1件に削減してトレーナーの負担を下げる
- ボイスチェックの頻度問題は3月に向けてなーすけさんが持ち帰り検討
