# エスカレーションフロー設計（提案）

## この資料の目的

Mustを守れないオーナーが出た場合の対応フローを標準化し、「誰が・いつ・何をするか」を全員で合意する。

---

## 前提：すでに決まっていること

- 未達の原因は3つに分類する：**①仕組みが悪い ②手が足りない ③やる気・優先度**
- 2月は「学びのための記録」スタンス
- ステップ1〜2（事実確認・支援）は2月から運用中
- ステップ4（契約条件の見直し）は3月末の全体MTGで方針共有後に運用開始
- 2月の4店舗の1on1では、未実施の原因の大半が**①仕組みの問題**だった（態度の問題ではない）

---

## エスカレーションの全体像

```
通常 → 注意 → 期限設定 → 条件見直し
               ↑
          ここが今回の設計対象
```

| ステップ | 発動条件 | 対応内容 | 実行者 |
|----------|----------|----------|--------|
| 1. 事実確認 | 今週「未実施」が1つ以上 | 1on1で原因を確認。補足メモに記録 | 1on1担当者 |
| 2. 支援 | 原因が①②と判明 | 仕組みの調整・負荷軽減を実施 | 1on1担当者 |
| **3. 期限設定** | 同じMustが**3週連続**未実施 | 具体期限を設定し、実行を求める | **なーすけさん** |
| **4. 条件見直し** | ステップ3を**2サイクル**回しても改善なし | FC契約条件について対話 | **なーすけさん** |

---

## 論点1：ステップ3「期限設定」の具体ルール

### 提案

| 項目 | ルール |
|------|--------|
| 期限 | **次の水曜日まで**（1on1が火曜想定のため、翌日では短すぎ、1週間では長すぎ） |
| 伝え方 | 1on1の中で口頭で伝える＋**チャットワークで文面を送る**（記録を残す） |
| 伝える内容 | 「○○（Must名）を水曜日までに実施してください。実施したらチャットで連絡してください」 |
| 実施報告がない場合 | **木曜日にリマインド**（チャットワークで1文）。それでも金曜までに報告がなければ、翌週の1on1冒頭で確認 |
| 期限内に実施された場合 | 通常運用に戻す。翌週から「連続未実施」のカウントをリセット |
| 期限内に実施されなかった場合 | 翌週の1on1で再度期限設定（**2サイクル目**）。2サイクル目も未実施ならステップ4へ |

### 根拠

柏店に対して「水曜までに実施→連絡→翌日リマインド」をすでに運用しており、これを標準化するのが最も自然。新しい仕組みを作るのではなく、今やっていることをルールにする。

---

## 論点2：ステップ3→4の移行条件

### 提案

> **ステップ3を2サイクル（＝2週連続で期限を切って、2回とも未達）**でステップ4に移行

### ステップ4で話し合う内容

以下の3段階を順に検討する（いきなり契約解除の話はしない）：

1. **Mustの内容を変える**：「このMustは現実的に守れないので、代替のMustに変更できないか」
2. **支援体制を変える**：「本部側の関与を増やす（例：事務局がログ入力を代行する期間を設ける）」
3. **FC契約条件の見直し**：上記1・2で解決しない場合に初めて、ロイヤリティ条件や契約継続について対話する

### ステップ4の判断者

**なーすけさんが最終判断。** 事務局（ななこちゃん・鈴木）は事実整理と選択肢の準備を行い、判断自体はなーすけさんに残す。

---

## 論点3：原因が「①仕組み」「②手不足」の場合の分岐

### 提案

**エスカレーション（ステップ3以降）は「③やる気・優先度」の場合のみ発動する。** ①②の場合は、仕組みの再調整を優先し、連続未実施のカウントに含めない。

ただし、以下の条件を満たした場合は③として再判定する：

> **仕組みの調整を行ってから2週間経っても改善が見られない場合**

具体的な流れ：

```
未実施を検知
  ↓
1on1で原因を確認 →「①仕組み」or「②手不足」→ 調整を実施 → 2週間様子を見る
                                                     ↓
                                               改善あり → 通常運用に戻す
                                               改善なし → ③として再判定 → ステップ3へ
  ↓
「③優先度」→ 連続未実施カウント開始 → 3週連続でステップ3
```

### 判断の権限

**1on1実施者（現在はなーすけさん）が、1on1の中で判断する。** 判断に迷う場合は、事務局と相談の上で決定。判断結果はMUST運用シートの「守れなかった理由」列に記録する（今のフォーマットのまま）。

## 論点4：誰がエスカレーションを実行するか

### 提案

| ステップ | 現在（2〜3月） | 移行後（4月〜） |
|----------|---------------|-----------------|
| 1. 事実確認 | なーすけさん | **ななこちゃん** |
| 2. 支援 | なーすけさん | **ななこちゃん** |
| 3. 期限設定 | なーすけさん | **なーすけさん**（残す） |
| 4. 条件見直し | なーすけさん | **なーすけさん**（残す） |

### 根拠

ステップ1〜2は「事実の確認と仕組みの調整」なので、1on1を引き継ぐタイミングでななこちゃんに渡せる。ステップ3以降は「プレッシャーをかける判断」が伴うため、オーナーとの信頼関係が深いなーすけさんに残す。ななこちゃんが1on1に同席する中で、ステップ3の場面を見る機会を作り、将来的な移行に備える。

## 論点5：オーナーへの事前開示

### 提案

| 時期 | 伝える内容 |
|------|------------|
| **2月（実施中）** | 「1on1で実施状況を一緒に確認し、困っていることがあれば支援します」（ステップ1〜2） |
| **3月末の全体MTG** | 「テスト運用の結果を踏まえ、4月からはこのフローで運用します」と**フロー全体を開示** |
| **4月以降** | 個別のオーナーに「今ステップ○です」とは言わない。ただし、ステップ3に入る際は「期限を決めましょう」と事実上伝わる形になる |

### 3月末全体MTGでの伝え方（案）

- フローの4ステップを見せる
- 「2月のテスト期間で、未実施の大半は仕組みの問題だったので改善した」と事実を先に共有
- 「4月からは、仕組みを整えた上で実行できるかを見ていきます」と方針を説明
- ステップ4の存在は隠さないが、強調もしない。「ここまで来ることはないと思いますが、枠として決めています」程度
