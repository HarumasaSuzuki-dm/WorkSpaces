# [なーすけさんへ｜発信論点整理](https://www.mindmeister.com/app/map/3953057979?t=LVwtndimQu)

 - 全体の発信順序（必ず守ること）
    - オーナー3店舗に個別1on1で伝える（←
まだ独立の話はしていない）
    - 各店舗の顧客へ告知
    - コミュニティ全体（Lステップ等）へ公式発表
    - YouTube / SNS発信
 - A｜オーナーへの1on1 論点
    - 入り口のトーン（最初の入り方）
        - 責めるのではなく、感謝と労いから入る
        - 「一緒にやってきた」という共通の文脈を最初に置く
        - Must運用を続けてきたが、お互いにとって理想的な形ではなかったことを率直に認める
    - 伝える本題
        - 実は双方の望みは一致している、という文脈に持っていく
            - オーナー側：「自由にやりたい」「ロイヤリティを下げたい」
            - 本部側：コア事業に集中したい
        - だから、独立・卒業はどちらにとってもいいことだという提案として伝える
        - 「切る」ではなく「卒業」として前向きに伝える
    - 条件の説明
        - システム移行には猶予期間を設ける（1〜2ヶ月）
        - 金銭精算の方針について説明する
        - 本部がサポートできること・できないことを明確にする
    - クロージング
        - オーナーに持ち帰り・検討の時間を与える
        - 「決めてください」ではなく「一緒に考えましょう」のトーンで終わる
 - B｜コミュニティ向け公式発表 論点
    - 何が起きるか
        - 3店舗が独立・卒業する
        - 「切られた」のではなく、自分たちの力で次のステージへ進む
    - なぜポジティブな話なのか
        - Must運用を通じて、各店舗の自走力が証明された
        - 各オーナーが、自分のビジョンで進めるステージに来た
        - 本部として、ここまで一緒に成長できたことへの感謝
    - 本部はこれからどうするか
        - コア事業（本店・SFF・SFC等）に全リソースを集中させる
        - Shapeそのものをさらに進化させていく
    - 各店舗はどうなるか
        - 同じ想いで営業は継続する
        - Shapeの仲間であることは変わらない
 - C｜YouTube / SNS発信 論点
    - 発信の目的を決める
        - コミュニティへの公式発表とは別に、なぜYouTube/SNSでも発信するのか
        - 例：「Shapeを知らない人にも届けたい」「なーすけ自身の言葉で直接伝えたい」
    - 発信形式を決める
        - YouTube動画にするか、テキスト投稿（SNS）にするか、両方か
        - 動画の場合：Vlog・トーク形式か、編集動画か
        - なーすけさんの自分の言葉で話す形式が自然（原稿読み上げは不要）
    - 発信タイミングを決める
        - コミュニティへの公式発表の直後が原則
        - 「知らない人が先に見てしまう」事態を防ぐため、順序を明確に
    - 発信内容の範囲を決める
        - コミュニティ向けと同じ内容でいいか、外向けに補足が必要か
        - 各店舗の個別事情（金銭・システム）には触れない
        - 「なぜ今これを話すのか」という自分なりの文脈を加えるとより届きやすい
 - この場で決めたい・聞きたいこと
    - それぞれの日程
        - オーナーへの1on1
        - コミュニティ向け公式
        - YouTube / SNS発信
    - なーすけさんからヒアリングしたいこと
 - システム関連QA｜オーナー1on1 参考資料
    - Q1. 今のシステムはいつまで使えるの？
        - 独立合意後、1〜2ヶ月の猶予期間を設ける予定
        - 猶予期間中は今のシステムをそのまま使い続けられる
        - 猶予期間終了後は完全に切り離す（本部の関与はゼロになる）
    - Q2. 独立後は何のシステムを使えばいい？
        - 予約・決済・顧客管理それぞれで使えるSaaS候補を本部が選んでリスト化する予定
        - 2〜3案に絞って提示するので、オーナー自身が選べる形にする
        - ※現在選定中。詳細は追って案内する
        - 可能な限り本部の工数がかからないものを選定予定
            - ※サポートが充実しており、既存システムからの移行をサポートしてもらえるサービスなど
    - Q3. 顧客データはどうなるの？
        - きちんと引き渡す（データはオーナーのもの）
        - 引き渡しの方法・タイミングは移行計画の中で一緒に確認する
    - Q4. 移行にどのくらいの期間・手間がかかる？
        - 猶予期間（1〜2ヶ月）の中で完結できるよう設計する
        - 本部が移行手順マニュアルを用意するので、一から自分で調べる必要はない
        - 不明点があれば猶予期間中は本部に質問できる
    - Q5. 移行後にトラブルが起きたら誰に聞けばいい？
        - 猶予期間中：本部に質問OK
        - 猶予期間終了後：本部のサポートは終了（各自で新しいSaaSのサポートを使う）
        - だからこそ、猶予期間中にしっかり移行を完了させることが重要
