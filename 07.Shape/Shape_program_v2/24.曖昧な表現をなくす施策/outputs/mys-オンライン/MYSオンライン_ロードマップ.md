> 作成日：2026/03/07　最終更新：2026/03/08　対象期間：2026年3月〜2027年以降

---

## 全体の流れ

```
Phase 0（今すぐ）   一緒に走る前に、地図を共有する
Phase 1（3〜4月）   なーすけさんの思想を、渡せる言葉にする

```

## Phase 0｜一緒に走る前に、地図を共有する
**期間：今すぐ**

> このプロジェクトはかずひらさん主導で進める。最初のすり合わせなしに動き出すと、あとで方向がずれる。まずここを完了させることが最優先。

### やること
- かずひらさんとのキックオフMTGを設定（なーすけ・鈴木）
- 「今できているものの完成度」を確認する（前倒し判断の根拠にする）
- 全体像・進め方・役割分担を合意する

### すり合わせで決めること
| 項目 | 内容 |
|---|---|
| このプロジェクトの目的 | 誰に・何を・なぜ届けるか（MYSのコンセプト共有） |
| かずひらさんの役割範囲 | カリキュラム設計・マネージャー育成のどこまでを一任するか |
| なーすけさんの役割 | 体験設計の思想を言語化する人。かずひらさんへのインプット責任者 |
| 鈴木の役割 | 進行管理・各フェーズの成果物管理 |
| 現状の完成度確認 | 体験設計・カリキュラムの骨格がどこまでできているかを棚卸しする |
| スケジュールの確認 | 10〜11月テスト開始から逆算して、各フェーズの締め切りを合意 |
| 延期ライン | 何が揃わなければテスト開始を延期するか（基準を先に合意） |

### 成果物
| 成果物 | 内容 |
|---|---|
| すり合わせ議事録 | 合意内容・決定事項・ネクストアクションが書かれたもの |
| 役割定義書 | かずひら・なーすけ・鈴木それぞれの担当範囲と責任を明文化したもの |

### ⚠️ 注意
このすり合わせの結果によってPhase 1以降が変わる可能性がある。Phase 0が完了するまで、後続フェーズの詳細は仮置きとして扱う。

## Phase 1｜なーすけさんの思想を、渡せる言葉にする
**テーマ：「なーすけさんの頭の中を、かずひらさんが受け取れる形にする」**
**期間：2026年3〜4月**

> 集客・募集は行わない。この期間の唯一のゴールは「体験設計の言語化」。ここが固まると、カリキュラムも価格もLPの言葉も自然に決まっていく。なーすけさんの中にすでにあるイメージを、かずひらさんに渡せるドキュメントに変換することがこのフェーズの本質。

### やること
- かずひらさんとのセッションを重ね、体験設計を言語化する（なーすけ主体）
- MYSオフラインテスト（3〜4月）を観察し、知見をメモに残す
- 「なぜその問いをしたか」「何を見てそう判断したか」をセッション後に毎回記録する
- 3プランの役割分担と価格の妥当性を確認する
- リスト育成の方針を決める（SNS・YouTube発信内容・LINE動線の設計）

### 成果物
| 成果物 | 内容 |
|---|---|
| ① 体験設計ドキュメント | 6ヶ月で受講者に何が起きるか。週ごとのゴールと問いの方向性が書かれたもの |
| ② 判断軸メモ | なーすけさんが「なぜその問いをしたか」「何を見てそう判断したか」の記録 |
| ③ リスト育成方針書 | SNS・YouTubeで何を発信するか。LINE登録の動線をどう設計するかの方針 |

### リスト育成の準備（並行して着手）
- SNS・YouTubeでMYSコンセプトに関連したコンテンツを発信開始
- 「興味がある方はLINEへ」という動線だけ用意する（売り込まない）
- 発信テーマの選定・投稿頻度・担当者を決める

## Phase 2｜サービスとして成立させる土台を作る
**テーマ：「10月に走り出せる状態を、6月末までに揃える」**
**期間：2026年5〜6月**

> カリキュラム作成だけでなく、プラットフォーム・LP・法務・マネージャー準備まで含めて「サービスとして成立する状態」を作るフェーズ。7月以降はSFFに集中するため、6月末を実質的な準備完了のデッドラインとして設計する。

### やること（かずひら主導）

**カリキュラム関連**
- 体験設計ドキュメントをもとに、カリキュラム初期4〜6週分を作成
- ワークシート・提出フォーマットの作成
- マネージャー向けFBガイドラインの初稿作成

**サービス基盤関連**
- 動画プラットフォームの選定・決定（UTAGE / Discord等）
- LP仕様の設計（本制作は9月以降でよい）
- 法務整備の着手（利用規約・特商法・契約書）

**マネージャー準備**
- 採用候補のリストアップ（コミュニティ内・SFF参加者等）
- 採用・研修のスケジュール設計（10月実働から逆算）

### 成果物
| 成果物 | 内容 |
|---|---|
| ① カリキュラム動画（初期4〜6週分） | テスト受講者に提供できる品質のもの |
| ② ワークシート・提出フォーマット | 受講者が毎週提出するもの |
| ③ マネージャーFBガイドライン初稿 | オフラインマニュアルをベースに転用 |
| ④ 動画PF確定 | UTAGEかDiscordか、どちらを使うか決定 |
| ⑤ LP仕様書 | LP本制作に渡せる仕様のドキュメント |
| ⑥ 契約書類初稿 | 利用規約・特商法・契約書のチェック済み初稿 |
| ⑦ マネージャー採用計画書 | 採用先・スケジュール・研修内容の設計 |

### リスト育成の継続
- SNS・YouTube発信を継続（Phase 1で設計したプランに沿って）
- LINE登録者のリスト管理と育成
- SFF参加者との関係構築を意識した動線設計

## Phase 3｜SFFに全力を注ぎ、MYSの火は守る
**期間：2026年7〜9月**

> なーすけさん・鈴木ともにリソースはSFF優先。MYSで「新しく進める」ことはしない。ただし、リスト育成だけは止めない。6月末までに土台が揃っていれば、このフェーズで止まっても10月のスタートは成立する。

### なーすけさんがやっていいこと（MYS関連）
- SFF参加者との接点でMYSへの興味を育てる（会話・SNS発信）
- 体験設計について「考える時間を積み上げる」（行動ではなく思考の蓄積）

### 鈴木がやっていいこと（MYS関連）
- 動画PF最終確認・LP仕様の最終調整など**意思決定不要の軽作業のみ**
- かずひらさんとの進捗確認（短時間）

### やってはいけないこと（この期間）
- マネージャー採用・研修の開始
- テスト受講者の募集
- 新しい設計作業の着手

### 成果物
| 成果物 | 内容 |
|---|---|
| SFF完走 | 大会の運営・本番・振り返りが完了している状態 |
| 10月再開準備メモ | 9月末時点での「残タスク・確認事項・ネクストアクション」を整理したもの |

### ⚠️ 9月は特に
SFF関東大会の運営・本番・振り返りに専念する月。10月以降に備えた状態を維持することだけを優先する。

## Phase 4｜10〜20人と一緒に、設計を検証する
**期間：2026年10〜11月〜**

> SFF関東大会終了・振り返り完了後、10月から本格再開。広告は使わず、育てたリストとSFF参加者への告知で集める。

### テスト開始のGo条件（これが揃わなければ延期）
| 項目 | 状態 |
|---|---|
| カリキュラム動画（初期4〜6週分） | 完成済み |
| LP（最低限の品質） | 公開できる状態 |
| 決済導線 | UTAGE/Stripe等で稼働 |
| マネージャー1名 | 実働できる状態 |
| LINEリスト | 見込み顧客が一定数いる状態 |

### 集客方法
- 広告には頼らない
- 育てたLINEリスト・SNSフォロワーへの告知
- SFF参加者・既存コミュニティへのDM・口コミ

### テスト期間中にかずひらさんと確認すること（KPI）
| 確認項目 | 目安基準 |
|---|---|
| 受講者のポジティブ評価 | 過半数 |
| 途中離脱率 | 20%以内 |
| マネージャーの自走度 | なーすけ介入なしで回る受講者が半数以上 |
| 友人に勧める抵抗感（NPS） | 事前に閾値を合意しておく |

### 成果物
| 成果物 | 内容 |
|---|---|
| KPI記録 | 上記4項目の計測結果 |
| 受講者フィードバック | 「自分の物語が動いた瞬間」を自分の言葉で語れているかの記録 |
| Go/No-Go判断レポート | 本ローンチに向けた判断と根拠をまとめたもの |

## Phase 5｜検証済みの設計を、広げる
**期間：2027年〜**

> テストの結果を踏まえてプラン・価格・マネージャー体制を確定させ、広告を本格投下。

### やること
- 3プラン構成での正式提供開始（セルフ・スタンダード・プレミアム）
- 広告本格投下（Instagram等）
- マネージャー体制の拡充（最終的に3〜6名体制）

---

## 「設計しきる」の完了定義

今期テーマ「体験を設計しきる」が完了したと言える状態の定義。

| レベル | 完了条件 |
|---|---|
| **最低ライン** | カリキュラム初期4〜6週分の動画が完成し、マネージャー1名がFBできる |
| **今期の本当のゴール** | なーすけさんの体験設計の思想が文書化され、実行メンバーが「なぜこの順番か」を自分の言葉で説明できる |
| **設計しきった状態** | 受講者が「この週で自分の物語が動いた」と言える瞬間が設計に組み込まれている |

---

## ⚠️ 延期ライン（テスト開始を延期する条件）

以下のどれか一つでも揃っていなければ、10〜11月のテスト開始を延期する。

1. カリキュラム動画（初期4〜6週分）が完成していない
2. マネージャーが実働できる状態にない
3. LINEリストに見込み顧客がほぼいない
4. LP・決済導線が稼働していない

> **延期した場合の代替：** 2027年1〜2月スタートとし、本ローンチとの差を縮める形で設計し直す。
