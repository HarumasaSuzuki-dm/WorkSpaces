# SFF（Shape Fit Festival）ロードマップ
> 作成日：2026/03/08　対象期間：2026年3月〜10月

---

## 全体の流れ

```
Phase 0（今すぐ）    全員の地図と役割を揃える
Phase 1（3〜5月）   スポンサーを獲りにいく土台をつくる
Phase 2（5〜7月）   スポンサーを動かしながら、大会を磨く
Phase 3（7〜9月）   全力で2大会を完走する
Phase 4（9〜10月）  今期の動きを、来年の財産にする
```

---

## Phase 0｜全員の地図と役割を揃える
**テーマ：「同じ地図を持って、スタートラインに立つ」**

> 誰が何をやるかが決まっていない状態でスタートすると、確認・判断の往復が増える。最初にここを完了させることが最優先。

### やること（なーすけ × 鈴木）
- SFF関係メンバー全員とキックオフMTGを設定
- 今期テーマ「スポンサー収益の仕組みを確立する」の意味を全員と共有する
- 以下の役割分担を合意する

### 役割分担（たたき台）
| 役割 | 担当者 | 内容 |
|---|---|---|
| スポンサー営業リード | 鈴木 | ターゲット選定・アプローチ・交渉の主担当 |
| スポンサー営業サポート | ななこちゃん | 提案書作成・日程調整・記録・徐々に主担当へ |
| 大会品質・演出設計 | しおんさん | 選手体験・当日演出・コンテンツ企画 |
| スポンサー引き継ぎ元 | しおんさん | Phase 1前半：鈴木・ななこちゃんへの知識・関係性の引き継ぎ |
| 仕組み化・進行管理 | 鈴木 | 型の言語化・ドキュメント管理・フェーズ管理 |
| 経営判断・最終確認 | なーすけ | 外部提出物の確認・重要判断 |

> ⚠️ 上記は叩き台。Phase 0のMTGで全員合意してから確定する。

### 決めること（MTGで合意する項目）
| 項目 | 内容 |
|---|---|
| 今期テーマの共有 | 「仕組みを確立する」が日常の判断軸になるよう言葉で確認 |
| 役割分担の確定 | 誰がスポンサー・大会設計・記録を担うか |
| 相談トリガーの合意 | 10分考えて進まなければ相談・外部に出るものは事前確認 |
| 依頼分類ルールの導入 | 「承認がほしい」「フィードバックがほしい」「情報共有だけ」の3種に分ける |
| なーすけさんへの確認ラインの設定 | どんな判断はなーすけさんを通すか先に決める |

### ⚠️ 注意
Phase 0が完了するまで、営業活動は正式にスタートしない。ただしターゲット候補のリストアップは先行してよい。

## Phase 1｜スポンサーを獲りにいく土台をつくる
**テーマ：「しおんさんの知識と関係性を、チームの力に変える」**
**期間：2026年3〜5月**

> スポンサー営業を「個人の頑張り」で終わらせない。しおんさんが持っているスポンサー対応のノウハウ・関係性を鈴木・ななこちゃんに引き継ぎ、チーム全体で動ける状態をつくる。提案書・ターゲットリスト・進め方の流れを記録し、来年以降に別の人（業務委託を含む）が同じように動ける状態にする。

### 引き継ぎのステップ（しおんさん → 鈴木 → ななこちゃん）

#### STEP 1｜しおんさんから鈴木への引き継ぎ（3月）
- しおんさん × 鈴木でのヒアリングMTGを設定
- 現状のスポンサー対応状況・候補企業・交渉の経緯を共有
- 鈴木が理解を整理し、ドキュメントに落とす
- **初回の企業MTGは鈴木がメインで発話。しおんさんはサポートに回る**

#### STEP 2｜ななこちゃんの段階的な同席（4月〜）
- 引き継ぎが一定完了したタイミングで、ななこちゃんを企業MTGに同席依頼
- 最初はななこちゃんは聞く側。徐々に発話の場面を渡していく
- 鈴木 → ななこちゃんへの日常的な共有・フィードバックを習慣化

#### STEP 3｜チームとして動ける状態へ（5月〜）
- 鈴木・ななこちゃんそれぞれが、スポンサー企業との打ち合わせを自分で進行できる
- しおんさんは大会品質・演出設計へリソースを移す
- スポンサー対応の主体がチームに完全移行する

### MYSとの関係（この時期）
- MYS Phase 1（体験設計の言語化）と並走中
- なーすけさんのリソースはMYSの思想整理にも使われる時期
- SFF側での判断相談は「必要最小限・事前にまとめて持ち込む」を意識する

### この期間の完了条件（成果物）
| 成果物 | 内容 |
|---|---|
| ① ターゲットリスト | 業種・担当者・接触履歴・優先度が書かれたもの |
| ② 提案書テンプレート | 誰でも使える提案書の雛形（なーすけ確認済み） |
| ③ アプローチ記録 | 誰にいつどんな方法でアプローチしたかの記録 |
| ④ 断られた理由の記録 | 「なぜ断られたか」を蓄積する（型づくりの素材） |
| ⑤ 引き継ぎドキュメント初稿 | しおんさんのノウハウ・関係性が言語化されたもの |

### 日常の判断軸
- 新しいスポンサー候補が出たとき → 「来年も続けられる関係になりそうか」で判断
- 提案書や資料をつくるとき → 「自分以外の人が読んでも意味がわかるか」を確認してから提出
- 何かを始めようとするとき → 「この動きは仕組みになるか？来年も同じやり方で動けるか？」で確認

## Phase 2｜スポンサーを動かしながら、大会を磨く
**テーマ：「獲るフェーズから、育てるフェーズへ」**
**期間：2026年5〜7月**

> スポンサー営業と大会の演出・選手体験の設計は「対立」しない。時期によって前面に出すものが変わるだけ。並走の優先順位を先に合意しておくことでリソースの摩擦を防ぐ。

### やること
**スポンサー（鈴木・ななこちゃん）**
- アプローチ中の企業との交渉・クロージング
- 新規アプローチの継続
- 獲得した企業への対応・関係構築
- **ななこちゃんが打ち合わせの進行を主体的に担える場面を増やす**

**大会設計（しおんさん）**
- 演出・コンテンツ・当日タイムラインの設計
- 選手体験・ファン体験の設計（「この大会への出場が物語の一部だった」と感じてもらう）
- SFF参加者をMYSの見込み客リストに入れる動線の設計

**仕組み化（鈴木）**
- 進行中の営業プロセスを記録・整理
- 提案書テンプレートのアップデート（実際に使って出てきた改善点を反映）
- 大会設計の判断記録（何を決めたか・なぜそうしたか）

### リソースが競合したときの優先順位
> 迷ったときは「スポンサーか大会か」ではなく「仕組みになるか」で判断する

- スポンサー対応と大会準備が同じ日にある → スポンサー対応を優先（期限があるため）
- どちらも仕組みになるなら → 記録の濃い方を優先

### この期間の完了条件（成果物）
| 成果物 | 内容 |
|---|---|
| ① 大会設計の初稿 | 演出・タイムライン・選手体験の骨格 |
| ② スポンサー獲得実績の記録 | 何社・どのルート・どの提案で決まったかの記録 |
| ③ MYS動線の設計 | SFF参加者からMYSに興味を持ってもらう導線の設計 |

## Phase 3｜全力で2大会を完走する
**テーマ：「スポンサーへの約束を、大会で証明する」**
**期間：2026年7〜9月（大阪：夏、関東：9月）**

> この期間はSFF優先。スポンサーの「新規獲得」は止めて、すでに獲得した企業への体験提供に集中する。「来年もまた出したい」と言ってもらえるかどうかが今期の勝負。

### 役割別のやること
**なーすけさん・鈴木**
- 両大会の運営・本番・振り返りへの専念
- スポンサー企業の当日対応（露出・コミュニケーション）

**しおんさん**
- 選手体験・演出の実現
- 当日の品質管理

**鈴木・ななこちゃん**
- スポンサー当日対応のメイン担当
- 当日の気づき・スポンサーの反応をその場で記録
- 「来年また出したいか」を感じ取る観察

### やってはいけないこと（この期間）
- 新規スポンサーへのアプローチ
- 大会とは関係ない新施策の着手
- 記録を後回しにする（その場で書く習慣を守る）

### ⚠️ 9月は特に
SFF関東大会の運営・本番・振り返りに専念する月。10月の「資産化期」に備えた状態を維持することだけを優先する。

## Phase 4｜今期の動きを、来年の財産にする
**テーマ：「個人の頑張りを、チームの仕組みに変える」**
**期間：2026年9〜10月**

> SFF関東大会の振り返りが終わったら、今期の全プロセスを「仕組み」として整理する。来年、別の人（業務委託を含む）がこのドキュメントを読めば同じ動きができる状態をゴールにする。

### やること（鈴木・ななこちゃん・しおんさん）
- スポンサー営業マニュアルの完成版作成
- ターゲットリストの来年版アップデート
- 大会運営プロセスの文書化
- 「機能した動き」「機能しなかった動き」の整理

### マニュアルに含めること（引き継ぎドキュメント）
| 項目 | 内容 |
|---|---|
| ターゲットリスト | 業種・担当者・優先度・来年のアプローチ方針 |
| 提案書テンプレート | 今期改善版・使い方の説明つき |
| アプローチフロー | 初回接触から契約までのステップと判断基準 |
| 断られパターンと対応 | よくある断りとその返し方・見極め方 |
| スポンサー当日対応マニュアル | 当日の動き・露出設計・アフターフォローの流れ |

### なーすけ × 鈴木の進捗確認（このフェーズで実施）
- 今期の協賛実績と来期目標のすり合わせ（二人のみで実施）
- 来期の仕組みを誰が担うかの検討（業務委託への移行可否を確認）
- 来期に向けた「やること・やらないこと」の合意

---

## 「仕組みを確立した」の完了定義まとめ

| レベル | 完了条件 |
|---|---|
| **最低ライン** | 今期のアプローチ記録・提案書・ターゲットリストが残っている |
| **今期の本当のゴール** | 来年、別の担当者がこのドキュメントを読めばゼロから動き出せる |
| **仕組みが確立した状態** | スポンサー企業が「来年もまた出したい」と自ら口にしている |

---

## ⚠️ 延期・方針変更ライン

以下のどれか一つでも発生したら、なーすけ × 鈴木で判断を持つ。

1. Phase 1でターゲットリストが揃わないまま5月を迎えた
2. Phase 2でスポンサー獲得ゼロが続き、提案書の方向性を見直す必要が出た
3. 大会集中期（Phase 3）にスポンサー対応と運営のリソースが明らかに不足している
4. Phase 4で「仕組みとして記録できた」と言える状態になっていない

> 問題が発生したタイミングで早めに共有し、判断を止めないことを優先する。
