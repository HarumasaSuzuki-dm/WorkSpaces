## ❷ このプロジェクトについて

### ひとことで言うと

FC（フランチャイズ）3店舗を円満に「独立・卒業」させ、本部のリソースをコア事業に集中できる状態をつくるプロジェクト。

### 背景

- FC支援にかかるコスト（システム月20万＋人的工数）が、ロイヤリティ収入に見合っていない
- 本部メンバーのリソースがFC支援に取られ、本店経営・SFF・SFC等のコア事業に集中できていない
- 1年間Must運用を試みたが、FCオーナーの腹落ちが得られず形骸化した。一方でFC側からも「自由にやりたい」「ロイヤリティを下げたい」という声が出ており、双方の望みは実は一致している

### ゴール

- ✅ 全FC店舗（柏・上板橋・大阪天六）が「独立・卒業」として円満に移行を完了している
- ✅ オーナーが「切られた」ではなく「自分の力で次に進む」とポジティブに受け取っている
- ✅ Shapeコミュニティに悪影響が出ておらず、本部のリソースがコア事業に集中できている

### スコープ

| やること | やらないこと |
| --- | --- |
| FC3店舗の独立に向けた契約・金銭・法務の整理 | 本店の経営強化（別プロジェクト） |
| オーナーとの個別対話の設計・実施 | SFF・SFC等のサービス開発（別プロジェクト） |
| システム移行（予約・決済・顧客DB）の計画・実行 | 既存のMust運用プロジェクトの運用・改善（別プロジェクト） |
| ブランド移行（屋号・看板・SNS・Googleビジネス）の調整 | FC以外のパートナーシップの見直し |
| コミュニティ向けの公式発表の設計・実施 |  |
| 移行期間中のMust運用の段階的縮小方針の決定 |  |

## ❸ 体制と役割

### 体制図

| 名前 | 役割 | 主な担当 | 判断を仰ぐ人 |
| --- | --- | --- | --- |
| なーすけ | オーナー | オーナーとの1on1、最終判断（金銭精算・伝え方）、公式発表 | ー |
| ぼんちゃん | PM | 契約・金銭の整理、移行設計、資料作成、スケジュール管理 | なーすけ |
| かおん | ー（情報共有のみ） | 定例に参加し、情報提供や進捗共有を受ける | ー |
| ななこちゃん | ー（情報共有のみ） | 定例に参加し、情報提供や進捗共有を受ける | ー |

※外部の関係者

- **宮崎さん**：現在のシステム運用を担当（月20万）。システム移行時に調整が必要
- **弁護士**：契約不備があった場合に相談（必要に応じて）

### 判断ルール

**なーすけさんに確認が必要なこと：**

- オーナーへの伝え方・トークスクリプトの最終決定
- 金銭精算の方針（回収/放棄/分割）の最終判断
- 公式発表の内容・タイミングの決定
- 「一緒にやる」選択肢の条件定義
- 移行期間中のMust運用方針の最終判断

**PMの判断で進めていいこと：**

- タスクの優先順位変更や期限の微調整
- 調査・数値整理などの作業の進め方
- 弁護士・宮崎さんとの事務的なやり取り
- 資料のドラフト作成

## ❹ フェーズと全体スケジュール

### フェーズ一覧

| フェーズ | 意味 | 完了の目安 |
| --- | --- | --- |
| **① 企画** | 事実を揃え、方針を固める | 契約・金銭・システムの事実が整理され、なーすけさんと独立移行の方針が確定した状態 |
| **② 準備** | オーナーと個別に対話し、合意を形成する | 全店舗のオーナーとの対話が完了し、店舗ごとの移行条件が確定した状態 |
| **③ 実行** | 合意に基づき、システム・ブランド・顧客の移行を進める | 全店舗の移行が完了し、残務処理も終わった状態 |
| **④ 振り返り** | 移行全体を振り返り、学びをまとめる | 振り返りが完了し、コア事業へのリソース集中が始まっている状態 |

### スケジュール概要

```
① 企画：2025/2月〜3月第2週 🔵
  └ 契約確認、金銭整理、システム棚卸し、方針確定、トークスクリプト設計
    ├ ステップ1（事実を揃える）：2/18〜2月末
    └ ステップ2（方針確定）：3月第1〜2週

② 準備：2025/3月中旬〜4月末
  └ 柏店→上板橋店→大阪天六店の順に個別1on1、検討期間、条件調整
    ├ 柏店との対話・合意：3月第3週〜4月第1週
    ├ 上板橋店との対話・合意：4月第1〜3週
    └ 大阪天六店との対話・合意：4月第2〜4週

③ 実行：2025/5月〜7月中旬
  └ 移行チェックリスト作成、システム切り替え、ブランド移行、顧客告知、公式発表
    ├ 5月：移行設計の確定
    ├ 6月：移行の実行
    └ 7月：完全独立の完了・残務処理

④ 振り返り：2025/7月下旬
  └ 移行全体の振り返り、学びのまとめ、コア事業へのリソース集中開始
```

## ❺ 動き方のルール

### 定例・報告

| 項目 | 内容 |
| --- | --- |
| 定例ミーティング | 毎週水曜 16:00〜17:00
参加者：なーすけ、ぼんちゃん、かおん、ななこちゃん。
進捗確認・課題の共有・次週のアクション確認を行う |
| 進捗報告 | 定例ミーティングの場で口頭＋議事メモで共有 |
| 緊急時の連絡先 | ChatWorkで、ぼん → なーすけ に連絡 |

### タスクの進め方（共通ルール）

- 新しいタスクが発生したら、自分でタスク管理に追加する。PMに言われる前に書く
- タスクが終わったら、自分でステータスを「完了」に変える
- 期限に間に合わないと分かった時点で、PMに相談する（期限を過ぎてから言わない）
- 判断に迷うことがあったら、体制図の「判断を仰ぐ人」に聞く

### プロジェクト固有のルール

- **オーナーとの対話は必ずなーすけさんが行う。** PMは対話の準備（資料・トークスクリプト）と事後の条件調整を担当する
- **トークスクリプトは必ず事前にリハーサルする。** ぶっつけ本番でオーナーとの1on1に臨まない
- **店舗間の情報伝播に注意する。** 柏店との対話後1週間以内に、他店舗へ「順番にお話しする」旨の一報を入れる
- **「切られた」と感じさせない言葉づかいを徹底する。** 「解約」ではなく「独立・卒業」として伝える
- **Must運用の成果を「独立の根拠」として使う。** 例：「Mustを通じて各店舗の自走力が見えた → だから独立できる」
