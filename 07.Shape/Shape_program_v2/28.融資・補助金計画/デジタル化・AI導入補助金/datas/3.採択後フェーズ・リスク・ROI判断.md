# 🏢 IT導入補助金（デジタル化・AI導入補助金）採択後フェーズ 実務ガイド

> **制度名称の注記**：2025年度まで「IT導入補助金」として運営。2026年度（令和8年度）より「**デジタル化・AI導入補助金2026**」に名称変更。構造・フローは概ね継承されており、本ガイドは両制度に適用可能。

## 1️⃣ 実績報告の手続き

### ▶ 必要書類一覧（全類型共通＋条件付き）

| # | 書類名 | 詳細・注意点 | 必要条件 |
|---|--------|------------|---------|
| ① | **請求書** | IT導入支援事業者が発行。ITツール名・数量・単価・請求日が必須 | **全類型共通** |
| ② | **支払証憑** | 支払方法により異なる（下表参照） | **全類型共通** |
| ③ | **ソフトウェア利用確認** | 管理画面・ダッシュボードのスクリーンショット（ログインIDが確認できるもの） | **全類型共通**（セキュリティ枠を除く） |
| ④ | **実施内容説明資料** | 指定様式に記載（役務提供の実績を証明） | 大分類Ⅲ役務導入時のみ |
| ⑤ | **ハードウェア納品書・写真** | 設置状態＋型番ラベルが写り込んだ写真 | HW導入時のみ |
| ⑥ | **補助金受取口座情報** | 通帳表紙＋取引ページ、またはネットバンキング画面 | **全類型共通** |
| ⑦ | **契約書または利用申込書** | サイバーセキュリティお助け隊サービスのもの | セキュリティ対策推進枠のみ |
| ⑧ | **取得財産等管理台帳** | 指定様式に入力 | 単価50万円以上かつ「買取」形態のツール |
| ⑨ | **従業員一覧** | 申請マイページからDLした様式を使用 | インボイス対応類型の小規模事業者のみ |

#### 支払証憑の種類（支払方法別）

| 支払方法 | 必要な証憑 |
|---------|-----------|
| ATM振込 | ATM利用明細 ＋ 通帳の表紙・取引ページ |
| 金融機関窓口振込 | 振込依頼書（口座振替依頼書）＋ 通帳の表紙・取引ページ |
| ネットバンキング | 振込完了画面（「承認待ち」「作成中」はNG）または取引状況照会ページ |
| クレジットカード | クレジットカード利用明細（支払日が明記されたもの） |

> ⚠️ **実務上の要注意点**：ネットバンキングは「**送金完了ステータス**」のみ有効。「承認待ち」「未完了」などは支払いが完了したとみなされず不備となります。

---

### ▶ 提出システム・期限

| 項目 | 内容 |
|------|------|
| **提出システム** | 「**申請マイページ**」（IT導入補助金専用。JグランツはIT導入補助金では使用しない） |
| **提出フロー** | ① 補助事業者が入力・証憑添付 → ② IT導入支援事業者が内容確認・入力 → ③ 補助事業者が最終確認・提出 |
| **提出期限** | 各公募回の定める「実績報告期間」内（補助事業完了後30日以内が目安 ※**要確認**：各回の公募要領で確定日付を確認のこと） |

> ⚠️ **重要**：**交付決定前に発注・契約・支払いを行うと補助金を受けられません**。交付決定通知を受け取ってから事業を開始することが大前提です。

---

### ▶ 事務局による確認・現地調査

| 確認手段 | 詳細 |
|---------|------|
| **確定検査（書類審査）** | 提出書類の内容・入力事項を事務局がチェック。書類の不備があると修正指示が来る |
| **現地調査** | 制度上は実施の余地があるが、一般的な案件での実施は稀。**要確認**（公募要領の確認が必要） |
| **不正調査** | 会計検査院が抜き取り調査を実施（2020-22年度調査では8%で不正が発覚） |

> **対応方法のポイント**：不備指摘を受けた場合、**指定期限内に修正・再提出しないと交付取消になります**。チェックリスト（手引き巻末）を活用した事前自己点検が効果的です。

## 2️⃣ 補助金の入金タイミング

### ▶ 実績報告から入金までのリードタイム（目安）

```
交付決定通知
  ↓
ITツール 発注・契約・支払い（自社で全額立替）
  ↓
実績報告（申請マイページから提出）
  ↓【審査：平均1〜2か月】
確定検査・補助金額確定
  ↓【補助事業者がSMS認証で承認】
補助金振込（確定後2〜4週間）
  ↓
入金
```

| フェーズ | 所要期間（目安） |
|---------|---------------|
| 実績報告の審査 | 1〜2か月 |
| 審査通過 → 振込 | 2〜4週間 |
| **実績報告から入金までの合計** | **約1.5〜2.5か月** |
| **採択（交付決定）から入金までの合計** | **最大5〜6か月** |

---

### ▶「後払い構造」の実務的影響

IT導入補助金は**完全な事後精算（後払い）型**です。これが実務上の最大の注意点です。

| リスク項目 | 影響 | 対策 |
|-----------|------|------|
| **全額立替の必要** | 補助率1/2なら補助額分も一時立替が必要。例：導入費200万円なら、補助金100万円分を含む200万円を先払い | 運転資金の事前確保 |
| **資金繰り悪化** | 交付決定〜入金まで最大5〜6か月の空白期間 | つなぎ融資・日本政策金融公庫の活用を検討 |
| **SMS承認の失念** | 確定検査通過後に補助事業者がSMS認証で「承認」しないと補助金が交付されない | 担当者への通知設定・メール確認フロー確立 |

---

### ▶ 概算払い・早期入金制度

| 項目 | 内容 |
|------|------|
| 概算払い | **⚠️ 要確認**：一般的な中小企業向けの通常申請では概算払い制度の存在は確認できず、現状は原則対象外とみられる |
| 早期入金制度 | **⚠️ 要確認**：特段の早期入金制度の記載なし。分割払い可能な事業者の選定や、ファクタリング活用が現実的な対策 |

[情報源：PULL-NET（2025年版入金時期解説）](https://www.pull-net.jp/it-subsidy-2025/when-payment/)

## 3️⃣ 採択後の「事業化報告・効果報告」義務

### ▶ 報告義務の期間（枠別）

| 申請枠 | 効果報告の義務期間・回数 |
|--------|------------------------|
| **通常枠（デジタル化・AI導入枠）** | 事業終了後 **3年間、年1回（計3回）** |
| **セキュリティ対策推進枠** | **3年度目に1回のみ** |
| **インボイス対応類型（旧デジタル化基盤導入枠）** | **原則1回**（賃上げ加点適用時は別途判定あり） |

### ▶ 報告内容

| 区分 | 報告項目 |
|------|---------|
| **経営数値** | 給与支給総額、事業場内最低賃金、売上高、原価、従業員数、就業時間 |

| **セキュリティ** | SECURITY ACTION継続状況、被害の有無、お助け隊サービス継続利用状況（セキュリティ枠） |
| **インボイス関連** | インボイス発行・受領機能の利用実績（インボイス類型） |
| **賃上げ達成状況** | 賃上げ加点を受けた事業者は、3年度目に賃上げ要件の達成状況を報告 |

### ▶ 未提出・未達の場合のペナルティ

| 事由 | ペナルティ |
|------|-----------|
| **効果報告期間内に報告がない・完了しなかった** | 補助金の全部または一部の**返還請求** |
| **賃上げ目標の要件が未達成と確認された** | 同上（返還請求） |
| **効果報告前に辞退** | 同上 |

## 4️⃣ 不正受給・コンプライアンスリスク

### ▶ 補助対象外経費の典型例（やってはいけないこと）

| 類型 | 具体例 |
|------|--------|
| **他補助金との重複計上** | 同一経費をものづくり補助金・事業再構築補助金等に二重申請 |
| **実質無償・値引きの隠蔽** | ツール代金を全額支払ったように見せてポイントや現金で後から返金を受ける |
| **未登録ツール・サービスへの流用** | 事務局に登録されていないホームページ制作、コンサルティング等の役務 |
| **自社以外が利用するソフトウェア** | 他者への供与・転売目的のソフトウェア導入費 |
| **申請金額のミス** | 金額の桁数間違い（1桁多く記載など）も不正受給とみなされるリスクあり |
| **補助要件偽装** | 従業員数の過少申告など企業実態の偽装 |

---

### ▶ IT導入支援事業者との間で起きやすいトラブル・不正の類型

> 🚨 **会計検査院報告（2024年10月公表）：2020〜22年度の376事業者のうち118事業者（**約31%**）で計約3億9,800万円の不適切受給が判明。不正受給率は約8%。**

| トラブル・不正の類型 | 手口の詳細 |
|-------------------|-----------|
| **キックバック（実質無料スキーム）** | ツール購入費用を後日、現金・口座振込・ポイントで申請者に還元。証憑上の金額と実際の負担額を乖離させる |
| **なりすまし代理申請** | 補助事業者のGビズIDをIT導入支援事業者が共有・利用し、申請手続き全体を代行 |
| **実態のない役務** | 「導入研修・コンサルティング」として計上したが実際には実施されていない |
| **役割分担の不明確化** | 申請書作成の責任の所在が曖昧になり、不正記載に気づかずに申請してしまう |
| **ITツール名・金額の誤りの放置** | 支援事業者主導の申請において、ツール名や金額の誤りがそのまま提出される |

---

### ▶ 発覚した場合の対応・ペナルティ

| ペナルティ | 詳細 |
|-----------|------|
| **交付決定取消** | 補助金の交付決定が遡及的に取り消される |
| **補助金全額返還** | 受給した補助金の全額を指定期日までに返還 |
| **加算金（年10.95%）** | 補助金を受け取った日から起算して**年率10.95%** |
| **延滞金（年10.95%）** | 指定期限までに返還しない場合、未返還額に追加で**年率10.95%** |
| **社名公表** | 中小機構のWebサイト上で不正事業者として公表 |
| **刑事事件化** | 詐欺罪に問われるリスク（懲役・罰金） |
| **今後の申請禁止** | 補助金・助成金への申請が一定期間不可になる |

> **知らなかった場合**：不正と知らずに関与した場合は「誓約書」を提出すれば加算金が免除される場合があります。知っていた場合は「自己申告書」の提出と加算金が必須です。

## 5️⃣ ROI・機会費用の判断軸

### ▶ 申請・報告にかかる工数の目安

| フェーズ | 作業内容 | 工数目安（自社対応の場合） |
|---------|---------|--------------------------|
| **申請準備** | gBizID取得、SECURITY ACTION宣言、IT導入支援事業者の選定・相談 | **5〜10時間** |
| **交付申請** | IT導入支援事業者と共同での申請書作成・入力 | **3〜8時間** |
| **実績報告** | 書類収集・整理、申請マイページ入力 | **3〜6時間** |
| **効果報告（毎年）** | 経営数値の集計、画面キャプチャ取得、入力 | **1〜3時間/回** |
| **合計（通常枠・3年間）** | 申請〜効果報告完了まで | **約20〜35時間**（目安） |

> ⚠️ **「要確認」の注記**：上記工数は筆者推計です。IT導入支援事業者のサポート水準によって大きく変動します（支援が手厚いベンダーなら半分以下になることも）。行政書士等に依頼する場合は追加費用（5〜20万円程度）が別途発生します。

---

### ▶ 「補助なし早期導入」vs「補助あり遅延導入」の損益分岐イメージ

以下は、**導入費用200万円・補助率1/2（補助額100万円）・月次効果10万円の削減効果**を想定した概算比較です。

```
【補助なし・今すぐ導入パターン】
  Month 0:  -200万円（全額自己負担）
  Month 6:   +60万円（6か月分の効果累積）
  Month 20:  +0万円（投資回収完了：200万÷10万/月）

【補助あり・6か月後導入パターン（採択・審査・交付決定を経て）】
  Month 6:  -100万円（実質自己負担200万−補助100万、入金後）
            ※ただし最初に200万の立替が必要
  Month 16: +0万円（実質自己負担回収：100万÷10万/月、Month6起算）

→ 補助あり遅延導入の方が、総コストは100万円有利
  ただし、立替200万円の資金繰り負担と6か月の導入遅延コストを要考慮
```

**損益分岐の考え方：**

| 判断軸 | 補助なし早期導入が有利 | 補助あり遅延導入が有利 |
|--------|----------------------|----------------------|
| **導入効果の即時性** | 月次効果が大きく、遅延コストが高い | 月次効果が小さい、または現状維持で問題ない |
| **資金繰り** | 立替資金の調達が困難 | 手元資金に余裕あり |
| **採択確率** | 採択率が低い競争的公募の場合 | 採択率が高い、複数回申請可能 |
| **申請工数の機会費用** | 経営者の時間単価が高い | 専門家サポートで工数を抑えられる |

### ▶ 採択されなかった場合のコンティンジェンシー（代替手段）

| 代替手段 | 特徴・使い分け |
|---------|--------------|
| **自己資金での即時導入** | 最もシンプル。特にSaaS型（月額費用型）であれば初期投資が少なく回収も早い |
| **レンタル・サブスク・月次払い交渉** | 初期一括払いを避けることで資金繰りリスクを軽減 |
| **ものづくり補助金** | 製造業・サービス業のデジタル設備投資に活用可能（補助率1/2〜2/3） |
| **事業再構築補助金** | ⚠️ 要確認：2026年度以降の継続有無を確認 |
| **小規模事業者持続化補助金** | 販促・業務改善ツール導入に活用（上限50〜200万円） |
| **各都道府県・市区町村の独自補助金** | IT導入に特化した地方版補助金が多数。競争率が低いものも |
| **日本政策金融公庫のDX融資** | 返済義務あり（補助金とは異なる）だが審査が比較的早い |
| **IT導入支援事業者の分割払いプラン** | 補助金申請しながら実質的な資金負担を分散 |

[情報源：One Step Beyond（ROI・費用対効果の考え方）](https://onestepbeyond.co.jp/blogs/) ／ [IT導入補助金公式サイト](https://it-shien.smrj.go.jp/)

## 📌 まとめ：採択後フェーズの実務チェックリスト

```
□ 交付決定通知を確認するまで発注・契約・支払いは行わない
□ 証憑書類（請求書・支払証憑・スクリーンショット）を事前に保管フォルダを作って整理
□ 支払証憑は「完了ステータス」のもののみ保存（ネットバンキングは特に注意）
□ 実績報告は補助事業者側から申請マイページで開始（支援事業者ではない）
□ 補助金額確定後のSMS承認を失念しない（承認しないと入金されない）
□ 入金まで最大5〜6か月の資金繰りを事前に手当て
□ 効果報告義務期間（通常枠3年）にITツールを解約しない
□ IT導入支援事業者からキックバック・ポイント還元の提案があれば即断る
□ 申請手続きは必ずgBizID保有者（自社担当者）が自ら操作する
```
