       ↓ 接触率 13.7%

**重要**：④以降はすべて推計値です。これは計測の仕組み（UTMパラメータ）が現状整備されていないためで、実態の数字は見えていません（後述 3-4）。

### 3-3. ボトルネックの特定

| ステップ | 課題度 | コメント |
|---|---|---|
| ① インプレ → ② 接触 | 低 | 業界上位水準。問題なし。 |
| ② 接触 → ③ プロフィール訪問 | 中〜高 | 動画内CTAの設計改善余地あり。 |
| ③ プロフィール訪問 → ④ EC流入 | 高 | 商品直リンクへの導線が最適化されていない。 |
| ④ EC流入 → ⑤ 購買 | 最高 | LP・商品訴求・計測のすべてに改善余地。 |

**結論**：ボトルネックは「視聴・接触」のあとに集中しており、コンテンツ（リーチ生成側）には問題が見当たりません。**コンテンツの方向性ではなく、視聴後の動線設計が課題**という整理になります。

### 3-4. なぜ売上が見えていないのか — 計測の欠落

**現状**

```
SNS視聴 → プロフィール → URLクリック → ECサイト訪問 → 購買
                              ↑
                     計測タグ（UTM）なし
                              ↓
        VITAS様のアクセス解析では「直接流入」「不明」に計上
```

SNS経由のアクセスに計測タグが付与されていないため、実際にSNS発の購買であっても、貴社のアクセス解析上は「直接流入」または「参照元不明」に分類されてしまっています。

**つまり、現在「SNS経由 ¥243万」と推計している数字は、計測できなかった分が抜け落ちた、実態の一部に過ぎません**。

**あるべき状態**

```
SNS視聴 → プロフィール → UTM付きURL → ECサイト訪問 → 購買
                              ↑
                     SNS起点と識別可能
                              ↓
       「YouTube経由の売上」「TikTok経由の売上」として計測可能
```

整備自体は技術的にシンプルで、ShapeFit側で設計はすでに完了しています。VITAS様側でURL書き換えの実装をご対応いただければ、5月から正確な計測が可能になります（ご対応工数：エンジニア1〜2時間程度）。

### 3-5. 診断のまとめ

1. **リーチ・認知は業界上位水準** → コンテンツの方向性は維持する。
2. **問題は「プロフィール訪問 → EC流入 → 購買」の動線** → 動線改修で大きな伸びしろがある。
3. **UTM未整備で売上が実態より小さく見えている** → 計測基盤の整備が施策の起点となる。

ここまでのご認識に違和感がございましたら、お聞かせください。問題なければ、具体的な施策のご説明に進みます。

## 5. KPI設計

### 5-1. 評価の考え方 — 責任領域で分離する

施策の評価指標を、ShapeFitがコントロール可能な範囲を踏まえて2層に分けてご提案いたします。

| 階層 | 内容 | 責任領域 |
|---|---|---|
| ShapeFitコミット指標（先行指標） | UTMカバレッジ、年間EC流入数 | SNS運用の範囲内でShapeFitがコントロール可能 |
| 共同観測指標（結果指標） | CVR、SNS経由売上、定期購入LTV | SNS×EC×商品の総合結果。LP・価格・商品力の影響を受ける |

**この分け方を採用する理由**

ShapeFitがコミットできるのは「サイトに人を連れてくるところまで」です。EC流入後の購買転換（CVR）は、サイト側のLP UX・価格設定・在庫状況・配送条件などの影響を強く受けるため、ShapeFit単独のKPIに置くと評価が公正でなくなる可能性があります。

「流入は伸びたのにCVRが伸びない」といった状況が起きた場合、原因をShapeFitに帰すのではなく、**両社で共同して原因分析と打ち手設計を行う運用** にするためのフレームです。

### 5-2. ShapeFitコミット指標（先行指標）

ShapeFitとして、以下の2点を年次でコミットいたします。

#### 指標1：UTM計測カバレッジ — 90%以上

- **対象**：VITAS様のPR投稿（VITAS製品を主題またはCTAとして明確に取り上げた投稿）
- **目標**：対象投稿のうち、UTMタグが正しく付与されている割合を90%以上に維持
- **評価タイミング**：月次
- **位置づけ**：計測基盤が運用上きちんと機能していることの担保

> なぜ90%か：理想は100%ですが、シェア経由のアクセスなど技術的に計測困難なケース、および稀な付与漏れリスクを考慮した保守的な水準です。

#### 指標2：年間EC流入数 — 80,000件以上

- **対象**：UTM計測されたSNS経由のECサイト流入（VITAS公式 / Amazon / TikTok Shop合計）
- **目標**：年間80,000件以上
- **評価タイミング**：月次進捗、年次評価
- **位置づけ**：SNSがもたらすトラフィックの絶対量

> なぜ年間80,000件か：施策後の試算値（年間100,957件）から、保守的に80%水準でコミットラインを設定しています。月次インプレッション比ではなく年間絶対値で置くのは、大会期などインプレッションが大きく変動する局面でも、安定して評価できるためです。

### 6-2. VITAS様にご協力いただきたいこと

#### 全体の流れ（前提のご共有）

UTMパラメータは、SNS投稿時にShapeFitが付与する「目印」のようなものです。VITAS様サイトに着地した際にGoogle Analytics（以下、GA4）が自動で読み取って記録するため、**VITAS様側で「UTMを処理する」ような特別な実装は不要** です。

ただし、計測を成立させるためには、VITAS様側で以下の環境が整っている必要があります。

```
SNS投稿（UTM付きURL）
   ↓ クリック
VITAS様サイト着地
   ↓
GA4が「UTM情報」と「サイト訪問」を自動記録    ← ②の権限が必要
   ↓
ユーザーが購買完了ページへ遷移
   ↓
GA4が「購買完了」をコンバージョンとして記録    ← ①の設定が必要
   ↓
ShapeFit／VITAS両社が「どのSNS投稿が何件の購買に繋がったか」を把握
```

#### ご対応いただきたい4項目

##### ① 【最重要】GA4で「購買完了」をコンバージョンとして計測する設定

**何をしていただくか**：購入完了ページ（「ご注文ありがとうございました」が表示されるページ）が表示されたタイミングを、GA4上で「コンバージョン」として登録していただきます。

**なぜ必要か**：UTMで「どこから来たか」はGA4が自動記録しますが、**「購入したかどうか」はGA4側で別途設定しないと記録されません**。この設定がないと、SNS経由の購買数が把握できません。

**ご対応工数**：エンジニア対応で1〜2時間程度。Shopify／ECCUBE等のプラットフォームをご利用の場合、標準機能で対応可能なケースが多くあります。仕様書はShapeFitから提供いたします。

**期限**：5月中旬目標（5/7週内に依頼書送付）

##### ② Google Analyticsアカウントの閲覧権限共有

**何をしていただくか**：ShapeFit指定のGoogleアカウント1つを、VITAS様のGA4プロパティに「閲覧者」として追加していただきます。

**なぜ必要か**：ShapeFit（実作業はまとばっち）が計測結果を確認し、月次レポート・ダッシュボードを作成するため。

**ご対応工数**：5分程度（GA4管理画面でメールアドレスを追加するのみ）。**閲覧権限のみ** で十分で、編集権限は不要です。

**期限**：5/7週内

##### ③ サイト内リダイレクト設定のご確認（技術担当者向け）

**何をしていただくか**：VITAS様のサイトでURL転送（リダイレクト）が発生する箇所において、**「クエリパラメータ（URL末尾の `?utm_source=...` の部分）が転送先まで保持される設定」** になっているかをご確認いただきます。

**なぜ必要か**：リダイレクト処理の設定によっては、転送の際にUTMパラメータが削除されてしまう場合があります。そうなるとSNS経由のアクセスとして識別できなくなり、計測が成立しません。

**ご対応工数**：確認のみであれば30分程度。修正が必要な場合でも、サーバー設定の軽微な調整で対応可能です。仕様書はShapeFitから提供いたします。

**期限**：5月中旬

##### ④ 定期購入データ・流入元データの共有

**何をしていただくか**：VITAS様の現在の定期購入実績および公式サイトへの流入元内訳を、集計データの形でご共有いただきます。

| データ | 用途 |
|---|---|
| 定期購入者数（月次） | LTV試算の精緻化 |
| 定期購入の継続率 | フェーズ3の施策設計 |
| 流入元内訳 | SNS経由売上の現状把握 |

**ご対応工数**：既存データのご提供のみ。**集計データで十分** で、個人特定可能な情報は不要です。

最低保証 ¥500,000/年 は **無条件** で適用されます。成果連動部分が年間¥50万を下回った場合でも、その差額が補填される形でVITAS様より頂戴いたします。

> 例：成果連動分が年間¥75万の場合 → ¥75万をそのまま頂戴（最低保証は発動せず）。

ShapeFit側の運用品質は、月次レポート、ダッシュボード共有、3ヶ月レビューの透明性で担保いたします。
