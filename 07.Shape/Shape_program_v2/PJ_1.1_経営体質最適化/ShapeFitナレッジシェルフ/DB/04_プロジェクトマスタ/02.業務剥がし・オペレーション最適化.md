# プロジェクト計画書：業務剥がし・オペレーション最適化

## 1. エグゼクティブサマリー
本プロジェクトは、紫音様が抱える経理・総務等の「非戦略業務」を組織的に切り離し、本来注力すべき「ビジョン・広報（天才の仕事）」へリソースを完全集中させるための緊急施策である。
現状、紫音様の工数が定型業務や確認作業に忙殺されており、事業成長のボトルネックとなっている。これを2.5ヶ月で解消する。
PMが主導して「業務の可視化・標準化・委譲」を完遂し、紫音様が意思決定とクリエイティブのみに没頭できる体制を構築する。

## 2. プロジェクト基本定義
* **正式プロジェクト名**: 業務剥がし・オペレーション最適化
* **プロジェクトオーナー**: 新harumasa suzuki
* **プロジェクトマネージャー**: 神保彩
* **期間**: 2.5ヶ月（10週間）

## 3. 現状の課題と構造分析 (Why Now?)
紫音様および組織が抱える以下の課題（Pains）を解消するため、本プロジェクトを遂行する。

* **直面している課題 (Pains)**:
    * **経理作業の属人化**: 本来紫音様が行うべきでない「入金チェック」等を本人が実施しており、戦略時間が削がれている。マニュアル不在が原因。（T001）
    * **業務仕分けの未実施**: 紫音様が「天才性」を発揮する業務以外にも忙殺されており、組織全体のエネルギー効率が低下している。（T023）
    * **リモート指示のコスト高**: 指示出し準備（資料整理）に時間がかかり、リモートスタッフを有効活用できていない。「依頼セット」が未整備。（T009）
    * **チャット待機ロス**: 返信待ちで作業が止まる頻度が高い。「急ぎのルール」や「権限範囲」が曖昧。（T017）
    * **依頼・受託の齟齬**: ゴール確認不足による手戻りが発生している。（T010）
    * **方針伝達の心理的負担**: 説明手順が決まっておらず、紫音様が過度なプレッシャーと工数を背負っている。（T004）

* **根本原因 (Root Cause)**:
    * **「暗黙知」への依存**: 業務プロセスが言語化されておらず、紫音様の頭の中にしかない（ブラックボックス化）。
    * **「渡す技術」の欠如**: 業務を他者に委ねるための標準フォーマット（依頼セット・判断基準）が存在しない。

## 4. 目指す姿と解決策 (To-Be & Solution)
* **プロジェクトの目的**:
    * 経理・総務・雑務を徹底的に標準化し、紫音様から他メンバーへの完全移譲を実現する。
    * 紫音様を「天才の仕事」に集中させ、事業成長ドライバーとしてのパフォーマンスを最大化する。

* **定量的ゴール (KGI/KPI)**:
    * **紫音様の定型業務工数**: 現状比 **80%削減**
    * **経理・総務タスクの委譲率**: **100%**（紫音様の手を離れること）
    * **リモートスタッフ稼働率**: 現状からの **2倍増**（待機時間ゼロへ）

## 5. 実行計画詳細 (Execution Roadmap)
「誰が」「いつ」「何をするか」を明確にし、PM（神保氏）主導で紫音様の負担を最小限に抑えつつ進行する。

### Phase 1: 棚卸しと緊急処置（Week 1 - Week 3）
**目的**: 現状の全業務を可視化し、即座に捨てられるムダを排除する。

* **Week 1: 業務仕分けワークショップ (T023)**
    * **【PM/紫音様】** 全業務をリストアップし、3つのバケツ（①天才/コア、②凡人/ノンコア、③廃棄）に即時分類する。
    * **【PM】** 「②凡人/ノンコア」業務の担当候補者（誰に渡すか）を仮決めする。
* **Week 2: コミュニケーションルールの即時導入 (T017, T010)**
    * **【PM】** 全社チャットルール（緊急時のメンションルール、「確認しました」スタンプの徹底等）を策定し、アナウンス。
    * **【PM】** 依頼テンプレート（背景・期限・完了定義・参考資料）のβ版を作成し、チームへ展開。
* **Week 3: 経理業務の録画・可視化 (T001)**
    * **【紫音様】** 実際の入金チェック・振込作業を行う際、Zoom/Loom等で画面操作を録画する（説明不要、作業するだけでOK）。
    * **【PM】** 録画データをもとに、手順書（ドラフト版）を作成する。※紫音様にドキュメント作成させない。

### Phase 2: 標準化と型化（Week 4 - Week 7）
**目的**: 「誰でも同じ品質で」実行できるツール（マニュアル・型）を完成させる。

* **Week 4: 経理マニュアルの完成とテスト (T001)**
    * **【PM】** 作成した手順書を見ながら、PMまたは別スタッフが実務を代行してみる（トライアル）。
    * **【紫音様】** トライアル結果の最終承認（ダブルチェック）のみ実施。不明点をPMが修正し、マニュアルを完成させる。
* **Week 5: リモート指示「依頼セット」の運用開始 (T009)**
    * **【PM】** リモートスタッフへの指示出しに必要な「フォルダ構成」「権限付与」「素材格納場所」をパッケージ化（＝依頼セット）。
    * **【PM】** 実際に1つのタスクを「依頼セット」を使って発注し、スムーズに動くか検証。
* **Week 6: 方針伝達フローの整備 (T004)**
    * **【PM】** 紫音様が方針を語る際の「定型アジェンダ（Why/What/How）」を作成。
    * **【PM】** 定例MTGのアジェンダ管理をPMが引き取り、紫音様は中身を話すだけにセットアップする。
* **Week 7: 中間レビュー**
    * **【All】** Phase 1, 2での削減時間を計測。進捗確認。

### Phase 3: 完全移譲と自走化（Week 8 - Week 10）
**目的**: 紫音様が実務から手を離し、最終承認のみのポジションへ移行する。

* **Week 8: 権限委譲の実行（シャドーイング終了）**
    * **【PM/Staff】** 経理・総務業務を実担当者が単独で遂行。
    * **【紫音様】** CC（共有）に入り、プロセスを見守るが、口出しは原則しない（緊急時除く）。
* **Week 9: 運用フローの微修正**
    * **【PM】** 発生したイレギュラー対応やQ&Aをマニュアルに追記。
    * **【紫音様】** 定型業務のチャットグループから退出、または通知OFF設定へ移行。
* **Week 10: プロジェクト完了確認**
    * **【PM】** 3ヶ月間の成果報告（削減時間、移行完了タスク一覧）を作成。
    * **【Owner】** プロジェクトクローズの承認。
