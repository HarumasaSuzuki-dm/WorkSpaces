### ステータス＝「資料の完成度 / 公開状態」（必ず1つだけ）

- `公開・納品`

**ステータスの決め方（原則）**
- タイトルにフラグがあればそれを優先する

  - `作成中 / WIP / 編集中` → `進行中`
  - `【納品用】/ 公開 / 確定版` → `公開・納品`（または `確定`）
  - `没：/ old / 旧 / アーカイブ` → `アーカイブ・没`
- 何も書かれていなければ、基本は `確定`（ただし明らかに途中なら `進行中`）

## 1. 何をしたい仕組みか（超かんたんに）

いまは、資料が **Google Drive / Canva / Notion** にバラバラにあります。
これを、まず **NotionのInbox（受け箱）** に全部集めて、あとから整理します。

- **作った人**：作ったらすぐ Inbox に入れる（登録する）
- **担当者（ドキュメント整理担当）**：定期的に Inbox を見て、タグとステータスを付けて整理する

これで「資料がどこ行った？」が無くなります。

---

## 2. ツールの使い分け（迷ったらこのルール）

### Notion（文章・メモ系）
- 議事録、方針、FAQ、台本など
→ **Notionページ**が本体

### Google Drive（ファイル系）
- Excel、PDF、契約書、提出物（完成版PDF）
→ **Driveファイル**が本体

### Canva（デザイン系）
- 画像、バナー、デザインの元データ
→ **Canvaリンク**が本体
※でも、外に出すPDFは **Driveに書き出して保存**（これ大事）

---

## 3. 入口ルール（作った人が必ずやること）

作ったら **5分以内** にこれだけやる。

### Inboxに入れるときの最低条件（これだけ）
- ドキュメント名（タイトル）
- URL（リンク）
- 保管場所（Drive / Canva / Notion）
- 作成者（名前）

※タグやステータスがまだ分からなくてもOK。とにかく入れる。

### やっちゃダメ（散逸する原因）
- チャットにリンク貼って終わり（Inboxに入れない）
- URLなしで登録（あとで探せない）

---

## 4. 整理ルール（ドキュメント整理担当が定期的にやること）

### 頻度（おすすめ）
- 週2回：火・金の15分
（忙しい週は“毎日10分”でもOK）

### 整理の手順（この順番でやる）
1. URLが開くか確認
2. 「本体はどこか」を決める（Drive / Canva / Notion）
3. **タグ（大）**を1つ付ける（領域）
4. **タグ（小）**を1〜2個付ける（用途）
5. **ステータス**を付ける（下書き/進行中/確定/公開・納品/アーカイブ・没）
6. 似た資料があるなら、どれが最新版か決める（古い方は `アーカイブ・没` にする）

---

## 5. ステータス（状態）の意味（簡単に）
- **下書き**：叩き台。まだ変わる前提
- **進行中**：作成・編集が続いている
- **確定**：チーム内で“これを使う”状態（完成）
- **公開・納品**：外部に出した/納品に使った（版管理の基準点）
- **アーカイブ・没**：旧版・不要・没。消さないが基本は参照しない
