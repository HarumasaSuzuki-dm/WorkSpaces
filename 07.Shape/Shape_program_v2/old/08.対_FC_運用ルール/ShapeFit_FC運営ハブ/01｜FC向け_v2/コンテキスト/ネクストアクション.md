## 4. 各作成物の定義

### D1. Must運用定義書

| 項目 | 内容 |
|------|------|
| **目的** | 4つのMustについて「誰が・いつ・何を・どうやる」を明文化し、迷わず実行できる状態を作る |
| **読み手** | FCオーナー、トレーナー |
| **決めるべき論点** | ①対象者 ②頻度・タイミング ③提出先・記録方法 ④「守れている」の判断基準 ⑤例外時のフロー |

### D2. Q&A集（初期版）

| 項目 | 内容 |
|------|------|
| **目的** | よくある疑問への回答を蓄積し、自己解決を促す |
| **読み手** | FCオーナー、トレーナー |
| **決めるべき論点** | ①初期版で用意すべき質問（各Must 2〜3問） ②更新ルール |

### D3. 例外申請ガイド

| 項目 | 内容 |
|------|------|
| **目的** | Mustを守れない時の申請手順と判断基準を明示する |
| **読み手** | FCオーナー |
| **決めるべき論点** | ①申請フロー ②伝えるべき情報 ③OK/NG/条件付きOKの基準 |

### D4. 週次1on1 運用ガイド

| 項目 | 内容 |
|------|------|
| **目的** | 週次1on1の進め方を標準化し、形骸化を防ぐ |
| **読み手** | なーすけ、事務局 |
| **決めるべき論点** | ①確認すべき観点 ②時間配分 ③課題の処理方法 |

### T1. セッションログ提出フォーム

| 項目 | 内容 |
|------|------|
| **目的** | 毎セッション後の記録を統一フォーマットで収集する |
| **入力者** | トレーナー |
| **決めるべき論点** | ①入力項目 ②「本音・弱みの開示」の判断基準 ③提出期限 |

### T2. NPS質問票

| 項目 | 内容 |
|------|------|
| **目的** | 顧客の「状態」を月次で測定する |
| **入力者** | 顧客 |
| **決めるべき論点** | ①質問項目 ②実施タイミング ③目標回収率 |

### T3. Must遵守状況 管理シート

| 項目 | 内容 |
|------|------|
| **目的** | 4つのMustの遵守状況を「事実（ログ）」で確認できるようにする |
| **利用者** | 事務局、オーナー |
| **決めるべき論点** | ①シート構成 ②測定指標 ③更新頻度・確認者 |

### T4. オーナー対話記録シート

| 項目 | 内容 |
|------|------|
| **目的** | オーナーとトレーナーの1on1実施を可視化する |
| **入力者** | FCオーナー |
| **決めるべき論点** | ①記録項目 ②記録の粒度 ③活用方法 |
