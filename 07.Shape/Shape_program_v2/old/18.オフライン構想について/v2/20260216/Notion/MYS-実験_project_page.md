## ❷ このプロジェクトについて

### ひとことで言うと

なーすけさんのコーチング判断基準を「他の人に渡せる形」にするために、実験的にコーチングセッションを実施し、判断の原型を取り出すプロジェクト。

### 背景

- Make Your Shapeの設計思想とオンラインの定義は固まったが、オフラインのコーチングで「なーすけさんが何を見て、どう判断しているか」はまだ言語化されていない
- 今後コーチを増やして全国に展開するには、この判断基準を取り出して渡せる形にすることが必要
- まず小さく実験して判断基準の原型を抽出し、そこからコーチ設計の土台を作る

### ゴール

- ✅ なーすけさんの判断基準が「思想（絶対に外せないこと）／型（流れとして決めること）／裁量（コーチ個人に任せること）」の3層に分類・言語化されている
- ✅ コーチ設計の土台ドキュメント（初版）が完成している
- ✅ レビュワーがコーチの判断を「Shapeかどうか」判定できる基準（初版）ができている

### スコープ

| やること | やらないこと |
|----------|------------|
| 3名へのコーチングセッションの実施・記録 | オンラインとの併用テスト（今回はオフライン単体のみ） |
| セッション後の振り返り・FBの蓄積 | コーチの育成・研修の実施（今回は「土台づくり」まで） |
| 判断基準の「思想／型／裁量」3層への分類・言語化 | サービスとしての有効性の検証（今回の目的は判断基準の抽出） |
| コーチ設計の土台ドキュメント（初版）の作成 | 収益モデルや価格の設計 |
| レビュワー判定基準（初版）の作成 | |

## ❹ フェーズと全体スケジュール

### フェーズ一覧

| フェーズ | 意味 | 完了の目安 |
|---------|------|-----------|
| ① 企画 | 何をやるか決める。方針・体制・実験設計を固める | プロジェクトページが完成し、全員が内容を理解した状態 |
| ② 準備 | 実行に必要なものを揃える。フォーマット作成・対象者確定・Notion構築 | 「日時・場所・相手・記録方法・事後FB体制・蓄積先」がすべて決まった状態 |
| ③ 実行 | セッションの実施・振り返り・FB・すり合わせを3人分繰り返す | 3人分のセッション・振り返り・FB・すり合わせがすべて完了し、Phase 1完了レポートが作成された状態 |
| ④ 振り返り | 蓄積した判断基準を整理し、コーチ設計の土台として言語化する | コーチ設計の土台ドキュメント（初版）とレビュワー判定基準（初版）が完成し、共有された状態 |

### スケジュール概要

```
🔵 ① 企画＋② 準備：2025/2月（〜2/26）
  └ 方針確定、体制確定、フォーマット作成、対象者確定、Notion構造セットアップ

③ 実行：2025/2/27〜4/30
  └ 1人目（2/27〜3/19）→ 2人目（3/20〜4/9）→ 3人目（4/10〜4/30）
    各サイクル：セッション2週 + FB完了・バッファ1週 ≒ 約3週間

④ 振り返り：2025/5/1〜5/31
  └ 素材の棚卸し → 3層の言語化 → コーチ設計ドキュメント（初版）完成
```

## ❺ 動き方のルール

### 定例・報告

| 項目 | 内容 |
|------|------|
| 定例ミーティング | 【要確認：キックオフMTGで決定予定。週1〜2回を目指す】 |
| 進捗報告 | 【要確認：キックオフMTGで決定予定】 |
| 緊急時の連絡先 | Chatworkで連絡する |

### タスクの進め方（共通ルール）

- 新しいタスクが発生したら、自分でDBに追加する。PMに言われる前に書く
- タスクが終わったら、自分でステータスを「完了」に変える
- 期限に間に合わないと分かった時点で、PMに相談する（期限を過ぎてから言わない）
- 判断に迷うことがあったら、体制図の「判断を仰ぐ人」に聞く

### プロジェクト固有のルール

- なーすけさんはセッション後、当日中に振り返りを記入する
- なーすけさんは動画を翌日までにアップロードする
- かおりさん・ぼんちゃんは動画共有後1週間以内にFBを返却する
- すり合わせMTGで出た仕分け結果は、ぼんちゃんがMTG当日中にNotionに記録する
- セッション②のFB後に「継続判断チェック」を行い、③を実施するか決める
