# 「タスク翻訳」運用設計書＆導入ロードマップ
**～「よしなに」を卒業し、手戻りゼロのタスク委譲を実現する～**

## 1. 目的とゴール
* **現状の課題：** 神保さんに「よしなに」任せると、期待値と異なる成果物が上がり、修正コスト（手戻り）が多発している。
* **解決策：** かおんさんの抽象的な要望を、作業可能なレベルまで**「翻訳（具体化）」**して渡すプロセスを挟む。
* **ゴール：** 神保さんが迷わず作業でき、一発でOKが出る状態を作ること。また、その「翻訳スキル」をかおんさんが習得すること。

---

## 2. 導入・習得ロードマップ（OJTプラン）

いきなりかおんさんに全てお任せするのではなく、まずは鈴木が「翻訳の型」をお見せするところから始めます。

### 【フェーズ1：モデリング期間】（直近数回～慣れるまで）
**鈴木が「翻訳の師範代」として、ロジックと言語化のプロセスを実演します。**

1.  **かおんさんからの素材提供**
    * 神保さんに依頼したいタスクが発生したら、以下のセットを鈴木に送ってください。
    * **(A) 画面収録動画（Loom/Zoom等）：** 実際の資料や参照画面をPCで投影しながら、「これをこうしてほしい」と口頭で説明してください。（整理されてなくてOK、思いつきで話して大丈夫です）
    * **(B) 関連資料データ：** 作業に必要なファイルやURLがあれば全て送付してください。

2.  **鈴木による「翻訳」と「解説」**
    * いただいた素材をもとに、鈴木が以下の2つを作成してお返しします。
    * **タスク定義書（ドラフト）：** 神保さんに渡せる状態にまで因数分解・言語化したテキスト。
    * **解説動画（かおんさん向け）：** 「なぜここまで細かくするのか？」「かおんさんの言葉をどう変換したか？」という**翻訳のロジック**を解説します。

3.  **かおんさんから神保さんへ伝達（重要）**
    * 鈴木の作成した定義書を参考に、**かおんさん自身の言葉**で神保さんに依頼してください。
    * ※鈴木の動画や定義書をそのまま転送するのはNGです。かおんさんが理解し、指揮をとる形を作るためです。

### 【フェーズ2：伴走期間】
**かおんさんが自分で「翻訳」を行ってみて、投げる前に鈴木がチェックします。**
1.  かおんさんがタスク定義案を作成。
2.  鈴木に「これで神保さんに伝わるか？」を確認出し。
3.  鈴木がOKまたは修正アドバイス → 神保さんへ依頼。

### 【フェーズ3：自走期間】
**かおんさんと神保さんだけで「15分翻訳MTG」を回せる状態。**
※判断に迷う難しい案件の時だけ、鈴木に相談する運用へ移行します。

---

## 3. 定常運用時の「タスク翻訳MTG」ルール（フェーズ3の姿）

自走フェーズに入ったら、以下のルールで運用します。

### 基本ルール
* **開催トリガー：** 1時間以上かかるタスクを依頼する際は、チャットで済ませず必ず開催。
* **所要時間：** **最大15分（延長禁止）**。伝わらない場合はタスクを分割する。
* **必須ツール：** 画面共有 ＋ タスク管理ツール（Notion等）。

### 進行フロー（15分1本勝負）
1.  **【翻訳（5分）】かおんさん → 神保さん**
    * 画面共有しながら、後述する「4要素」を伝達。
2.  **【チケット起票（5分）】神保さん**
    * **神保さんが画面共有**し、その場でタスクツールに入力。（かおんさんは認識ズレがないか確認）
3.  **【リバース・ブリーフィング（5分）】神保さん → かおんさん**
    * 入力した内容を見ながら、「つまり〇〇までに××をやれば良いですね？」と説明し返す。
    * 合意できれば着手スタート。

---

## 4. 翻訳すべき「タスク定義」の4要素（Definition）

フェーズ1〜3を通じて、常に以下の4点を埋めることを「完了条件」とします。

| 要素 | 項目 | 具体的な翻訳内容（例） |
| :--- | :--- | :--- |
| **WHAT** | **成果物イメージ** | ・最終アウトプットは何か？（PDF？ スプシ？ Notion？）<br>・分量は？（A4 1枚？ 10行程度？）<br>・**「完了」の状態は？**（誰かに送付したら完了？ 作成したら完了？） |
| **WHEN** | **期限とチェック** | ・最終納期はいつか？（日時まで指定）<br>・**中間チェックは必要か？**（「30%できたら一度見せて」等） |
| **HOW** | **手順とリソース** | ・参考にする過去資料やフォルダはどこか？<br>・分からない時に聞くべき相手は誰か？<br>・まず何から手を付けるべきか？（初動の指定） |
| **LEVEL** | **品質基準** | ・**完成度はどのくらい必要か？**<br>　Lv.1：たたき台・メモレベルでOK（スピード重視）<br>　Lv.2：誤字脱字チェック済みの提出レベル<br>　Lv.3：デザインまで整えた完全版 |

---

## 5. マインドセット
* **翻訳ミスは「指示側の責任」：** 成果物がズレていたら「神保さんの能力」ではなく「定義の抜け漏れ」を疑い、次回修正する。
* **質問は「着手前」に：** この翻訳プロセス（MTG中）は「バカな質問をしていい時間」です。着手後の「分かりませんでした」を無くすために、とことんすり合わせます。
