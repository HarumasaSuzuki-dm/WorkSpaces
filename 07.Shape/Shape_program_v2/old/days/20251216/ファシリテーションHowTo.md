# 【完全版】会議運用・ファシリテーション統一キット

**〜会議を「儀式」から「成果製造マシン」へ〜**

このドキュメントは、ファシリテーションの経験に関わらず、誰でも最短・最速で会議の目的を達成するための「型」と「進行マニュアル」をまとめたものです。

-----

## 1. 【基本思想】Google流・会議の鉄則（4つのルール）

会議を始める前に、参加者と以下のルールを握ってください。

1.  **ゴールなき会議は開催しない**
      * 「情報を共有する」はメールでOK。「何かを決める（合意する）」場合のみ開催する。
2.  **意見ではなく事実で話す**
      * 「なんとなく」禁止。「数字・データ・過去の事実」に基づいて判断する。
3.  **脱線は「パーキングロット」へ**
      * 本題と違う話が出たら、即座にメモに残して議論を止める。
4.  **「誰が・いつまでに」がない決定は無効**
      * アクション担当者が決まらない限り、会議は終わらない。

## 2. 【実践ツール】万能型ミーティング・キャンバス

これが **「会議中に埋めるべき共通フォーマット」** です。ホワイトボードや画面共有でこれを映し出し、空欄を埋める作業自体を会議の進行としてください。

### A. 事前設計

※主催者が会議前に必ず記入すること。これが埋まらない場合は開催キャンセルとする。

| 項目 | 記入・設計のポイント |
| :--- | :--- |
| **会議名** | 具体的かつ一目で内容がわかるものにする。 |
| **終了条件** | **「この会議室を出る時、何が決まっていたら成功か？」**<br>□ 抽象的な「相談」ではなく、「AかBか決まる」「担当者が決まる」と定義する。<br>□ 終了条件を満たしたら、時間が余っていても即終了する。 |
| **意思決定者** | **「議論が割れた時に、最後に決める人」**<br>※この人が不在の会議は無意味なのでリスケする。 |
| **事前資料** | **「これを読んでいないと議論に参加できない資料」**<br>※未読者がいる場合、会議冒頭5分を「沈黙の読書時間」にする。 |

### B. 進行・決定ログ（Real-time Log）

※会議中にリアルタイムで記入する。

| 決定事項 | その根拠・データ |
| :--- | :--- |
| **【重要】ここで何が決まったか？**<br>（例：A案を採用する） | **なぜそう決めたか？**<br>（例：過去データでA案の方がCVRが5%高いため） |
| | |
| | |

### C. パーキングロット

※本題（終了条件）に関係ない話が出たら、ここに入れて議論を打ち切る。

  * （例：来月のイベントの話、ツールの不具合の話など）
  *   * ### D. アクションプラン（WWW: Who/What/When）

## 4. 【司会用台本】初心者向けファシリテーション・スクリプト

この通りに読み上げるだけで、プロフェッショナルな進行が可能です。

### ① 開始時（ゴール設定）

> 「お疲れ様です。本日の会議のゴール（終了条件）は、**『〇〇を決めること』**です。
> 時間内にこのシートのアクションプランまで埋め切ることを目指します。
> 目的以外の話はパーキングロットへ逃がしますので、ご協力お願いします。」

### ② 議論中（脱線の防止・ファクトの要求）

> **（話が逸れた時）**
> 「すみません、その話は本日のゴールである『〇〇の決定』に必須でしょうか？
> もし別の論点であれば、**パーキングロットに入れておき、後日扱いましょう。**」

> **（感覚的な意見が出た時）**
> 「なるほど。それは個人の感想でしょうか、それとも何か**データや過去の事実はありますか？**
> 決定事項の『根拠』欄に書きたいので、ファクトベースでお願いします。」

### ③ 終了時（コミットメント確認）

> **（アクションの確認）**
> 「では決定事項に基づき、次のアクションを確認します。
> 佐藤さん、このタスクは**いつまでに**完了できますか？ 金曜日の12時でいいですか？
> （記入しながら）はい、書きました。これで決定です。」

> **（解散の言葉）**
> 「終了条件を達成し、アクションも決まりましたので、会議を終了します。
> 決定事項には全員がコミットして進めていきましょう。お疲れ様でした！」

## 5. フレームワーク活用のHow To & ポイント

このシートを機能させるための「裏側のポイント」です。

### Q1. アジェンダはどう作ればいい？

  * **MECE（漏れなくダブりなく）を意識する**
    PDFの例のように、フェーズを分けるのがコツです。「Phase 1：調査」「Phase 2：検証」「Phase 3：運用」 のように時系列で切ると、今何を議論すべきかが明確になります。
  * **問い（Question）の形にする**
    アジェンダの項目は「〇〇について」ではなく、「〇〇はAかBか？」や「〇〇の担当は誰にするか？」という**問いの形**にすると、答えが出やすくなります。

### Q2. 議論が膠着（こうちゃく）したら？

  * **「決定者」に振る**
    議論が平行線になったら、ファシリテーターが悩む必要はありません。「意見が出尽くしましたね。では、**意思決定者の〇〇さん、AとBどちらにしますか？**」とボールを渡してください。

### Q3. 何も決まらなかったら？

  * **「決められなかった理由」を成果にする**
    時間切れで決まらなかった場合、決定事項欄には**「情報不足のため持ち越し」**と書き、アクション欄に**「誰が・いつまでに・その不足情報を集めてくるか」**を書いてください。これだけでも一歩前進です。
