# Shape Fitの勝ち方：正解ではなく“言語化”を起こす v9.1

※FC/現場/本部で“同じ判断”をするための共通ルール（理念＋約束＋判断軸＋最小測り方＋例）

---

## 0) この1枚の目的（FCが“楽になる理由”）

この1枚は、**評価と是正を「感覚」から切り離し、事実（ログ）で揃える** ためのルールです。
その結果、FCオーナーは次がラクになります。

- **育成がラク**：新人/若手に「何を目指せば合格か」が明確（“開示”で教えられる）
- **品質が揃う**：店舗/担当者で判断がブレにくい（揉めの種が減る）
- **クレーム対応がラク**：感情論でなく「何が起きたか」を引用・弱みで説明できる
- **本部との摩擦が減る**：口出しではなく、基準（ログ）で会話できる
- **採用がラク**：理念に合う人材（言語化を起こせる人）を面談で見抜ける

---

## 1) 理念（1行定義）

株式会社Shape Fitは、トレーナーが顧客（自分自身）に“言語化”を起こすことで、顧客が「本音で迎えたい未来」へ挑戦できる状態をつくる会社。

---

## 2) 判断軸（迷ったらここ）

**良い指導＝セッション中に、顧客が「自己開示＋弱みの開示」までできている状態をつくれたか。**

- 弱みの例：できていない行動／できなかった事実／他人には当たり前でも自分には難しい行動／本当はやりたいが怖くて言えなかったこと。
- **未達の定義**：開示（特に弱み）が出ていないなら、その指導は未達。
- 影響力・知識量・雰囲気・トレーナーの思い“だけ”では合否を決めない（合否は開示で決める）。

### 合格の最低ライン（これだけ揃えばOK）

**「顧客の言葉（引用）＋弱み（できなかった/怖い/避けてた）＋次回の一歩（顧客が選ぶ）」** が揃っている。

---

## 3) 境界線（やらないこと／誤魔化し防止）

- 雑談・モチベだけ・筋トレだけで終わる指導を “Shapeの本当の価値” とは呼ばない。
- **引用NG**：トレーナー解釈を“顧客の言葉”として書くのは禁止（評価が主観に戻るため）。

**良い引用 / ダメな引用（例）**

- ✅ 良い：「また失敗して自分を嫌いになるのが怖い」「見られるのが怖くて行けなくなる」
- ❌ ダメ：「やる気が出た」「前向きになった」「意識が変わった」（＝解釈）

---

## 4) 約束（Promise：Shapeである最低ライン）

1. **評価の約束**：合否基準は「顧客の開示」。開示がなければ未達。
2. **自由の約束**：やり方は縛らない。だが「開示が出ていない」を自由で正当化しない。
3. **是正の約束**：未達は放置しない。罰ではなく、振り返り→次回の改善点を1つ決める。
4. **責任の約束**：質の最終責任は店舗責任者（将来：コーチ）。本部は口出しより「基準と仕組み」を提供する。
5. **境界線の約束**：開示に関わらない指導（雑談・モチベだけ・筋トレだけ等）を“Shapeの本当の価値”とは呼ばない。
6. **成長の約束**：トレーナーは正解を与える人ではなく、言語化を起こす人。顧客が決める余白を残す。

---

## 5) 測り方（仕組み：迷わない・不公平にしない）

> 方針：評価を“感覚”から切り離し、事実（アウトプット） で揃える。
> 

### 仕組み①：開示の可視化（セッション後アウトプットを必ず残す）

- **頻度**：毎セッション
- **所要**：2〜3分
- **必須（この3点で十分）**
    1. 顧客の言葉（引用）
    2. 出た弱み（できなかった／怖い／避けてた）
    3. 次回の一歩（顧客が“自分で選んだ”行動）

**記録ルール（安全性）**

- 個人情報（氏名・勤務先・住所・家族の詳細など）は書かない。
- “本音の要点”だけを引用する（短く）。
- **アウトプット例**
    
    ```markdown
    【セッションログ（開示の可視化）】
    日付：2025-12-30 / 店舗：渋谷 / 担当：田中 / 顧客：Aさん（30代女性・会社員）
    
    今回のテーマ（顧客の持ち込み）：
    - 「飲み会が増えて、また失敗しそうで怖い」
    
    1) 顧客の言葉（引用：本音）
    - 「痩せたいより、“また失敗して自分を嫌いになる”のが怖いです」
    - 「できない自分を見られるのが嫌で、外に出たくなくなる」
    
    2) 弱みの開示（できなかった／避けてた／怖かった）
    - 夜：不安が来ると“先に食べて落ち着かせる”→食後に自己嫌悪
    - 避けてたこと：体重計に乗る（現実を見るのが怖い）
    - 言えなかったこと：家族の前で食事を整えるのが気まずい（流されやすい）
    
    3) 本音で迎えたい未来（言語化：短く）
    - 「外食があっても、自分を責めずに戻せる人になりたい」
    
    4) 次回の一歩（顧客が自分で選んだ行動：1つ）
    - 夜に不安が来たら「5分散歩→それでも食べたいなら“選んで食べる”」にする
    
    5) 次回の確認（合否の根拠：1つ）
    - 「怖い／避けたい／本当はこうしたい」が顧客の言葉（引用）で1文以上出ているか
    
    ```
    

---

### 仕組み②：未達の振り返り強制（未達を放置しない）

- **トリガー（未達）**：仕組み①の「引用（顧客の言葉）」と「弱み」が取れていない
- **頻度**：未達のたび
- **所要**：2〜3分（※“長文反省文”は禁止。短く次へ）
- **必須**：仮説→次回変えること（1つ）→次回確認（1つ）
- **アウトプット例**
    
    ```markdown
    【未達振り返り（強制）】
    日付：2025-12-30 / 店舗：渋谷 / 担当：田中 / 顧客：Bさん（40代男性）
    
    未達判定（事実）：
    - 顧客の言葉（引用）が取れなかった
    - 弱み（できなかった／怖い／避けてた）の開示が出なかった
    - 次回の一歩が「頑張ります」で終わった（顧客が“選ぶ”形になっていない）
    
    1) 仮説（なぜ開示が出なかった？：短文で2つまで）
    - 助言を急いだ（質問→沈黙→待つ ができず、すぐ提案に入った）
    - 顧客が「正解（メニュー）を求めるモード」なのに、安心の確認が足りなかった
    
    2) 次回どう変える？（行動1つだけ：具体）
    - 冒頭3分でこの順番：
      ①「今日いちばん困ってることは？」（現状）
      ②「それが続くと何がいちばん怖い？」（恐れ）
      ③「本当はどうなりたい？」（未来）
      ※質問の後は“10秒沈黙”を守る（埋めない）
    
    3) 次回の確認観点（1つ）
    - 「怖い／避けたい／本当はこうしたい」のどれかが顧客の言葉で1文以上出ている（引用で残す）
    
    ```

### 仕組み③：評価の階層化（対立を消す）

- **本部はトレーナーを直接評価しない**
- **本部→店舗責任者（コーチ）を評価**
- **店舗責任者（コーチ）→トレーナーを評価**
- **評価軸は「開示できているか」一点**（事実ベースで進め、管理されてる感を減らす）
- **アウトプット例**
    
    ```markdown
    【コーチ評価ログ（トレーナー）】対象：2025-12 / 店舗：渋谷
    評価者（コーチ）：佐藤 / 被評価者（トレーナー）：田中
    
    1) 事実指標（開示に紐づくものだけ）
    - セッションログ提出率：28/30（93%）
    - 引用1文が“顧客の言葉”として残っている率：23/30（77%）
    - 弱みの開示（できなかった／怖い／避けてた）が書かれている率：18/30（60%）
    - 未達振り返り実施率：6/6（100%）
    - 未達→次回の一歩が“顧客が選んだ形”になっている率：4/6（67%）
    
    2) 良かった例（ログから1つ：貼る）
    - 「また失敗して自分を嫌いになるのが怖い」→ 次回の一歩「5分散歩→選んで食べる」
      ※“正解提示”ではなく“選択を作れている”
    
    3) 次月の約束（改善：1つだけ）
    - 毎回1回は「怖い／避けたい」を聞く質問を入れ、引用で残す（弱み率60%→75%）
    
    ```
    
    ```markdown
    【本部→コーチ評価（軽量監査メモ）】2025-12-30
    対象店舗：渋谷 / 対象コーチ：佐藤 / 監査者（本部）：鈴木
    
    監査方法（軽量）
    - ランダム抽出：セッションログ5件＋未達振り返り2件
    - 見るのは“開示の証跡”だけ（結果数字・指導スタイルには踏み込まない）
    
    結果（事実）
    - 5件中OK：4 / NG：1
      - NG理由：引用が「やる気が出た」など“トレーナー解釈”で、顧客の言葉になっていない
    
    是正（次回まで：小さく1つ）
    - コーチが週次でログ3件だけレビューし、「引用の取り方」を1つフィードバックする
    
    ```

## 6) 困った時の使い方

- **採用**：面談で「開示（弱み）を引き出す会話ができるか」を見る（正解を語る人より、問いが立てられる人）
- **育成**：未達を“叱る”のではなく、仕組み②で「改善の型」に乗せる
- **品質**：仕組み③（監査）で店舗差を早期発見（揉める前に潰す）
- **クレーム**：ログの引用・弱み・一歩を見れば「何が起きて、どう立て直すか」で会話できる

---

## 7) 迷ったら（超短い判断ルール）

- **顧客の“本音の言語化”が起きたか？**
- **弱みの開示が出たか？出てないなら未達。**
- **未達なら、罰ではなく振り返り→次回改善。**

---

### 使い方（運用メモ：最小）

- セッション後：仕組み①（2〜3分）
- 未達のときだけ：仕組み②（2〜3分）
- 月1回：仕組み③（コーチ5分／本部監査10分）
