# インタビュー設計書：紫音様 (SFF/Ops)

## 1. インタビューの目的 (Objectives)

1.  **「感覚」から「数値」への翻訳**: 紫音様が現在「肌感覚」で行っている意思決定（価格、開催判断）を、鈴木が分解し、ダッシュボード化するための**「不安の正体」**を特定する。
2.  **デリゲーションの阻害要因の特定**: なぜ神保様や町田様といった外部パートナーに業務を渡せないのか、その「具体的障壁（ドキュメント不足？信頼？複雑性？）」を明らかにする。
3.  **「翻訳機能」の実態把握**: なーすけCEOのビジョンが現場に降りてくる際、紫音様がどのように防波堤（翻訳機）となっているか、その負荷を確認する。

## 2. タイムラインと構成 (60min)

  * **00-05分**: アイスブレイク・趣旨説明（鈴木より：「紫音さんが『一番得意なこと』に集中できるよう、雑音（迷い・作業）を取り除くためのヒアリングです」）
  * **05-20分**: 【Being】役割定義と誇り（演出家としてのコア）
  * **20-35分**: 【Logic】意思決定の苦悩と「欲しい数字」（ガバナンス）
  * **35-50分**: 【Style】業務の抱え込みと「手放し方」（R\&R/オペレーション）
  * **50-60分**: クロージング（鈴木からのコミットメント提示）

## 3. 質問リスト詳細 (Interview Script)

### Part 1: 【Being】役割定義とミッション

**狙い:** 紫音様が死守したい「SFFの魂」を言語化し、効率化してはいけない聖域を決める。

| No. | 質問項目 (Question) | 深掘り・意図 (Intent & Deep Dive) |
| :--- | :--- | :--- |
| **Q1** | **「紫音さんの仕事を、社内の誰もがイメージできる『職業名』で例えるなら何ですか？」**<br>（例：映画監督、舞台プロデューサー、現場監督、母親…） | **[自己認識の確認]**<br>・資料では「演出・現場統括」。<br>・「監督（決める人）」なのか「現場（走る人）」なのか、どちらに重心があるか確認。 |
| **Q2** | **「直近のSFFや業務で、一番『私、いい仕事したな』とガッツポーズした瞬間はいつですか？」** | **[成果の定義]**<br>・「参加者が泣いた時（感情）」か「トラブルなく終わった時（管理）」か。<br>・ここが紫音様のモチベーションの源泉。 |
| **Q3** | **「逆に、仕事をしていて一番『心が削られる』『無力感を感じる』のはどんな時ですか？」** | **[ストレス要因の特定]**<br>・「なーすけさんの無茶振りへの対応」なのか、「決められない自分への苛立ち」なのか。<br>・課題の真因（Root Cause）に近づく問い。 |

### Part 2: 【Logic】意思決定と「数字」への不安

**狙い:** **鈴木が支援すべき「ダッシュボード（判断基準）」の要件**を引き出す最重要パート。

| No. | 質問項目 (Question) | 深掘り・意図 (Intent & Deep Dive) |
| :--- | :--- | :--- |
| **Q4** | **「イベントの価格設定や開催可否を決める際、今の手元情報だけで判断することに、正直どれくらいの『怖さ』がありますか？（0-100%）」** | **[意思決定の不全]**<br>・「感覚での判断に限界」を感じているという仮説の検証。<br>・その怖さは「赤字になる怖さ」なのか「客が来ない怖さ」なのか。 |
| **Q5** | **「もし目の前に魔法の画面があって、『この数字』さえ表示されていれば即決でGOサインが出せる、というデータは何ですか？」** | **[KPIの要件定義]**<br>・PLの利益率？ 集客見込み数？ キャッシュ残高？<br>・鈴木が作成する「SFF事業収支シミュレーション」 に直結する答え。 |
| **Q6** | **「なーすけさんから新しいアイデア（Vision）が降ってきた時、現場責任者として『一番困ること（ハレーション）』は何ですか？」** | **[翻訳コストの可視化]**<br>・「スケジュールの組み直し」か「スタッフへの説明」か。<br>・この負担を減らすための会議体やルールが必要。 |

### Part 3: 【Style】業務の抱え込みと権限委譲

**狙い:** 「自分でやった方が早い」病の治療法（パッケージ化）を探る。

| No. | 質問項目 (Question) | 深掘り・意図 (Intent & Deep Dive) |
| :--- | :--- | :--- |
| **Q7** | **「正直、『これ、私の時給でやる仕事じゃないな』と思いながら手を動かしている作業は何ですか？」** | **[業務の棚卸し]**<br>・事務連絡、シフト調整、入金確認など具体的に挙げてもらう。<br>・それが「神保さん」や「香音さん」に渡っていない業務。 |
| **Q8** | **「上記（Q7）の業務を、神保さんや町田さんに『丸投げ』できていない理由は何ですか？」**<br>（A: 説明が面倒、B: 彼らのスキル不足、C: クオリティへの不安、D: そもそも渡す発想がなかった） | **[委譲のブロッカー特定]**<br>・回答によって解決策が変わる。<br>・Aならマニュアル化（鈴木担当）。<br>・Cなら品質基準の言語化。 |
| **Q9** | **「他の人がやると1日かかるけど、紫音さんなら一瞬で（感覚的に）さばける『得意技』は何ですか？」** | **[不当な優位性 (Unfair Advantage)]**<br>・これだけは手放してはいけない業務。<br>・例：現場の空気読み、トラブル時の瞬時の判断、香音さんへの根回し。 |
| **Q10** | **「半年後、ご自身が事務作業ゼロで『戦略と演出』だけに集中できるとしたら、まず何に着手したいですか？」** | **[未来への接続]**<br>・B2B展開？ 新規コンテンツ開発？<br>・鈴木が作るロードマップの「ゴール」として設定する。 |

## 4. インタビュアー（鈴木）へのメモ

  * **スタンスの取り方**:
      * 鈴木は「監査役」ではなく**「武器商人」**として接してください。「紫音さんの感性を守るために、論理という防具を渡しに来ました」という姿勢が重要です。
  * **確認すべき「表情」**:
      * **Q4（数字の怖さ）**: ここで「ホッとした顔」をするか「拒絶反応」を示すか。
          * **ホッとした場合**: 数字を求めています。早期にPLを見せましょう。
          * **拒絶反応の場合**: 数字＝管理・監視だと思っています。「数字は自由になるための道具です」と再定義（リフレーミング）が必要です。
  * **深掘りのコツ**:
      * 「自分でやった方が早い」と言われたら、**「それを3回繰り返すと、マニュアルを作るより時間がかかりますが、それでも自分でやりますか？」**と優しく問いかけてみてください。これが業務委譲（Delegation）への気づきになります。
