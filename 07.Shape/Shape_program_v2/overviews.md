# 【最終確定版】プロジェクト支援方針：理念と実務の「接続」と「整流」

「理念先行型COO」である中井様の想いや熱量を、鈴木様が「論理・数値・構造」で補完し、組織全体が迷わず走れる状態を作ります。特に**「SFF事業」「コミュニティ」「法人研修」**という個別の事業が、全体としてどう繋がり、どの財布（PL）で回るのかを可視化します。

## 1. 支援の中核テーマ：「点」を「線」にする

これまでの課題（PL、組織、IT、ロードマップ）を個別のタスクとして処理するのではなく、全てが**「どう紐づいて、どのような優先順位で動くべきか」**という全体像の接続を行います。

*   **Vision Logic（ビジョンと数字の接続）**
    *   **課題:** なーすけCEOの掲げる「10年後のビジョン（理想）」と現場の「3年後の現実」の間に乖離がある。特にSFF事業の価格設定や投資判断において、**「PL（損益計算書）」という判断基準が機能しておらず**、感覚的な意思決定になっている。
    *   **支援:** 鈴木様がPL作成・運用を巻き取り、「これをやれば達成できる」と信じられる根拠（数字・リソース計算）を付与したロードマップを作成する。
*   **Mental Sorting（思考の棚卸しと選別）**
    *   **課題:** 目の前のミクロな課題（誰がタスクを拾うか）から未来の話（2026年の展開）までが混在し、何に集中すれば「大丈夫」なのかが見えていない。
    *   **支援:** 中井様が抱えるモヤモヤを全て吐き出す場を設け、「これは今やるべき」「これは一旦放置でOK」という判断・交通整理を行う。

### 【Update】最新の重点論点（4つの争点）
最新のミーティング（sff.md）で合意された、組織として機能するために合意すべき4つの論点です。本支援ではこれらを優先的に解決します。

1.  **意思決定の「数値基準」と「サイクル」**
    *   いつ、誰が、どの数字を見て意思決定を下すのか（ダッシュボードと会議体のセット運用）。
2.  **組織の「境界線」と「権限委譲」**
    *   リーダー層が実務に埋没しないよう、業務委託への「委譲ライン」を引き、パッケージ化して渡す。
3.  **ビジョンと現場をつなぐ「翻訳機能」**
    *   「高い理想」を「直近のアクション」に翻訳し、現場のハレーションを防ぐ。
4.  **会議体（MTG）と情報流通の再設計**
    *   「ビジョン浸透」と「係数管理」を機能させるための会議アジェンダと頻度を設定する。

## 2. ロードマップの「正体」と構成要素

クライアントがロードマップ作成において「何が必要か不明確」と感じている現状を踏まえ、全員が腹落ちして推進するための「ロードマップの定義」を以下のように設定します。ロードマップとは単なる日程表ではなく、**「完了条件（成果物）の合意」**です。

| 構成要素 | 定義（Why） | 問い（Check） |
| :--- | :--- | :--- |
| **1. 成果物**<br>(Deliverables) | 各フェーズで**「何ができあがればゴール（完了）とするか」**という具体的なアウトプット。これがないと「やったつもり」で終わる。 | 「これさえあれば、次の判断ができるか？」 |
| **2. 依存関係**<br>(Logic) | **「Aが終わらないとBは始められない」**という論理的な順序。ここを無視すると手戻りやボトルネックが発生する。 | 「なぜ、今これをやる必要があるのか？」 |
| **3. マイルストーン**<br>(Checkpoints) | 意思決定を行い、次のフェーズに進むための**「関所」**。ここで承認（Go/No-Go）を行う。 | 「いつまでに、誰が承認するのか？」 |
| **4. リソース**<br>(Resources) | 実行するために必要な**「人の稼働」**と**「資金」**。 | 「絵に描いた餅になっていないか？」 |

**★重要な合意事項:**
スケジュール（日付）はあくまで結果であり、最も重要なのは**「必要な成果物が、正しい順序で定義されているか」**です。

| | **アクション** | モヤモヤ（不安・課題・構想）を全てリストアップし、吐き出す。 | 吐き出された内容を「課題のリスト化」と「優先順位付け」を行い、解決策（ロードマップ・組織図）に落とし込む。<br>**※PL作成・IT調査の実務を巻き取る。** |
| | **ゴール** | メンバーへ想いを伝え、チームを鼓舞する。 | 「この通り進めば大丈夫」という安心感と論理的裏付けを提供する。 |

## 5. 現状の構造的課題と解決の方向性

音声ミーティングで顕在化した具体的な「詰まり」に対し、以下のアプローチを取ります。

*   **組織マネジメントの不全（タスクの滞留）**
    *   **課題:** 香音（Kaon）様が責任感から抱え込み、自信を喪失しやすい。的場（Matoba）様（マーケ）との連携が受動的。香織（Kaori）様のリソース配分（事務偏重）への懸念。神保（Jimbor）様・町田（Machida）様へのタスク委譲不全。
    *   **対策:** 「役割定義」と「権限委譲のライン」を明確化した組織図を再設計する。特に香織様については、強みである**コーチング領域（法人研修・社内ケア）へのシフト**を検討する。
*   **IT/CRM導入の順序（早期着手へ変更）**
    *   **課題:** コミュニティからジム・SFFへの顧客導線を管理する基盤がなく、機会損失や管理コスト増が起きている。
    *   **対策:** 実装（契約・設定）自体は業務フロー整理後だが、**「どのツールで何を管理するか（要件定義・調査）」はPhase 1より前倒しで開始**し、全体設計に組み込む。

## 6. 修正後のネクストアクション（進行ステップ）

前回設定した「Step 0」を、より具体的な**「デトックス＆構造化ミーティング」**へと昇華させます。

**Step 0：不安の棚卸しと接続ミーティング（最重要）**
*   **準備（中井様）:** ご自身の中で起きているモヤモヤ、課題、つながりが見えない部分を「リスト化・書き出し」していただきます。
*   **実施内容（鈴木様 - ヒアリングの観点）:**
    1.  **全出し:** 「SFFの価格設定の迷い」「香音様や的場様との連携の悩み」「PL運用の実態」など、固有名詞ベースで吐き出していただく。
    2.  **仕分け:** 「今解決すべきこと」と「後回し・無視して良いこと」をその場でジャッジする（**成果物B-3**の作成）。
    3.  **接続:** バラバラに見える課題が、全体戦略の中でどう位置づくかを解説・構造化する（**成果物A-4**の骨子作成）。

**Step 1：根拠あるロードマップと組織図の策定**
*   鈴木様が持ち帰り作業で**「数字的根拠（PL）」「実行体制（組織図）」「時間軸（ロードマップ）」**が整合した計画書を作成します。
*   **SFF事業のマイルストーン:** 集客、価格決定、開催時期の決定。
*   **IT/CRM調査（前倒し）:** コミュニティ〜ジム〜SFFを繋ぐ顧客管理要件の洗い出しを開始。

**Step 2：IT/CRM（Salesforce）の実装設計**
*   全体戦略（ロードマップ）と要件定義に基づき、具体的なツールの設定・導入を進めます。
