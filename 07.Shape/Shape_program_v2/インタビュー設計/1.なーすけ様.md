# インタビュー設計書：なーすけ様 (CEO)

## 1. インタビューの目的 (Objectives)

1.  **「感覚」の言語化**: なーすけ様の中にしかない「GO/NO-GO」の判断基準（美学・品質ライン）を言葉にし、他者が扱える「基準」にする。
2.  **ボトルネックの真因特定**: 権限委譲が進まない本当の理由が、「現場のスキル不足」なのか「なーすけ様の心理的不安（任せることへの恐怖）」なのかを切り分ける。
3.  **CEOのOS解析**: なーすけ様が最もパフォーマンスを発揮する条件（勝ちパターン）と、組織を混乱させる条件（負けパターン）を特定し、周囲の関わり方を定義する。

## 3. 質問リスト詳細 (Interview Script)

### Part 1: 【Being】役割定義とミッション

**狙い:** 「なーすけ」という個人のリソースをどこに全振りすべきか、**「やらないこと」**を決めるための問い。

| No. | 質問項目 (Question) | 深掘り・意図 (Intent & Deep Dive) |
| :--- | :--- | :--- |
| **Q1** | **「もし明日、なーすけさんがこのプロジェクト（Shape）から完全に抜けたら、最初に崩壊・停止する機能は何ですか？」** | **[代替不可能性の特定]**<br>・答えが「ビジョン」なのか「営業」なのか「コンテンツ」なのか。<br>・逆に挙がらなかったものは、能力的に手放せる業務。 |
| **Q2** | **「能力的には他人に任せられるとしても、『情熱』として絶対に手放したくない業務（聖域）は何ですか？」**<br>（例：YouTubeの企画、SFFのMC、0→1のアイデア出し等） | **[聖域の定義 (Sanctuary)]**<br>・「これは俺がやらないと死ぬ（つまらない）」という業務を特定。<br>・逆に挙がらなかった業務は全て「権限委譲（RACI）」の対象として組織図に落とし込む。 |
| **Q3** | **「ご自身の役割を、社内の誰もが知っている『職業・役割名』で例えるなら何ですか？」**<br>（例：建築家、伝道師、特攻隊長、演出家…） | **[セルフイメージの確認]**<br>・「文化発信」 と自己認識のズレを確認。<br>・「経営者」という言葉を使わずに答えてもらうのがポイント。 |
| **Q4** | **「3年後、なーすけさんは何をしている時が一番幸せですか？逆に、何をしている自分は『失敗』だと思いますか？」** | **[理想の配置図]**<br>・To-Be（トップセールスやメディア露出に専念など）と、As-Is（細かい管理業務）のギャップを明確にする。 |
| **Q5** | **「3年後（2028年）、会社の『稼ぎ頭（メインエンジン）』はどこにあるイメージですか？」**<br>A：法人研修（B2Bシフト）<br>B：SFF等のイベント事業（興行収益）<br>C：ジム店舗展開（安定収益） | **[事業ポートフォリオの定義]**<br>・ロードマップ作成の最重要分岐点。<br>・現在の延長線上ではなく、「理想の到達点」としてPL構造をどう設計すべきか（B2B型かB2C型か）を決定する。 |

### Part 2: 【Logic】意思決定と権限委譲

**狙い:** **「なぜ任せられないのか」**の構造的要因（Criteriaの欠如）を暴く。

| No. | 質問項目 (Question) | 深掘り・意図 (Intent & Deep Dive) |
| :--- | :--- | :--- |
| **Q6** | **「現場（紫音さんや香音さん）に仕事を任せるとき、一番『怖い』と感じることは何ですか？具体的にどんな失敗を恐れていますか？」** | **[不安の正体]**<br>・品質低下？顧客からのクレーム？理念の形骸化？<br>・「SFFの演出」や「法人研修の質」 における**「絶対に譲れない品質ライン（撤退ライン）」**を言語化させる。 |
| **Q7** | **「SFF（イベント事業）の『成功定義』はどちらですか？」**<br>A：利益重視（チケット・スポンサーでしっかり黒字化）<br>B：広告宣伝（トントンor赤字でもOK、認知・熱狂優先） | **[KPI/PL設計の基準]**<br>・値付けや予算配分の「松竹梅」を決める基準。<br>・事業単体で自走させるのか、マーケティングコストと割り切るのか。 |
| **Q8** | **「今、現場から上がってくる報告で『判断できない（情報が足りない）』とイライラする瞬間はいつですか？」** | **[判断基準の欠如]**<br>・PL（数字）がないから決められないのか、定性情報（顧客の温度感）がないからなのか。<br>・「この数字さえあれば即決できる」というKPIを特定する。 |
| **Q9** | **「10年後のビジョン（理想）を語ったとき、現場が『ポカンとしている』あるいは『困惑している』と感じることはありますか？それはなぜだと思いますか？」** | **[翻訳ロスの自覚]**<br>・ご自身の「思いつき」が現場で「絶対命令」や「混乱」に変換されている自覚があるか。<br>・そのギャップを埋める「翻訳者」に誰を据えたいか。 |

### Part 3: 【Style】仕事のスタイル・価値観・スキル

**狙い:** なーすけ様の**「取扱説明書」**を作る。スキルセットと感情のトリガーを知る。

| No. | 質問項目 (Question) | 深掘り・意図 (Intent & Deep Dive) |
| :--- | :--- | :--- |
| **Q10** | **「他の人がやると1時間かかるけど、なーすけさんなら5分で（息をするように）できることは何ですか？」** | **[不当な優位性 (Unfair Advantage)]**<br>・これこそが「コア・スキル」。<br>・例：初対面の相手の懐に入ること、カオスな状況を楽しめること、物語を作ること。 |
| **Q11** | **「仕事をする上で、『これだけは絶対に許せない』『生理的に無理』と感じることは何ですか？」** | **[価値観の境界線 (Landmines)]**<br>・Shapeの文化として定着させるべき「Do Not」リスト。<br>・例：嘘をつく、遅い、熱量がない、ダサい等。 |
| **Q12** | **「新しいアイデアを思いついた時、誰に最初に話したいですか？また、その相手には『ブレーキ』と『アクセル』のどちらを期待しますか？」** | **[参謀への期待値]**<br>・Go前提で「実現方法」を一緒に考えてほしいのか（アクセル）。<br>・リスクや既存計画への影響を冷静に指摘して止めてほしいのか（ブレーキ）。<br>・鈴木の立ち回りを決定する。 |
| **Q13** | **「法人研修のリード役として、ご自身の『現場稼働』はいつまで続けたいですか？」**<br>（ずっと現役で教えたい vs 早期に仕組み化して抜けたい） | **[プレイヤー指向 vs マネジメント指向]**<br>・「自分が教えること自体が価値」なのか「メソッドが価値」なのか。<br>・採用計画（認定トレーナーの必要時期）とリソース計算に直結する。 |

## 4. メモ

  * **確認すべき「表情」**:
      * **Q6（任せる恐怖）**を聞いた時、即答できるか、言い淀むか。言い淀む場合、まだ「言語化できていない美学」が深く眠っています。「例えば、去年のSFFで『違うな』と思った瞬間は？」など、過去の事例から掘り下げてください。
  * **想定される回答パターンと対策**:
      * **パターンA「全部大事（聖域だらけ）」**: Q2で全て自分でやりたいと言う場合。「物理的に時間が足りない時、どれを捨てますか？」と極限の選択を迫ってください。
      * **パターンB「数字は苦手」**: Q5, Q7, Q8で数字を見たくない反応の場合。「数字を見なくて済むように、誰に財布を預けますか？」と**RACI（役割分担）の合意**を取り付けてください。
  * **回答後のアクションへの接続**:
      * **Q2（聖域）**: ここで挙がらなかった業務は、鈴木・神保・町田（敬称略）へ渡す組織図の原案になります。
      * **Q5（3年後） & Q7（SFF定義）**: ここの回答で、作成するPLの「売上構成比」と「利益率設定」が決まります。
      * **Q12（ブレーキ）**: 鈴木のスタンス（イケイケで伴走するか、リスク管理の門番になるか）がここで決まります。
  * **記録のポイント**:
      * 回答の「内容」だけでなく、「熱量が高まった瞬間（早口になった、身を乗り出した）」と「トーンダウンした瞬間」をメモしてください。それが彼の**「Zone of Genius（天才性）」**のヒントです。
