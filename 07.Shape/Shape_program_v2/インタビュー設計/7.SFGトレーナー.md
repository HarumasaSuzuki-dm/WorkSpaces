# インタビュー設計書：SFGトレーナー（ジム責任者）【再修正版】

## 1. インタビューの目的 (Objectives)

1.  **責任境界線の「発見」**: 本部と現場の間で「誰がボールを持つべきか」が決まっていないグレーゾーン（ポテンヒット領域）を洗い出し、これから定義するための材料を集める。
2.  **コミュニティの「種」の発掘**: まだ形の決まっていないコミュニティ事業について、現場の顧客リアクション（事実）に基づき、「どういう方向なら機能しそうか」という仮説をボトムアップで吸い上げる。
3.  **無意識の負担の可視化**: 「役職名」や「契約」に書かれていないが、現場の良心で回している"名もなき業務"を特定する。

## 2. タイムラインと構成 (60min)

  * **00-05分**: アイスブレイク（鈴木より：「決まっていないことが多いので、皆さんの『実感』を教えてください。これからルールを作るためのヒアリングです」）
  * **05-20分**: 【Definition】責任と役割の「境界線探索」（Where is the line?）
  * **20-35分**: 【Community】顧客の繋がりと「コミュニティの種」（Seeds of Connection）
  * **35-50分**: 【Operations】本部との距離感と現場の負担（Reality Check）
  * **50-60分**: クロージング（鈴木からのフィードバック約束）

## 3. 質問リスト詳細 (Interview Script)

### Part 1: 【Definition】責任と役割の「境界線探索」

**狙い:** 「責任が正しく設定されていない」という仮説を前提に、現状現場が\*\*「どこまでが自分の仕事だと思っているか（認識の範囲）」\*\*を確認する。

| No. | 質問項目 (Question) | 深掘り・意図 (Intent & Deep Dive) |
| :--- | :--- | :--- |
| **Q1** | **「現在、日々の業務の中で『ここまでが私の責任』で、『ここからは本部（または事務）の責任』という境界線は、どこにあると感じていますか？」** | **[責任範囲の認識]**<br>・「セッション中だけ」なのか「売上まで」なのか。<br>・不明確な場合、「誰の仕事か分からないボール」がどこに落ちているか聞く。 |
| **Q2** | **「もし、会員様が『退会したい』と言い出した時、それを引き止める（または理由を聞く）のは『誰の役割』だという認識ですか？」** | **[成果責任の所在]**<br>・「私の仕事」と答えるか、「システム上の手続き」と答えるか。<br>・LTV（継続率）に対する当事者意識の有無ではなく、「権限設定」の有無を確認。 |
| **Q3** | **「『これ、誰もやってないから私が拾ってるけど、本当は私の仕事じゃないよね？』とモヤモヤしているタスクはありますか？」** | **[グレーゾーン業務の特定]**<br>・掃除、備品発注、SNSの返信、クレーム対応など。<br>・これらが「職務記述書（JD）」を作るための生データになる。 |

### Part 2: 【Community】顧客の繋がりと「コミュニティの種」

**狙い:** 方向性が未定のコミュニティについて、机上の空論ではなく\*\*「現場の事実」\*\*から勝筋（勝ちパターン）を探る。

| No. | 質問項目 (Question) | 深掘り・意図 (Intent & Deep Dive) |
| :--- | :--- | :--- |
| **Q4** | **「ジムの中で、会員様同士が『自然に仲良くなる』とか『会話が生まれる』瞬間って、具体的にどんな時ですか？」** | **[自然発生的な交流の観察]**<br>・「ロッカールームで」「きついトレーニングの後で」「イベント前後で」。<br>・ここにコミュニティ設計のヒント（強制させないきっかけ）がある。 |
| **Q5** | **「もし今、本部が『会員様専用のオンライングループを作る！』と言い出したら、現場感覚として『盛り上がる』と思いますか？それとも『スベる』と思いますか？」** | **[肌感覚のFeasibility Study]**<br>・「スベる」ならなぜか。（例：客はトレーニングしに来てるだけで、交流は求めていない等）。<br>・現場が感じる「顧客の温度感」を確認。 |
| **Q6** | **「一番長く続いている（ロイヤリティが高い）お客様たちは、Shapeの『何』が好きで居続けてくれていると思いますか？」** | **[コア・バリューの特定]**<br>・「トレーナーの人柄」か「仲間がいるから」か「メソッド」か。<br>・「仲間」というワードが出るなら、コミュニティ化のポテンシャルが高い。 |

### Part 3: 【Operations】本部との距離感と現場の負担

**狙い:** 本部（経営層）との「認識のズレ」を確認し、業務プロセスの不備を洗い出す。

| No. | 質問項目 (Question) | 深掘り・意図 (Intent & Deep Dive) |
| :--- | :--- | :--- |
| **Q7** | **「本部（なーすけさん達）からの指示や方針で、『現場の状況を分かってないな』『理想論すぎるな』と感じて困ったことはありますか？」** | **[Vision vs Reality]**<br>・具体的な指示（例：全員に声をかけろ、イベントに呼べ等）と、現場の物理的限界とのギャップ。<br>・「無理です」と言える環境かどうかも確認。 |
| **Q8** | **「もし、ジムの運営ルールを『一つだけ』明日から自由に変えられるとしたら、何を変えますか？」** | **[最優先のボトルネック]**<br>・予約システム？ 報告フロー？ 設備の修繕？<br>・現場が最もストレスを感じている「運用バグ」を特定。 |
| **Q9** | **「現場で判断に迷った時（例：特別対応など）、『誰に聞けば正解が返ってくるか』は明確ですか？」** | **[エスカレーションフローの確認]**<br>・「紫音さんに聞くけど返事が遅い」「毎回違う人に聞いている」など。<br>・指揮命令系統の混乱がないか確認。 |

## 4. インタビュアー（鈴木）へのメモ

  * **コミュニティについての聞き方**:
      * 「どんなコミュニティがいいと思いますか？」とアイデアを聞くのではなく、\*\*「お客様は普段どんな話をしていますか？」**と**事実（Fact）\*\*を聞いてください。
      * アイデアは「外す」可能性がありますが、現場で起きている事実は嘘をつきません。そこから鈴木様が戦略を組み立てます。
  * **責任範囲についてのスタンス**:
      * 「なんでやってないの？」ではなく、\*\*「どこまでが自分の陣地だと思っていますか？」\*\*と、線引きを確認するスタンスで聞いてください。
      * 「決まっていないこと」自体が課題（Issue）として抽出されます。
