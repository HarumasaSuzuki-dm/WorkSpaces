### 2. 比較対象の整理

**何を比較するのか**
- 現時点で候補となりうるシステム・プラットフォームの選定
- キックオフ資料やTMC様からの情報では「vista（Databricks）」の存在が示唆されている
- 他の候補（SaaS型、スクラッチ開発、既存システム拡張など）との比較が必要

**論点**
- TMC様が既に検討している候補はあるか？（ヒアリングで確認予定だが、骨子段階では想定候補を設定）
- 想定候補案：
  1. vista（Databricks）連携型
  2. クラウドSaaS型（Salesforce、Kintone等）
  3. スクラッチ開発（AWS/Azure上に新規構築）
  4. 既存システム拡張（SharePoint + Power Automate強化）

---

### 3. 比較の観点（評価軸）

**TMC様の課題から導き出される評価軸**

1/21週次定例資料およびキックオフ資料から、以下の評価軸が重要と考えられます。

| 評価軸 | 具体的な内容 | 根拠（TMC様の困りごと） |
|--------|----------|------------------|
| **統一ID管理** | 共通の案件IDで全関係者が同じ情報を見られるか | 「共通のID管理ができない」 |
| **入力のしやすさ** | デジタルリテラシーが低い施工会社でも使えるか | 「施工会社のリテラシーがバラバラ」 |
| **フォーマット統一** | 各社バラバラのフォーマットを統一できるか | 「書類フォーマット、ファイル形式がバラバラ」 |
| **ステータス管理** | 工事の進捗を一覧で把握できるか | 「工事がいつ終わるかわからない」 |
| **セキュリティ** | 個人情報を安全に管理できるか | 「個人情報が紐づくのでTQ-NETには入れられない」 |
| **既存システム連携** | ai21、TQ-NET等との連携可否 | 「充電器はフレームと紐づけられない」制約 |
| **コスト** | 初期費用・ランニングコスト | キックオフ資料「低コスト」の要件 |
| **導入リードタイム** | セキュリティ審査を含めた導入期間 | 「開発キックオフから約3ヶ月の審査期間」 |

### 4. アウトプット形式・フォーマット

**想定される構成**

```
1. エグゼクティブサマリー（1ページ）
   - 結論・推奨案
   - 主要な比較ポイント

2. 比較対象システム概要（各1ページ）
   - 製品/サービス名
   - 特徴・強み
   - TMC様環境での適用イメージ

3. 比較表（1〜2ページ）
   - 評価軸×候補システムのマトリクス
   - ◎○△×の定性評価 + 補足コメント

4. 詳細評価（各観点2〜3ページ）
   - 各評価軸ごとの詳細比較
   - メリット・デメリット

5. リスクと留意事項
   - 各候補の導入時リスク
   - セキュリティ審査の観点

6. 推奨案と次のステップ
```

**フォーマットの論点**
- PowerPoint形式 vs Word形式 → TMC様の社内承認フローに合わせる
- 比較表はExcelで別添するか、本文に埋め込むか
