## 利用想定
調査にあたり、以下の利用条件を前提としてください。
- 社内ユーザー：管理者 約10名、一般ユーザー 約50名
- 社外ユーザー（外部企業）：約4,500名（うち約4,000店舗＋約500社）がシステムにアクセスする必要がある
- 取扱データ：個人情報（氏名・住所等）、業務記録（写真・図面含む、1件あたり数十枚）、契約情報
- 主な用途：案件管理、進捗ステータス管理、入力フォーム、帳票出力、承認ワークフロー、通知・アラート、ダッシュボード

## 調査項目

### A. サービス概要
1. 正式名称と最新のエディション/バージョン
2. 提供元企業と日本法人の有無
3. 主要な特徴（3〜5つ）
4. 大企業・製造業での導入実績（あれば）
5. 日本国内での導入規模感

### B. ライセンス・料金体系
1. プラン/エディション一覧と各月額・年額単価
2. 社内ユーザー（管理者10名＋一般50名）の場合の費用
3. **社外ユーザー約4,500名がアクセスする場合のライセンス形態・概算コスト**（外部ポータル、ゲストアクセス等の方式と料金）
4. ストレージ費用（大容量の写真・図面データ格納を想定）
5. 初期導入費用の目安
6. 月額ランニングコスト合計の概算
7. 5年間総コスト概算（初期＋ランニング×60ヶ月）

### C. 機能対応状況
以下の各機能について、**標準機能/カスタマイズ/追加開発が必要/対応困難** のいずれかを明記し、具体的な実現方法を記載してください。

1. **統一ID管理：** 案件への一意の管理番号の自動採番。全ユーザーが同一IDで参照可能か
2. **ステータス管理：** 多段階ステータスの一覧管理・表示。一定期間停滞時の自動アラート
3. **入力フォーム：** カスタムフォーム作成。スマートフォン対応。ITリテラシーが低いユーザーでも使えるシンプルなUI
4. **ファイル添付・ストレージ：** 写真・図面の添付・格納。1件あたり数十枚。容量制限と拡張性
5. **帳票・出力：** テンプレートからのPDF出力（報告書・見積書・保証書等）
6. **承認ワークフロー：** 多段階承認フロー（3段階以上）。差し戻し・再申請
7. **通知・アラート：** メール/アプリ内通知。ステータス変更時の自動通知。期限超過アラート
8. **ダッシュボード・KPI：** 集計・グラフ表示。カスタムダッシュボード構築
9. **アカウント・権限管理：** RBAC。組織単位でのデータ分離（マルチテナント的運用）
10. **監査ログ：** 操作履歴の記録・閲覧（誰が・いつ・何を変更したか）

### D. 外部ユーザー（社外）対応
1. 外部ユーザーのアクセス方式（外部ポータル、ゲストアカウント、専用アプリ等）
2. アカウント発行・管理方法（一括発行、セルフサービス登録等）
3. データアクセス制限の柔軟性（自社案件のみ閲覧等）
4. 約4,500名利用時の性能・スケーラビリティ
5. 認証方式（SSO、ID/パスワード、多要素認証等）

### E. セキュリティ・コンプライアンス
1. ISMAP登録状況
2. SOC報告書（SOC1/SOC2/SOC3）の取得状況
3. ISO認証（27001/27017/27018等）の取得状況
4. データ暗号化（保存時・通信時の方式）
5. データ所在地（日本リージョンの有無）
6. バックアップ（頻度・RPO/RTO）
7. IP制限・条件付きアクセス
8. 監査ログの保持期間
9. 個人情報保護対応（データマスキング、匿名化等の機能）

### F. 外部連携
1. **Databricks連携：** 連携方法（API、コネクタ、ETL等）。具体的なコネクタ名があれば記載
2. REST API / GraphQLの提供状況とレートリミット
3. CSV/Excel/JSONの一括インポート・エクスポート
4. SharePoint・Teams・メール（Outlook等）との連携
5. Webhook/トリガーによるリアルタイム連携の可否

### G. 導入・運用
1. 環境構築から本番稼働までの標準的な導入期間（上記利用規模を想定）
2. 開発方式（ノーコード/ローコード/プロコード）
3. 日本語UI対応状況
4. 日本語での技術サポート体制・国内サポート拠点
5. SLA（稼働率保証）
6. 管理者向け管理画面の充実度
7. 公式トレーニング・日本語ドキュメントの充実度

### H. 拡張性
1. データ項目追加の容易性
2. 業務パターン追加の柔軟性
3. AI/ML機能の活用可能性
4. モバイルアプリの提供状況
5. ユーザー数・データ量増加時のスケールアップ対応
