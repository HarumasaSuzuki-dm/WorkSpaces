## 2. 解決したい課題（TMC様の困りごと）

### 2.1 課題の全体像

#### 【最大の課題】催促業務の負担 
「出してください」という催促・リマインドが日常的に発生
→ 今どの工程か分からないから、催促していいかも分からない

#### 【根本原因】情報が見えない・散らばっている
- 工事記録がなかなか上がってこない
- 工事予定・完了時期が把握できない
- 発注数と工事申込数と工事記録数が突合できない
- 受領手段がバラバラ（メール、Teams等）
- 共通ID管理ができていない  
- フォーマット・ファイル形式がバラバラ

#### 【運用上の制約】関係者の多さ・リテラシー差
- 販売会社：全国約250社
- 販売店：全国約4,000店
- 施工管理会社：トヨタホーム＋地場の施工管理会社（不特定）
- 施工会社：各施工管理会社と連携する会社（不特定）
 → デジタルリテラシーに大きな差がある 

### 2.2 数字で見る現状

| 項目 | 数値 | 備考 |
|------|------|------|
| 販売会社数 | 約250社 | - |
| 販売店数 | 約4,000店 | - |
| トヨタホーム利用率 | 約50%（確認中） | 全国販売会社の約半数 |
| 工事記録の自動化対象 | トヨタホームのExcel形式のみ | PDF形式は対応不可 |
| セキュリティ審査期間 | 約3ヶ月 | 開発キックオフ〜リリースまで |

### 2.3 アフターサービス業務の課題

| 課題 | 詳細 |
|------|------|
| 修理完了報告書の滞留 | 不具合発生時、報告書・保証求償書の提出が遅れがち |
| 二重管理の発生 | 個人情報を含むExcelを仕入先と共有できず、複数Excelで管理 |
| ステータス一覧管理不可 | 1件1件の成績確認に工数がかかる |
| 分析困難 | 不具合件数・種別での分析をExcel手作業で実施 |

## 4. システム候補の比較

### 4.1 候補一覧

| 候補 | 名称 | 概要 | 位置づけ |
|------|------|------|---------|
| A | **Microsoft Power Apps + Dataverse** | TMC社内データ基盤（vista）と連携可能なMicrosoft製品 | **本格策（推奨）** |
| B | **Salesforce Platform** | 世界最大手のクラウド業務システム基盤 | 本格策（代替案） |
| C | **kintone** | 国産の業務アプリ作成プラットフォーム | 本格策（代替案） |
| D | **既存ツール強化** | SharePoint + Power Automate拡張 | 応急措置 |

### 4.2 候補別の特徴

#### 候補A：Microsoft Power Apps + Dataverse（推奨）

**メリット**
- TMC社内で既に利用実績あり（Microsoft 365環境）
- 社内データ基盤（vista）との将来的な連携が容易
- セキュリティ審査が比較的スムーズ（ISMAP登録済）
- 外部ユーザー500名規模に対応可能（Power Pages）

**デメリット**
- カスタマイズには一定の技術知識が必要
- 複雑な要件の場合は追加開発費用が発生

**コスト目安（月額）**

| 項目 | 内訳 | 金額 |
|------|------|------|
| 管理者（10名） | Power Apps Premium × 10 | 約22,000円 |
| 社内ユーザー（50名） | Power Apps Per App × 50 | 約30,000円 |
| 社外ユーザー（500名） | Power Pages認証ユーザー | 約150,000円 |
| ストレージ追加 | 約120GB分 | 約3,600円 |
| **合計** | | **約20〜30万円/月** |

**セキュリティ・コンプライアンス**

| 項目 | 対応状況 |
|------|---------|
| 個人情報の取り扱い | ◎ 利用規約で許容（GDPR/APPI対応） |
| データ所在地 | ◎ 日本リージョン選択可能（東京・大阪） |
| SOC報告書 | ◎ SOC1/SOC2 Type II 取得済 |
| ISO認証 | ◎ ISO27001/27017/27018 取得済 |
| ISMAP | ◎ 登録済 |
| 暗号化 | ◎ 保存時・通信時ともにAES-256 |
| 監査ログ | ◎ 操作履歴を最大68年保存可能 |

#### 候補B：Salesforce Platform

**メリット**
- 世界最大の導入実績、機能が豊富
- CRM機能との統合が強力
- 外部ポータル（Experience Cloud）が成熟

**デメリット**
- ライセンス費用が高め
- TMC社内基盤との連携に追加開発が必要
- セキュリティ審査を新規で実施する必要あり

**コスト目安（月額）**：約50〜80万円

#### 候補C：kintone

**メリット**
- 国産製品で日本語サポートが充実
- 直感的なUIで習得が容易
- 中小企業での導入実績が豊富

**デメリット**
- 大規模（4,000店舗）での利用実績が少ない
- 外部ユーザーの大量招待に制約あり（ゲストスペース）
- 高度なワークフローには追加開発が必要

**コスト目安（月額）**：約15〜25万円

#### 候補D：既存ツール強化（応急措置）

**メリット**
- 追加費用なし（既存ライセンス範囲内）
- すぐに開始可能
- セキュリティ審査不要

**デメリット**
- 根本的な課題解決にならない（催促業務は残る）
- 4,000店舗の外部ユーザー管理が困難
- 自動化できる範囲が限定的

**位置づけ**：本格システム稼働までの「つなぎ」として活用

## 5. 候補A（Power Apps + Dataverse）の詳細評価

### 5.1 必須機能への対応

| 機能 | 対応状況 | 詳細 |
|------|---------|------|
| **統一ID管理** | ◎ | 自動採番機能あり（例：CJR-0001001） |
| **ステータス管理** | ◎ | 選択肢フィールドでカスタム定義可能 |
| **入力画面** | ◎ | スマホ対応、オフライン入力も可能 |
| **ファイル添付** | ◎ | 1ファイル128MBまで、容量追加購入可 |
| **出力機能** | ◎ | Excel/CSV出力、PDF帳票生成可能 |
| **承認ワークフロー** | ◎ | Power Automateで多段階承認を構築 |
| **監査ログ** | ◎ | 操作履歴・変更履歴を自動記録 |
| **権限管理** | ◎ | ロールベース＋行・列単位の制御 |

### 5.2 外部ユーザー（施工会社等）の利用

| 項目 | 内容 |
|------|------|
| アカウント発行方法 | Azure AD B2Bゲスト招待 または Power Pagesポータル |
| ライセンス形態 | Power Pages認証ユーザー（100名/$200/月） |
| 500名の場合のコスト | 約$1,000/月（約15万円） |
| アクセス制御 | 自社案件のみ閲覧可能（行レベルセキュリティ） |
| アカウント管理 | 管理者によるCSV一括登録・削除可能 |

### 5.3 セキュリティ審査への準備

| 審査項目 | 対応状況 | 備考 |
|---------|---------|------|
| 個人情報の保存 | ◎ | 利用規約上許容、顧客がデータ管理者 |
| データの国内保管 | ◎ | 日本リージョン選択で国内完結 |
| 海外への転送 | ◎ | サポート時の一時アクセス以外なし |
| SOC報告書 | ◎ | Service Trust Portalで入手可能 |
| 暗号化（保存時） | ◎ | TDE（AES-256）標準適用 |
| 暗号化（通信時） | ◎ | TLS 1.2以上必須 |
| 多要素認証 | ◎ | Azure AD条件付きアクセスで設定 |
| IPアドレス制限 | ◎ | Azure AD条件付きアクセスで設定可能 |
| 監査ログ | ◎ | 保存期間は最大68年まで設定可能 |

## 7. 参考情報

### 7.1 用語説明

| 用語 | 説明 |
|------|------|
| Power Apps | Microsoftが提供するローコード開発プラットフォーム。プログラミング知識がなくても業務アプリを作成可能 |
| Dataverse | Power Appsのデータを保存するデータベース。セキュリティやワークフローの機能も含む |
| Power Pages | 外部ユーザー（社外）向けのWebポータルを作成する機能 |
| Power Automate | 業務の自動化ツール。承認フローや通知の自動送信などを設定可能 |
| vista | TMC社内のデータ管理基盤（Databricks）。FY26以降のデータ活用の中核となる予定 |
| Azure AD | Microsoftのユーザー管理・認証サービス。多要素認証やシングルサインオンを提供 |
| ISMAP | 政府情報システムのためのセキュリティ評価制度。登録済みのクラウドは政府機関での利用が認められている |
| SOC報告書 | 外部監査人による内部統制の評価報告書。SOC1は財務報告、SOC2はセキュリティ・可用性等を評価 |
