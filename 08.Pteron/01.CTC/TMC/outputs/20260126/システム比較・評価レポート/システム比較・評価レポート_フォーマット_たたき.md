## 2. 解決したい課題（TMC様の困りごと）

### 2.1 課題の全体像

#### 【最大の課題】催促業務の負担

「出してください」という催促・リマインドが日常的に発生
→ 今どの工程か分からないから、催促していいかも分からない

#### 【根本原因】情報が見えない・散らばっている

- 工事記録がなかなか上がってこない
- 工事予定・完了時期が把握できない
- 発注数と工事申込数と工事記録数が突合できない
- 受領手段がバラバラ（メール、Teams等）
- 共通ID管理ができていない
- フォーマット・ファイル形式がバラバラ

#### 【運用上の制約】関係者の多さ・リテラシー差

- 販売会社：全国約250社
- 販売店：全国約4,000店
- 施工管理会社：トヨタホーム＋地場の施工管理会社（不特定）
- 施工会社：各施工管理会社と連携する会社（不特定）

→ デジタルリテラシーに大きな差がある

### 2.2 数字で見る現状

| 項目 | 数値 | 備考 |
|------|------|------|
| 販売会社数 | 約250社 | - |
| 販売店数 | 約4,000店 | - |
| トヨタホーム利用率 | 約50%（確認中） | 全国販売会社の約半数 |
| 工事記録の自動化対象 | トヨタホームのExcel形式のみ | PDF形式は対応不可 |
| セキュリティ審査期間 | 約3ヶ月 | 開発キックオフ〜リリースまで |

### 2.3 アフターサービス業務の課題

| 課題 | 詳細 |
|------|------|
| 修理完了報告書の滞留 | 不具合発生時、報告書・保証求償書の提出が遅れがち |
| 二重管理の発生 | 個人情報を含むExcelを仕入先と共有できず、複数Excelで管理 |
| ステータス一覧管理不可 | 1件1件の成績確認に工数がかかる |
| 分析困難 | 不具合件数・種別での分析をExcel手作業で実施 |

## 5. 候補A（Databricks vista）の詳細評価

### 5.1 vistaの概要

Databricksは、データの収集・蓄積から分析・機械学習モデル開発・運用までを一気通貫で効率的かつ安価に行うことができる統合プラットフォームです。

TMC様では、FY26以降の「データドリブン経営」に向けて、社内のデータ一元管理環境の構築を開始しており、FY25に構築する「会社全体のデータを見渡せる場所」をvistaと命名し、現在CTCにてシステム構築中です。

**vistaの主な目的**

- 企業内のデータを一元管理し、全体像を把握できる環境を提供する
- 効率的にデータを処理・分析し、アプリケーションを開発できる環境を整備する
- データガバナンスとセキュリティを強化し、データの安全性を確保する

### 5.4 推奨理由のまとめ

1. **既存基盤との親和性：** TMC社内で既に構築中のvistaと直接連携でき、将来的なデータ活用の中核となる
2. **セキュリティ審査の効率化：** 既存基盤のため新規審査が不要または大幅に短縮される可能性がある
3. **将来的な拡張性：** AI/ML機能を活用した不具合予測・分析など、将来的なデータ活用が可能
4. **統合的なデータ管理：** 充電器販売・施工データを全社データと統合管理でき、経営判断への活用が期待できる
