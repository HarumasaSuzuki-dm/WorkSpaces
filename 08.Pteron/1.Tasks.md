# プロジェクト参画に向けた準備計画（Preparation Plan）

**ゴール:** 12月のプロジェクト参画初日から即座に価値を発揮できるよう、圧倒的な事前準備を完了させる。

---

## フェーズ1： 10/30のミーティングで「答え合わせ」と「深掘り」を行う

**目的:**
- `0.2.Issues.md` の論点整理の精度を検証する。
- 自身の分析力と思考の深さを示し、ポジティブな第一印象を形成する。
- プロジェクトの深層にある、ドキュメント化されていない情報を引き出す。

**アクション:**
- ミーティングの冒頭で、「`0.0.PreInfo.md` を元に作成した論点整理（`0.2.Issues.md`）の認識が正しいか、皆さんのご意見を伺いながら深掘りしたい」と切り出す。
- `0.2.Issues.md` に追記した「戦略的質問」を起点に、議論をファシリテートする。

---

## フェーズ2： 参画までの自己学習ロードマップ（11月）

**目的:** プロジェクトの主要な「言語」と「文脈」を完全にマスターし、即戦力となるための知識基盤を構築する。

### 1週目： 体系的知識の再確認と応用
- **アクション:**
    - PMBOKの「統合」「ステークホルダー」「コスト」「リスク」管理の知識エリアを、本プロジェクトの文脈に当てはめて再読する。
    - COBIT/ITILの概要を把握し、IT投資の意思決定フレームワークとしての側面を理解する。

#### 【詳細学習計画】1週目：体系的知識の再確認と応用

**目的:** プロジェクト固有の課題（組織の壁、財務との対立、長期リスク）に対応するため、PMBOKとITガバナンスの知識を理論と実践の両面から深掘りする。

| 知識エリア | 学習内容（理論） | 【重点】学習内容（実践） |
| :--- | :--- | :--- |
| **統合マネジメント** | - プロジェクト憲章、計画書、変更管理、終結プロセスのインプット/ツール/アウトプットを再確認する。 | - **超長期・多段階（Wave）移行における統合変更管理:** ビジネス要件の変更やスコープクリープに対し、PGMOとしてどう影響を評価し、どの会議体で、誰の承認を得るべきかをシミュレーションする。 |
| **ステークホルダー・マネジメント** | - ステークホルダー特定、エンゲージメント計画、パワー/インタレスト・マトリクス等のツールを復習する。 | - **非公式な力関係の分析:** 「根回しが大変」な状況を乗り越えるため、公式の体制図にはないキーパーソン（インフルエンサー、抵抗勢力）を特定し、各々の関心事と影響力を分析する仮想ステークホルダーマップを作成する。特に**財務部門、工場部門、IT部門**間の力学を重点的に考察する。 |

| **リスク・マネジメント** | - リスク特定、定性的・定量的分析、リスク対応計画、リスク登録簿の作成プロセスを再確認する。 | - **長期プロジェクト特有のリスクシナリオ分析:** 「主要メンバーの離脱」「技術の陳腐化」「ビジネス環境の変化」といった具体的なリスクシナリオに対し、PGMOとしてどのような予防策とコンティンジェンシープランを策定すべきか考察する。 |

**2. ITガバナンス・フレームワークの戦略的理解**

| フレームワーク | 学習内容（理論） | 【重点】学習内容（実践） |
| :--- | :--- | :--- |
| **COBIT** | - ITと事業戦略を整合させるための原則と構成要素を理解する。「何をすべきか（What）」のフレームワークであることを認識する。 | - **「IT投資のビジネス価値」の可視化:** PGMOとして単なる「進捗管理屋」ではなく、400億円の投資が「Value Chain全体の業務改革」というビジネス目的にどう貢献するのかを、**COBITのフレームワークを用いて経営層に説明するストーリー**を構築する。 |
| **ITIL** | - ITサービスマネジメントのライフサイクル（サービスストラテジ、デザイン、トランジション、オペレーション、継続的改善）の概要を把握する。「どうすべきか（How）」のベストプラクティス集であることを認識する。 | - **刷新後を見据えた品質設計:** SAP S/4HANA導入後の**安定運用（可用性、パフォーマンス、インシデント管理等）**を見据え、プロジェクトのどの段階で、どのような非機能要件を定義・担保すべきかを考察する。これはPwC/IBMとの差別化ポイントになり得る。 |

### 1〜2週目： 事業ドメイン知識の徹底的インプット
- **アクション:**

    - タイヤ業界のサプライチェーン（製造・物流・販売）に関する書籍や記事を読み、業務フローの解像度を上げる。

#### 【詳細学習計画】1〜2週目：事業ドメイン知識の徹底的インプット

**目的:** プロジェクトの核心である「Value Chain全体の業務改革」の議論に深く関与するため、ブリヂストンの事業構造、業務プロセス、業界動向を解像度高く理解する。

---

**1. 経営・事業戦略の理解（マクロ分析）**

| 学習テーマ | インプット資料 | 【重点】学習内容（着眼点） |
| :--- | :--- | :--- |
| **ブリヂストンの経営課題と戦略** | - 有価証券報告書（事業の状況、事業等のリスク）<br>- 中期経営計画（2024-2026）<br>- 決算説明会資料 | - **400億円投資の背景:** このDXプロジェクトが、中期経営計画の中で**どの経営課題（例：収益性向上、サステナビリティ、新事業創出）を解決するために位置づけられているか**を特定する。 |
| **DX戦略の具体像** | - 統合報告書<br>- G-CDXOの対外発表資料、インタビュー記事 | - **「グローバル標準化」の狙い:** なぜ今、グローバルで業務とシステムを標準化する必要があるのか。それによって**どのようなビジネスメリット（例：データドリブン経営の実現、グローバルでの迅速な意思決定）**を期待しているのかを読み解く。 |
| **競合分析** | - 競合（ミシュラン、グッドイヤー等）のIR資料、中期経営計画 | - **業界全体のDXトレンド:** 競合他社はどのようなDXに取り組んでいるか。ブリヂストンのプロジェクトは、業界内で先進的なのか、それともキャッチアップなのかを把握する。 |

**2. 業務プロセスの理解（ミクロ分析）**

| 学習テーマ | インプット資料 | 【重点】学習内容（着眼点） |
| :--- | :--- | :--- |
| **サプライチェーン（SCM）** | - 業界レポート、専門書籍<br>- （可能であれば）ブリヂストンの工場見学レポートやCSRレポート | - **計画〜実行のフロー:** 「需要予測→生産計画→調達→製造→在庫管理→物流→販売」の一連のプロセスと、各プロセス間の**情報の流れ（分断や遅延が課題となりがちなポイント）**を理解する。SCM刷新の「痛み」と「理想像」を具体的にイメージできるようにする。 |
| **量産工場（MES）** | - 製造業・MESに関する解説記事<br>- スマートファクトリーに関する資料 | - **製造現場のデータ:** 工場でどのようなデータ（生産実績、品質、設備稼働率など）が収集され、それがどのように経営管理（原価計算など）に繋がっているのかを把握する。工場システムの刷新が、**原価管理の精度向上や生産性向上**にどう貢献するかを考察する。 |
| **販売・会計** | - 財務諸表（P/L, B/S, C/F）の基礎知識<br>- 業界の販売チャネルに関する記事 | - **お金の流れ:** タイヤが販売されてから売上として計上され、代金が回収されるまでのモノとカネの流れを理解する。グローバルでの**決算早期化や連結会計の効率化**がなぜ重要なのかを把握する。 |

### 3週目： 技術ドメイン知識の基礎固め
- **アクション:**

    - 「メインフレームの課題」「技術的負債」といったキーワードで、レガシー刷新の必要性を自身の言葉で説明できるように準備する。

#### 【詳細学習計画】3週目：技術ドメイン知識の基礎固め

**目的:** プロジェクトの技術的な議論（特にSAP S/4HANA導入）に主体的に貢献し、PwCやIBMなどの技術パートナーとも対等に会話できるだけの共通言語を習得する。

---

**1. 新システム（To-Be）の技術スタック理解**

| 学習テーマ | インプット資料 | 【重点】学習内容（着眼点） |
| :--- | :--- | :--- |
| **中核技術: SAP S/4HANA** | - SAP公式の解説ドキュメント<br>- SAPコンサルタントが書いたブログ、書籍 | - **モジュール構成:** プロジェクト範囲である**会計(FI/CO)、販売(SD)、購買(MM)、生産(PP)**などの主要モジュールの役割と概要を理解する。<br>- **"Fit to Standard"の思想:** 「グローバル標準化」の鍵となる、業務をSAPの標準機能に合わせる考え方と、それがなぜ重要なのか（アドオン開発の抑制、TCO削減）を説明できるようにする。 |
| **システム・アーキテクチャ** | - クラウド vs オンプレミスに関する一般的な解説記事<br>- API連携、ETL/EAIツールに関する技術記事 | - **連携方式:** SAPとSCM、MES（工場）等の周辺システムが、**どのような技術（API, IDoc等）で、どの程度の頻度（リアルタイム, バッチ）でデータ連携するのか**、そのアーキテクチャの一般的なパターンを理解する。これが業務プロセス全体の効率を左右する。 |

**3. 移行方法論の理解**

| 学習テーマ | インプット資料 | 【重点】学習内容（着眼点） |
| :--- | :--- | :--- |

### 4週目： 実践的知識のシミュレーション
- **アクション:**

    - Capex/Opex、減価償却、ROIといった財務会計の基礎用語が、IT投資管理でどう使われるかを復習する。
